North-star status dashboard - v0.99.363a

Major goals

Hey source -> LLVM-hosted Hey -> semantic HIR -> ABI MIR -> direct LLVM -> native executable

The private native bootstrap remains seed/recovery infrastructure. Actors,

stateless workers, Jobs, bounded streams, scheduler, and reactor remain one

runtime-managed concurrency model. Native resources remain opaque capabilities;

shared mutable state is not a general public language primitive.

Deployment means one self-contained target archive per OS/architecture. Internal

compiler/linker/runtime/stdlib files are allowed; recipient C/C++ toolchains and

instructions to weaken endpoint security are not. Public CLI porcelain stays

small while release/test plumbing remains below `bin/subcmds` and `tools`.

Current milestone

The next milestone is canonical CamelCase module closure:

one public module spelling
  -> imports, qualified calls, generated text, docs, packages, examples, specs
  -> semantic HIR -> ABI MIR -> direct LLVM
  -> no maintained lowercase compatibility namespace

v0.99.363a removed the benchmark-only mutable collection and proved that the

four maintained 1BRC programs work as ordinary Hey through the fixed-point

pipeline. Module naming is therefore the next dependency to stabilize before

Time/Duration coherence, independent package migrations, or television targets.

Current evidence

Compiler and ordinary applications

  • All four maintained 1BRC applications use read-only mapped bytes, ordinary

immutable object records, reusable Hey functions, actor-owned partial values,

and general byte-buffer output.

  • The specialized mutable byte-keyed table module, runtime kind, interpreter and

REPL dispatch, native-C lowering, direct-LLVM lowering/declarations, public

documentation, fixtures, and product tests are removed.

  • Semantic HIR and ABI MIR classify `div(...)` as scalar integer and

`json(text)` as boxed parsed data.

  • Mixed integer, boolean, and boxed values stringify through the general boxed

LLVM emitter; nested properties bind to their receiver instead of a same-named

lexical field.

  • Native-C output, both independent Hey frontend hosts, HIR, MIR, and direct LLVM

agree on the focused value-facts fixture. Four direct-LLVM application builds

produce exact deterministic default output.

  • Naive Fibonacci still uses ordinary recursive LLVM calls while the proven

accumulator retains LLVM `musttail`.

Package consumption

  • `hey package ensure --update-lock` resolves complete transitive graphs from

local or bounded HTTP/HTTPS registry/release/archive sources.

  • Selection is deterministic highest-compatible with backtracking; cycles,

impossible intersections, unsafe archives, checksum drift, command

collisions, and partial publication fail closed.

  • Lock v2 binds project identity, root declarations, complete graph edges,

release/archive/manifest/content hashes, exact archive sources, and command

registrations.

  • Locked restore is registry-independent and byte-exact; `verify` is offline and

complete-tree; `audit` rechecks mutable publication evidence and rejects

same-version content replacement.

  • Package authoring, release generation, website hosting, Git publication, and

application dependency selection remain independently owned.

Handoff and release evidence

  • Source archives carry an exact installer and a processor that verifies in a

disposable Git worktree, streams output, retains heartbeat/log receipts,

commits the verified candidate there, and fast-forwards only a clean unchanged

maintainer checkout.

  • `VERIFY-HEY-RELEASE.sh` attempts every scheduled gate after failures and keeps

one timed log per gate.

  • Linux x64 and Darwin arm64 target archives remain distinct product receipts;

Windows remains structural SDK/runtime-harness evidence until a contained

product compiler archive exists.

Known blockers

  • Public module filenames, import aliases, qualified references, generated text,

documentation links, package fixtures, and source inventories still use mixed

naming conventions. v0.99.364a must migrate them atomically.

  • Some direct-LLVM classifiers remain parallel to HIR/MIR facts. The focused

repairs are generalized and executable, but long-term ABI authority should

continue moving into MIR.

  • Release C/A/B and promotion are host-intensive and unclaimed unless the exact

commands complete.

  • Windows product closure, Darwin signing/notarization, complete license/SBOM

inventories, reproducible CI, and publication signatures remain incomplete.

  • Independent `hey_*` migrations require actual source releases and manifests.

Package names alone are not callable contracts.

  • Physical-device tvOS/Android TV, app-store publication, and the package-owned

party-quiz vertical slice remain future receipts.

Direction reflection

The evidence supports continuing with general compiler facts and ordinary values.

Deleting the one-benchmark mutable surface simplified every lane and exposed

reusable value/receiver improvements. The next migration should likewise prefer

one canonical module graph over aliases. Television work remains correctly

ordered after module and time stability, with application policy outside core.

Mobile and television extension

The same HIR/MIR pipeline expands through native iOS/iPadOS and Android, then

tvOS and Google TV/Android TV. Hey core owns target triples, object emission,

runtime ABI, callbacks, SDK inspection, capability discovery, and truthful

receipts. Packages own native TV adapters, QR rendering, service composition,

configuration, persistence, and UI contracts. Applications own questions,

lobby behavior, scoring, visual design, deployment, credentials, and cadence.
