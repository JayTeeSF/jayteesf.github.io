Hey Language Bootstrap Plan

Hey-lang is an alpha compiled language for small, readable programs that still want native builds, actor-style concurrency, immutable data, built-in specs, standard-library web tooling, and a path toward self-hosted LLVM-based tooling.

The command is `hey`, the compiler is `heyc`, and source files use `.hey`.

Current backend direction

  • `native-c` is the default oracle backend.
  • `self-hosted-c` is selectable and used as a dogfooding proof.
  • `llvm` is selectable and moving toward full backend-world coverage.

Detailed release history, version-by-version notes, and handoff status belong in release notes, not at the top of README files.

Build and test

make clean
make
./bin/test --full
make version-smoke docs-smoke source-zip-smoke

Release verification:

./bin/test --release-c
./bin/test --release-a
./bin/test --release-b

Full promotion proof:

time HEY_BACKEND_WORLD_JOBS=8 ./bin/test --promotion-report

Common commands

Run or test a file:

./bin/heyc path/to/file.hey --run
./bin/heyc path/to/file.hey --test

Build with a specific backend:

./bin/heyc build --backend native-c examples/hello.hey -o build/hello-native
./bin/heyc build --backend llvm examples/hello.hey -o build/hello-llvm

Generate user-facing docs:

./bin/hey-doc-site --source . --out build/hey_docs --theme bootstrap --jobs 8

Vendor a local Darwin dist into another app:

./bin/hey-vendor --repo ../elders_prayer_app --target darwin-arm64
