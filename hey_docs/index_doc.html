Hey Language Bootstrap Plan

Final name: **Hey-lang**. Command: `hey`. Compiler: `heyc`. Source extension: `.hey`.

Hey-lang is a proposed general-purpose compiled language with Ruby-like ease, C-like speed, Lisp-like composability, Erlang-like resilience, actor/supervisor concurrency plumbing, immutable core map/array values, built-in specification testing, and observability from the first compiler line.

Highest priorities now

Before returning to broad all-three-backend compatibility work, the release-readiness ergonomics gate is now at the top: CLI scripting, lightweight error handling/recovery, collection batteries, text/formatting, package workflow, debuggability, sockets/TLS/web runtime, and Track B canonical-symbol consumers. Error handling is deliberately not exception-first: use `fail`/supervisors for bugs and actor crashes, `result.ok(...)` / `result.error(...)` for expected failures, and add concise Ruby-like `rescue` later only if it stays cheap on the happy path. The long-term North Star is now explicit: Hey-written compiler/tools -> typed IR -> LLVM/object code -> native executable, with native-C kept as oracle/default until LLVM has full backend-world parity and self-hosted build time is controlled.

The bootstrap is now at **v0.99.162a**. The `a` suffix means alpha. Stay on explicit alpha patch versions until JayTeeSF approves a beta suffix or the real public `v0.0.1` line.

Current architecture truth:

• The plumbing/porcelain/data-model RFC is the architecture standard: scarce syntax, stable runtime plumbing, and user-friendly porcelain in libraries where possible.

• Timer and counter ideas are now documented as stdlib-first semantic intrinsics: public modules backed by compiler-known invariants, not public unsigned numeric types.

• Native-C remains the active oracle/default while LLVM is clean but incomplete.

• The North Star is self-hosted-on-LLVM: Hey-written compiler/tools should emit typed IR and then target LLVM/object code. Self-hosted-C remains the dogfooding proof; it is not the final center of gravity.

Implemented language/runtime surface

• I/O statements: `says`, `print`, `warn`, `write`, `log`, and `read`.

• Values: strings, bytes, integers, floats, booleans, `nil`, actors, arrays, and core map/object values.

• Immutable collection ergonomics: spread literals such as `[...xs, value]` and `{...obj, key: value}` are preferred in public examples.

• Map/array APIs: `get`, `at`, `put`, `push`, `len`, `length`, `has`, `del`/`delete`, `keys`, `values`, `entries`, `items`, `pairs`, `merge`, `type`, `shape`, and `shape_id`.

• JSON codec helpers: `json`, `to_json`, and `json_text`.

• Regex-lite helpers: `stdlib/re.hey` exposes `re.matches`, `re.contains`, `re.index`, `re.find`, `re.span`, `re.replace`, `re.scan`, `re.anchored`, `re.not_matches`, and `re.require_match` with anchors, wildcard dot, and `*`/`+`/`?` quantifiers while full regex parity is tracked.

• Bytes APIs: `b"..."`, `bytes`, `text`, `encode`, `decode`, `hex`/`from_hex`, `bin`/`from_bin`, `oct`/`from_oct`, `base64`/`from_base64`, `byte_at`, `slice`, and `is_utf8`.

• Multi-line core object/array literals and JS-object-style keys for valid identifiers.

• Functions/modules: `fn`, `module`, dotted calls, and module-scoped actors.

• Source imports: `import "file.hey"` links definitions from other files into one entry program.

• Actors/mailboxes: `actor`, `state`, `receive`, `spawn`, `send`, `join`, `stop`, and `exit`.

• Actor registry/introspection: `actors()`, `actor_info(actor)`, `pid(actor)`, `actor_by_pid(pid)`, and `send_pid(pid, message)`.

• Links/monitors: `link`, `monitor_to`, and `monitor_info`.

• Worker-pool actors through native builds with `--pool-actors`.

• Supervisor expression and jobs stdlib built over actors/supervisors.

• Evented I/O probe: `c10k_ping(client_count)`.

• 1BRC native runtime primitives: generated-data ramping, real-file aggregation, and summary-only proof mode.

• Native `heyc` command implemented in C for running source, specs, REPL, test suite, and native C backend builds.

• A C-native Hey REPL, not a Ruby REPL. It can import real Hey source files, including nested stdlib modules such as `stdlib/hey_c.hey`.

Current non-goals

• No package manager yet; v0.99.98a starts the package-workflow foundation with immutable package/dependency manifest values, but semver resolution, lockfiles, checksums, import aliases, registry trust, and packaging commands remain open.

• No user-facing async/await; Jobs plus actors/messages are the public concurrency model and async/await alternative.

• No active class/object syntax project; modules plus immutable core map/array values are the preferred direction unless a future RFC proves a performance-safe gap.

• No complete Erlang/OTP equivalent supervisor tree yet.

• Do not treat native full-suite success alone as backend parity; use `./bin/test --promotion-report` for parser/build/spec/REPL/backend-world self-hosted parity and LLVM lowering-impact evidence.

Next big goal

**Full three-backend compliance, sequenced by the [parity plan](docs/backend/parity-plan.md)**:

1. Keep Native-C healthy as the active/default backend and the behavioral oracle.

2. Keep runtime soundness guarded (Track R): the original thread-safety crash class is fixed, but shutdown leak auditing and interactive redefinition synchronization remain open.

3. Land tiny implementation-independent specs before any backend lowering (Track A), then one canonical symbol/import model for all backends and the REPL (Track B).

4. Make the typed IR the real backend input (Track C): `llvm` consumes IR instead of source lines, and the self-hosted compiler emits the same IR, so one LLVM lowering serves both the C-implemented and Hey-implemented compilers.

5. Climb the LLVM general-lowering ladder (Track D) — data model, text/file, modules, actors, jobs, evented I/O — with no benchmark-specific shortcuts, then decide promotion and no-toolchain distribution from whole-repository evidence (Track E).

6. Keep self-hosted and LLVM in the full backend-world gate; self-hosted currently matches native-C good rows, while LLVM remains a selectable supported subset with explicit unsupported rows.

7. Continue 1BRC, C10K, and ring diagnostics as recurring signals without making 1BRC under 5 seconds a beta blocker.

8. Grow the stdlib toward a Rails-class web/service stack: secure static files, HTTP(S), routing/controllers, ERB-like templates, Redis-like services, SQL adapters for SQLite/MySQL/PostgreSQL, auth/SSO/OAuth/MFA, jobs, cache, mail, observability, package management, and deployment.

9. Prefer Hey-native analysis tools over awk/Perl/Python/Ruby one-offs; missing text/file/report primitives belong on the roadmap.
