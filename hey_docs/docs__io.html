Input and output in Hey

Hey separates byte movement from application protocols. Files, pipes, sockets, and in-memory buffers all carry bytes; stdlib modules add safer, more convenient meanings.

Text and bytes

Strings are text. Bytes are the correct representation for arbitrary file, socket, hash, and protocol payloads. Use string builders for incremental text assembly and byte buffers for binary framing rather than repeated immutable concatenation in hot loops.

Files

Use `stdlib:files` for paths and discovery, and the built-in read/write operations for contents:

import 'stdlib:files'

program
  let path = files.join('build', 'hello.txt')
  write path, "hello\n"
  says read(path)
end

Pipes and child processes

`stdlib:process` accepts argument arrays and returns captured stdout, stderr, and status. This is the normal safe interface for Unix pipes between a Hey program and another process:

import 'stdlib:process'

program
  let result = process.run_argv(['printf', 'hello from a pipe\n'])
  says result['stdout']
end

Use explicit argument arrays instead of shell interpolation. Reach for a shell only when the pipeline itself is the feature being requested.

TCP sockets

`stdlib:net` provides TCP client/listener plumbing. Most applications should use `stdlib:web` or a protocol package such as `hey_web`, which handles HTTP framing and lifecycle details.

import 'stdlib:web'

module GreetingController
  fn show(request)
    return web.text('Hello over a real TCP socket!')
  end
end

program
  let routes = web.routes([web.get('/', GreetingController.show)])
  web.serve(routes, {host: '127.0.0.1', port: 3072, workers: 0})
end

`workers: 0` selects the runtime's online-CPU default. Socket workers should parse requests and produce responses; slow, retryable work belongs in Jobs.

Scheduling and backpressure

A fast socket accept loop is not permission to create unlimited work. Queue bounded or durable work through `stdlib:jobs`, set worker limits and a latency target, and expose queue depth/age metrics. This keeps request threads responsive while the Jobs runtime controls resource use.
