Retirement ladder — endgame breakdown to semantic_debt_files=0

Fable-authored 2026-07-23 from a full read-only inventory of

runtime/heyc.c (20,219 lines) and runtime/hey_repl.c (5,552 lines).

Mandate (todo.md header): retire both files; all LLVM. This doc breaks

todo items 6-8 into gated phases and adds the missing rung the

inventory exposed. Ordered R2->R8; each sub-slice uses the standard

cadence (pin-first, both lanes, survey, regressions, contracts gate,

one commit).

Load-bearing structural facts (from the inventory)

  • heyc.c line 3-4 does `#define main hey_repl_embedded_main` +

`#include "hey_repl.c"`: the tree-walking interpreter engine

(`exec_lines`, `eval_expr_with_env`, ~2.6k lines with its

sub-evaluators) is compiled INTO heyc and powers `heyc --run` AND

the `--test` spec runner. The dependency is one-directional

(hey_repl.c never calls heyc.c).

  • Therefore: THE WHOLE VERIFICATION APPARATUS (specs/language,

specs/repl, examples suites) currently executes on the interpreter

slated for deletion. No pre-existing todo item replaces that lane —

that is phase R5 below, and it gates R6/R7.

  • `bin/hey-repl --llvm-fragment` and `Eval.expression` (opt-in) are

ALREADY Hey-side (tools/hey_run.hey / stdlib/Eval.hey); only

REPL-native eval and `Eval.callable`/`into_session` still enter

hey_repl.c's evaluator.

  • The intended KEEP end-state in heyc.c: object/link mechanics

(`link-ll`, `heyc_build_object_app`, clang path resolution, ~700

lines) + thin CLI porcelain. Everything else deletes.

  • The Stage-0 native-C backend (~1586-3238) and the shared LLVM

emitter (~3239-18122, ~14.9k lines) SHARE low-level string/expr

helpers (e.g. `heyc_emit_multiplicative_c` calls the

`heyc_llvm_find_top_op` family) — they are not cleanly separable

regions; emitter deletion must split the helpers first.

R2 — Narrow the heyc-build surface (IN PROGRESS)

See rung2_narrow_plan.md: (1) HeyHir builtin return-type table +

corpus type-diff gate [in flight]; (2) HeyLlvm multi-arg typed fn

headers [spec 950 fixtures]; (3) Web facade boxed records [946/852];

(4) receipts extension beyond examples/; (5) leads into R3.

R3 — Native LLVM emitter deletion (~14.9k lines of heyc.c)

1. R3.1 Pinned-golden reference migration: the survey and both

regressions currently call live `build-mir --backend llvm` as the

byte-identity reference. Check in (or make regenerable-by-recipe)

per-example golden `.ll` files and repoint every gate at them.

Nothing native may be deleted before this lands.

2. R3.2 Helper split: separate the string/expr-scanning helpers shared

with the Stage-0 C backend (keep them with Stage-0), so the

`heyc_llvm_*` family becomes a deletable island.

3. R3.3 Delete MIR-path emitter branches shape-by-shape (batches with

the R3.1 goldens as the gate), mirroring how coverage was built.

4. R3.4 Delete the source/HIR entry points (`llvm-smoke`,

`build-ir`/`llvm-ir-from-ir`) and the typed-IR loader once nothing

routes there; `llvm-ir-from-mir` deletes last (it IS the reference

generator until R3.1 flips the gates to goldens).

R4 — Stage-0 labeling + callsite guard (todo item 6)

1. R4.1 Rename comparison-only `native-c` labels to explicit `stage0`

across the backend-world TSV producers/consumers

(tools/hey_backend_world_summary.hey, tools/hey_llvm_gap_report.hey,

tools/check_mir_consumers.hey), goldens, and manifest rows.

2. R4.2 New checker (manifest-registered) that fails when ordinary

tooling adds a Stage-0/native callsite outside the allowlisted

seed/recovery/reference set.

R5 — Compile-first verification (NEW RUNG; gates R6 and R7)

The spec runner must stop needing the interpreter before the

interpreter can die.

1. R5.1 Compiled spec harness: a Hey-side runner that compiles each

spec through the ordinary LLVM pipeline and checks `expect`

blocks (stdout/exit) against the compiled binary — the spec format

already separates expectation from program. Prove on a pilot

subset with dual-run (interpreted vs compiled) agreement gates.

2. R5.2 Migrate suites in batches (specs/language, examples,

specs/compiler already run as programs; specs/repl needs the R6

session tool instead). Dual-run until each batch's compiled lane

is the asserted one; interpreted becomes comparison-only.

3. R5.3 Flip `heyc --run` default to compile+run (mandate: all LLVM);

keep an explicit `--interpret` escape hatch only until R6.4, then

delete the lane (`heyc_run_file`, the loader regions it needs, and

the inlined semantic pre-check that guards it).

R6 — REPL retirement (todo items 2/3/4/8 converge here)

1. R6.1 ORC JIT / shared-object fragment loading for the persistent

session (todo item 8), warm latency < 100 ms.

2. R6.2 Port `Eval.callable` / `Eval.into_session` onto the fragment

session path. NOTE: todo item 4's "callable/into_session stay

repl-native" was a waypoint, not the end-state — the mandate

requires this flip; it is the last consumer of the C evaluator.

3. R6.3 Interactive shell ownership: raw-mode input + history move to

the Hey-side session tool (bin/hey-repl-session lineage);

`heyc repl` delegates there.

4. R6.4 Delete runtime/hey_repl.c; heyc.c drops the `#include`

(this also unblocks R5.3's final lane deletion). The C expression

evaluator (~2.6k lines), exec_lines, and history machinery go

with it.

R7 — Bootstrap fixed point, then Stage-0 deletion (todo item 7)

1. R7.1 Publish host bootstrap artifacts (seed binaries) per platform.

2. R7.2 Prove the Stage 1/Stage 2 fixed point (Stage 1 output

rebuilds itself byte-identically) without the native seed.

3. R7.3 Delete the Stage-0 native-C backend (~1586-3238) and the

`stage0-build*` subcommands; the recovery path becomes the

published artifacts.

R8 — Final assembly: semantic_debt_files=0

heyc.c is now: CLI porcelain + build-mode dispatch (execs Hey

tooling) + object/link mechanics. Rename/reshape it as the thin

native launcher+linker it is (no language semantics; the

semantic-debt counter reaches 0), update every doc/ledger row, and

close the mandate.

Dependency graph (why this order)

R2 -> R3 (emitter deletable only after Hey owns every routed shape and

goldens replace the live reference). R4 anytime after R2 (small).

R5 before R6.4 and before R7.3 (gates must not depend on what they

verify the deletion of). R6.1/R6.2 before R6.4. R7 after R5 (the

fixed-point proof runs on compiled verification). R8 last.
