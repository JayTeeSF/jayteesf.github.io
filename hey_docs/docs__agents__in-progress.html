v0.99.365a structured compiler boundaries

Major north-star goals

  • Preserve one fixed-point product pipeline: Hey source -> LLVM-hosted Hey

frontend -> semantic HIR -> ABI MIR -> direct LLVM -> native executable.

  • Assign semantic identity once and carry it forward. LLVM must consume stable

compiler-owned IDs rather than reconstructing scope identity from source-name

strings.

  • Make the cheapest authoritative phase test the default development loop, and

keep broad release acceptance ordered behind focused evidence.

Current milestone

Deliver v0.99.365a as a selective compiler-architecture slice. Function

parameters, lexical locals, and loop bindings receive deterministic `BindingId`

values at the semantic-HIR boundary; ABI MIR preserves them unchanged; direct

LLVM allocates by those IDs. Compiler phases gain explicit replayable artifacts,

and verification gains named preflight/focused/matrix/release/promotion tiers.

This slice deliberately does not rewrite the parser, actor runtime, scheduler,

or language. Actor receive/state identity remains a later extension of the same

artifact contract rather than an unconsumed annotation in this release.

Version-slice progress

  • Semantic HIR now declares `bindings hey-binding-v1` and assigns artifact-order

IDs (`b1`, `b2`, ...) to function parameters, lexical locals, and loop

variables. Source names remain diagnostic labels.

  • ABI MIR preserves each BindingId unchanged. Both self-hosted frontend hosts

produce identical HIR and MIR for the focused shadowing/replay fixture.

  • Direct LLVM maps diagnostic names to BindingIds at lexical scope entry and

allocates `%bN.addr`/`%bN.obj` slots. Legacy fallback slots use opaque

`legacyN` IDs; the emitter no longer manufactures `name__scopeN` identities.

  • `hey phase` exposes deterministic boundaries for AST, HIR, MIR, LLVM IR,

native build, and MIR execution. A stored MIR can be replayed through direct

LLVM after its source file is removed.

  • AST artifacts are canonical JSON carrying producer, language version, source

identity, source hash, and a normalized parser tree without transient cache

fields.

  • `hey verify preflight|focused|matrix|release|promotion` makes gate cost and

authority explicit. Promotion remains last and alone.

  • Focused contracts prove deterministic AST output, both frontend hosts,

HIR/MIR equality, source-independent MIR replay, direct LLVM execution, and

absence of string-derived `__scope` slots.

  • The slice retains the v0.99.364a LLVM correctness, installer, centralized

cache, source-index fast path, and `hey cache gc` work.

  • Warm shebang cache hits retain the minimal native dispatcher: the known

source-index key is validated without starting `hey-full`, then the cached

LLVM application is executed directly. Misses remain owned by the Hey-authored

full launcher.

  • Non-script command porcelain remains in `bin/hey-command`; compiler contracts

validate that `bin/hey` delegates there and that locked package commands are

dispatched by the command router rather than inflating the native fast path.

  • Self-host bootstrap is checkout-local. Disposable candidates cannot inherit

another checkout's stdlib, HIR/MIR producer, or compiler cache through an

installed shell environment.

  • Source ZIP publication includes root `install.sh` and validates the final

recompressed archive before announcing it.

  • Actor parameter BindingIds remain metadata during MIR consumption: the

compiler records them for LLVM slot identity without feeding hidden

`__hir_param` records into the actor `state`/`receive` source validator.

  • Primitive `Env` facades used by the launcher execute in-process rather than

recursively delegating through another compiler process.

  • The native dispatcher treats every existing source file as a program, not

only shebang files, and exports the adjacent SDK root before every full

launcher or command-router fallback.

  • Disposable handoff caches now live under the handoff workspace rather than

`build/`, so `make clean` cannot delete their parents. The native compiler

also recursively creates a configured runtime-object cache path, covering

callers that provide a fresh nested cache directory. Runtime-object identity

now includes `hey_package_project.inc`, preventing package-server changes

from reusing a stale cached runtime.

  • Release A REPL coverage now resolves named Job results through the lexical

`result` binding, and the backend matrix derives the three selectable public

backend names from `heyc --backends` instead of invoking retired aliases.

  • Runtime-object publication is serialized by a stale-owner-aware lock and the

parallel regression proves one compilation, atomic publication, recursive

cache creation, and no temporary/lock residue.

  • Relocated SDK compiler helpers recover the explicit packaged `HEY_ROOT` when

their private cache location is no longer adjacent to the SDK. The Linux

contained-toolchain contract now passes with `PATH` scrubbed.

  • Sanitizer binaries are incremental Make targets with independently selectable

TSAN/ASAN compilers. This agent keeps GCC TSAN and uses Clang ASAN; the complete

sanitizer smoke now passes in nine seconds with warm TSAN output.

  • Every language spec (630) and compiler contract (177) passes individually in

bounded sweeps. Stale prompt/porcelain assertions were updated to the compact

authoritative-index and thin-command-router architecture.

  • The strict-LLVM 67-program closure now uses an invocation-isolated default

output directory, so parallel compiler contracts cannot remove each other's

logs or result files. Package-index and naive-recursion focused contracts no

longer rebuild that complete matrix; they prove their own behavior and retain

inventory membership while Release A runs the broad closure once.

  • The canonical `./bin/test --full` aggregate passes 764 files with zero

failures in 107.479 seconds on the agent. Its slowest file is the repository

script audit at 28.039 seconds and is tracked as a performance target.

Known failures or blockers

  • The HIR/MIR golden suite passes after regeneration. The combined broad

`--mir-consumers` invocation exceeded the constrained handoff execution window

after HIR/MIR completion; it did not report a consumer failure. The maintainer

handoff must run the complete matrix.

  • Actor parameter BindingIds are now retained by the actor-specific MIR/LLVM

path. Actor receive bindings and actor state slots remain later extensions of

`hey-binding-v1` rather than being inferred from source-name strings here.

  • The first complete handoff failed Release A in the named-Jobs REPL transcript

and stale REPL backend matrix; both defects are fixed. The next handoff passed

focused verification and Release C, then exposed a parallel fixed-directory

race between compiler contracts 1082 and 1083. That race is fixed and the

exact concurrent pair passes with two complete 67-entry receipts.

  • The canonical `./bin/test --full` aggregate passes. The four-worker

compiler-contract aggregate exhausted/reset this constrained agent container

before publishing a receipt, although all 177 contracts pass individually and

the three changed contracts pass focused verification. The exact command and

performance target remain in `docs/agents/verification.md`.

  • Release C/A/B and promotion must still be replayed from the final archive.

Promotion remains the final, isolated acceptance gate.

Direction reflection

The evidence supports the selective approach. Stable IDs remove duplicated

scope inference where recent LLVM failures occurred, while deterministic phase

artifacts shorten compiler work from full self-host rebuilds to phase-local

replay. The runtime speed of already-compiled applications is not expected to

change materially in this slice; the immediate performance gain is compiler and

verification throughput. Structured IDs also create the prerequisite for later

liveness, ownership, slot-reuse, and unboxing optimizations without another

string-identity rewrite.
