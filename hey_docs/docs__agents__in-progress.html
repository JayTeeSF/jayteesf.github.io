v0.99.382a — LLVM-first consolidation, session follow-on slices

The v0.99.382a base slice (source-index v2 `request_mode`, Hey-owned

script-startup benchmark, LLVM product compiler for archive/audit builds) is

described in `docs/agents/done.md`. This file tracks the active follow-on

work; completed detail lives in the Git history and `docs/agents/lessons.md`.

Landed this cycle (summaries; see commit messages for full detail)

  • Fixed `Process.run_env_argv`/`exec_env_argv` duplicate-environment-key bug

(`hey_envp_from_object`); overrides now genuinely win over inherited vars.

  • Wired `.reflection.json` sidecars into `stage0-build` (native parity).
  • Normalized 36 stale lowercase stdlib paths in the Git index (fresh

worktrees/clones previously materialized wrong-case files).

  • Release finalizer now clears inherited `HEY_*` vars and isolates all cache

dirs under its disposable worktree.

  • Persistent LLVM REPL session, first sub-slices: `tools/hey_repl_session.hey`

+ `bin/hey-repl-session` prove binding/import persistence, function

redefinition, transactional rollback, and interactive stdin use — every

turn recompiles accumulated session source through the ordinary LLVM

pipeline (specs 1113–1118).

  • Two LLVM backend fixes: explicit `-> boxed` return annotations now beat the

stdlib-imported short-circuit in `heyc_llvm_is_boxed_function_depth`, and

`stdlib/HeySemanticBuiltins.hey` joined the compiler-library body-emission

exemption (`heyc_llvm_direct_emit_compiler_library_record`).

  • **HeySemanticBuiltins is now a compiled-in stdlib module** (the reframed

"cross-unit MIR linking" outcome — no linking exists or is needed; the

pipeline inlines the whole import closure into one unit at parse time):

parser gate exception ahead of `HEY_HIR_APPLICATION_MODE`

(`hey_parser_external_runtime_import?`), HIR facade exemption

(`hey_hir_runtime_facade_form?`), parser-cache schema bump to

`hey-parser-graph-v3` (cache trees are not invalidated by parser-behavior

changes within a version — bump the schema on any such change), and

loop-based scanners in `stdlib/HeySemanticBuiltins.hey` (MIR guaranteed

tail-call elimination supports scalar arguments only). Spec 1119 proves

default-pipeline build, interpreted parity, and genuine HIR inlining.

Cruft removal (user-directed alpha cleanup, in ordered slices)

  • Slice A (done): deleted the dead lowercase stdlib import allowlist

(`hey_parser_stdlib_import_supported?`, 117 lines, dead since the

CamelCase migration) and the consequently-no-op `HEY_HIR_APPLICATION_MODE`

bypass plus its setters; regenerated 4 embedding IR goldens and one

pre-existing stale golden (707). Provably behavior-preserving.

  • Slice B (done): the self-hosted pipeline now independently enforces the

builtin `get` arity contract before HIR emission

(`selfc_pipeline_validate_builtin_arities` in

`tools/hey_selfc_pipeline.hey`), closing the gap where direct

`hey-selfc`/`hey-selfc-pipeline` invocations had zero enforcement. Uses a

new raw-source scanner `HeySemanticBuiltins.validate_source` — a 1:1 Hey

port of the C header's algorithm (string/comment aware, multi-line calls,

declaration-line exemption), byte-matching its diagnostics on all four

maintained fixtures. The duplicated import-closure discovery walk was

extracted into `selfc_pipeline_source_graph_paths` (`-> boxed` annotation

is load-bearing; the classifier fix made cross-function array returns

work) with hash-identical `source-graph-hash` output verified. Spec 1120.

  • Slices C+D (done): `runtime/hey_semantic_builtins.h` deleted (content

inlined into `heyc.c` as the interpreter's single native pre-check, which

retires with `hey_repl.c`; canonical implementation is

`HeySemanticBuiltins.validate_source`). `HeySemantics` is parser-free,

fully type-annotated, and compiled-in (parser cache schema v4); the native

`analyze_text` mirrors, the whole `HeyParserExpanded` family (module, spec

470, example, check tool, native builtins, ledger rows), and the shallow

`HeyParser.parse_text` mirror are deleted -- compiled `parse_text` now

delegates to the real parser. Duplicated-semantics debt is now exactly

{`heyc.c`, `hey_repl.c`} (`semantic_debt_files=2`); one parser remains.

Landed: todo item 4 core — Eval through the fragment compiler

`Eval` is compiled-in (parser cache schema v5). `Eval.expression` in

compiled programs, behind an explicit `HEY_EVAL_SOURCE=1` opt-in, stages

the expression as an ordinary program fragment and compiles+runs it through

the ordinary `heyc build` LLVM pipeline; without the opt-in the exact

fail-closed error is preserved. Interpreted/REPL native eval unchanged

(specs 1094/223). Spec 1121. Facade delegation was deliberately avoided

(the delegation interpreter has native eval and would have broken the

restricted-capability policy). Remaining for item 4: `Eval.callable` /

`Eval.into_session` stay repl-native; the C expression evaluator inside

`hey_repl.c` deletes when `hey_repl.c` itself retires.

Landed: todo item 5 slice 1 — Hey-owned LLVM emission (say-only class)

`stdlib/HeyLlvm.hey` is the canonical Hey-owned textual MIR-to-LLVM emitter

(docs/humans/27 section 6.1), run as `hey-selfc-pipeline emit-llvm

<input.mir> <output.ll>`. Slice-1 class: `unit application` whose entry body

is `block entry`, zero or more `say str "<literal>"` records over printable-

ASCII literals without escapes/interpolation, and `end`. Anything outside

the class fails closed with no partial output, so the Stage-0 native

emitter stays authoritative there. Spec 1122 +

`test/regression_hey_llvm_emitter.sh` prove byte-identity against the

native `.ll` (fixture + the `001_hello` MIR golden), execute the linked

program, and prove the fail-closed rejection. The ~560-line fixed

declaration prelude is embedded as generated single-quoted raw Hey lines;

the version header comes from the MIR `language` record, so nothing is

duplicated. Because the pipeline tool imports it, HeyLlvm is registered as

a compiled-in compiler library (`heyc_llvm_direct_emit_compiler_library_record`

in `runtime/heyc.c`) — without that the selfc-compiled pipeline leaves

`@hey_fn_HeyLlvm__*` dispatch calls undefined and ~70 contracts fail on the

emitter build. Not yet in the product build path: `heyc build` still uses

the native emitter; when HeyLlvm joins the product path it must also join

the selfc pipeline cache fingerprint (`selfc_pipeline_cache_identity` in

`tools/hey_selfc.hey`).

Slice 2 (landed): escape-bearing say literals. The emitter maps the escaped

MIR literal body straight to LLVM constant text plus decoded byte count

(`\n \t \r \" \\ \#` and `\uXXXX` with >7F becoming `?`, control codes as

`\XX` hex), mirroring the native unescape composed with the native constant

escaping — no chr builtin needed (printable-table slice). Raw non-ASCII

bytes and `#{` interpolation stay fail-closed. Regression adds the escape

fixture byte-compare and a bad-escape (`\q`) fail-closed negative.

Slice 3 (landed): literal scalar output. `say i64`/`print i64` over

canonical decimal literals lower to the `@.fmt_i64`/`@.fmt_i64_raw` printf

calls, `print str` to `@.fmt_str`; the walker now accumulates the globals

and main-body text directly, and the receipt class renamed

say-only -> literal-output (spec 1122 updated in the same slice).

Slice 4 (landed): boolean literals and the runtime-required dimension.

`say bool` lowers to the `hey_llvm_says_bool` runtime call and flips

`hey-runtime-required: true` plus the argv/init/shutdown main plumbing —

byte-identical in both header states; `print bool` lowers to the raw i64

printf call, mirroring native.

Slice 5 (landed): i64 locals. `local bN name:i64` lowers to BindingId-named

allocas (`%bN.addr`), say/print over locals to load-then-printf, stores and

initializers to literal or single-binary-op (`add`/`sub`/`mul`) expressions

through the shared `%tN` temp counter. Native ordering mirrored exactly:

literal initializers alloca-then-store; expression initializers evaluate

first, then alloca, then store. Receipt class renamed

literal-output -> straight-line (spec 1122 updated in the same slice).

Slice 6 (landed): interpolated say/print. Each interpolated literal seeds a

fresh empty string constant, then left-folds segments — nonempty text

segments concat interned constants, `#{...}` integer expressions format via

`hey_llvm_format_fixed_string(value, 0)` and concat — ending in

puts/`@.fmt_str` printf on the accumulator; flips runtime-required. Empty

text segments between expressions emit nothing. Unknown locals, richer

expressions, and unterminated markers stay fail-closed.

Slice 7 (landed): the product path routes through HeyLlvm. `hey-selfc`

now tries `emit-llvm` first and hands the Hey-emitted `.ll` to the new

native link-only command `heyc link-ll`; outside the supported class the

emitter fails closed and the Stage-0 `build-mir` consumer remains

authoritative (receipt `selfhost_consumer=hey-llvm-text` vs `direct-mir`;

`HEY_SELFC_MIR_CONSUMER=stage0` forces the native route for bisecting).

`stdlib/HeyLlvm.hey` joined the selfc pipeline cache fingerprint (v6->v7)

and both `hey-selfc` Makefile prerequisite lists. The selfc smoke

regression now proves the hey route on `examples/hello.hey` plus a

product-path golden compare (`cmp` of hey vs forced-stage0 `.ll`).

Slice 8 (landed): control flow. `if bool`/`else`/`end` and `while bool`

over integer comparisons (`slt/sgt/sle/sge/eq/ne`) lower to the native

shared-label shape (`if.thenN/elseN/endN`, `while.condN/bodyN/endN`; an

empty else block always emits), with nesting via a control-frame stack.

Locals declared inside frames stay fail-closed (entry-block alloca

dominance remains Stage-0's job). Receipt class renamed

straight-line -> structured (spec 1122 updated in the same slice).

Slice 9 (landed): user functions. All-i64 `fn` MIR records lower to

`define i64 @hey_fn_<name>` with per-binding param allocas (`%<p>.in`

stored into `%bN.addr`), the module-global `%tN` counter, and functions

emitted before `main` (blank line after each). Calls are whole

expressions (`call name(args)` standalone; source spelling `name(args)`

inside interpolation markers) with operand args loaded left-to-right; the

native `heyc_c_name` mangling is mirrored for defines/calls

(`hey_fn_hey_c_reserved__double`) and unclean param names fail closed.

Function bodies beyond bindings + one integer return stay fail-closed

(proven negative), as do unknown callees and arity mismatches.

Slice 10 (landed): function-body statements. The walker's special

function mode dissolved into the shared dispatch — entering a `fn`

stashes the entry body/locals and restores them at close, so says/print,

locals, stores, if/while, and interpolation inside functions reuse the

exact main-body machinery (module-global string pool, temp and label

counters). Early `return` inside a branch sets a block-terminated flag

that suppresses that branch's closing `br` (native shape), statements

after a `ret` fail closed as unreachable, and a function whose final

block does not terminate fails closed. Comparisons now split on the

depth-0 operator, so whole-expression operands including multi-arg calls

work in conditions. Non-i64 signatures remain fail-closed (negative

updated: says-in-fn is now supported, the negative moved to a Str param).

Slice 11 (landed): frame-scoped locals. Investigation resolved the

hoisting question: the native reused-slot predeclare pass skips any

`let` carrying a BindingId, and MIR-lowered locals always carry one, so

on the MIR path native always emits allocas inline inside branch/loop

blocks — my inline emission was already the exact native shape, and the

fail-closed guard simply dropped. Covers same-named locals in sibling

branches (distinct BindingIds) and per-iteration loop locals in fn

bodies; the old frame-local negative became positive fixture coverage.

Slice 12 (landed): call-operand binary expressions. The integer

expression evaluator gained a depth-0 operator scan with operand-or-call

sides (subsuming the old 3-token split): MIR spells a leading call as

`call name(args)` and embedded operands keep the source spelling, sides

evaluate left-to-right through the shared temp counter — covering

initializers, stores, say payloads, function returns, condition sides,

and interpolation markers. Multi-op expressions and nested call

arguments stay fail-closed.

Class-coverage survey (examples corpus, 100 programs, post-slice-12):

3 wholly in-class and byte-identical, 97 fail-closed, ZERO divergent

(no in-class program emits different bytes than Stage-0). First-blocker

tally: 25x `local :boxed`, 11x `local :str`, ~20x actor records,

~10x worker records, 1x `local :bytes`; then richer int expressions

(parenthesized groups, `%`, nested call args like

`div(sum - div(count, 2), count)`), string concat in say payloads

(`say str "..." + var`), `write` ops, and non-i64 fn signatures

(str/bool/boxed params and returns, blocked behind the str ABI).

Slice 13 (landed): str locals. The locals map became typed

(`bid:type` entries); `local/store :str` lower to `alloca ptr` slots

with interned-literal (alloca-first) or expression (expr-first)

initializers, single `+` concat goes through `hey_llvm_concat_string`

(flips runtime-required), say/print accept string expressions, and a

str-local interpolation marker concats raw with no format call. Survey

after: 5/100 wholly in-class (was 3), still zero divergent — remaining

str programs cascade into str fn params and mixed str+int concat.

Slice 14 (landed, boxed-arc step 1a): mixed/multi-part concatenation.

String expressions left-fold `part + part + ...` (split at depth zero

outside quotes); str parts load, i64-local parts load-then-format

through `hey_llvm_format_fixed_string`, every fold step concats, and

the first part must be string-kind (int-first folds are unpinned and

fail closed). Pinning for step 1b found a native class boundary: the

native emitter itself rejects statements inside string-returning

functions ("non-string body"), so str-fn bodies are bindings+ret-only

in the class and HeyLlvm must fail closed there rather than expand

beyond byte-identity.

Slice 15 (landed, boxed-arc step 1b): typed function signatures. The

fns registry stores `ret_kind|param_kinds`; str params render as

`ptr %name.in` and are used directly in return expressions (`pstr`

locals entries — never reloaded from their allocas, per the native

shape), str returns `define ptr`, bool returns zext their comparison to

i64, call sites type each argument from the registry, bool-returning

calls branch via `icmp ne <result>, 0`, and `say bool <call>` routes

through hey_llvm_says_bool. String-constant threading now flows through

every expression evaluator (call args may intern literals). Native

rejects statements inside str/bool-returning bodies, so those stay

bindings+ret-only with a proven fail-closed negative. The survey's

divergence check caught a real bug during this slice: program-fn calls

are NOT runtime-facade calls and must not flip runtime-required — a

zero-arg literal-str fn keeps `hey-runtime-required: false`. Survey:

8/100 wholly in-class (was 5), zero divergent.

Slice 16 (landed, boxed-arc step 2): boxed literals and says. Boxed

locals use `.obj` slots predeclared (alloca ptr + null store) at the

top of main in record order — including frame-scoped declarations — and

released in the same order at hey.program.done via

`hey_llvm_release_value_ptr` (pre-scan pass mirrors the native

object-slot predeclare). Array/object literals build via

`json_parse_cstr` seeds (a fresh `[]`/`{}` constant per literal, no

dedupe) plus chained `push_i64`/`push_cstr`/`put_key_cstr`/`put_key_i64`;

`say boxed` loads and calls `hey_llvm_says_value`; `len`/`at`/`get`

read through `json_len_i64`/`json_get_index_value`/`json_get_key_value`.

Boxed stores/reassignment, boxed in fn bodies, nested literals, and

non-literal elements stay fail-closed (release-ordering and ownership

pins needed first). Survey stable at 8/100, zero divergent — real

boxed programs need stores, loops over arrays, and boxed params next.

Slice 17 (landed, boxed-arc step 2b): boxed stores. A reassigned boxed

slot owns its pointer: native clones the freshly constructed value

(`hey_llvm_value_clone_ptr`), releases the old one through

`hey_llvm_actor_value_release` (NOT `release_value_ptr`, which is the

program-exit release), then stores the clone — mirrored exactly,

including in loop bodies. Array construction gained i64-local elements

with the native temp quirk (the push-result temp is reserved before the

element load, so ids run load=%tN+1, push=%tN with load emitted first).

Boxed locals in fn bodies stay fail-closed (negative added). Next

rungs: iteration over arrays, boxed params/returns, then actors.

Slice 18 (landed, boxed-arc step 3): array iteration. `control boxed

for bN var in target` loads the target `.obj` and its `json_len_i64`

once, allocates a TEMP-NAMED index slot (`%tN = alloca i64`) plus an

inline loop-var `.obj` slot (nulled, never predeclared, never released

— mirroring native), then loops through `forin.cond/body/end` labels on

the shared counter: cond loads the index and `icmp slt` against the

length, body `json_get_index_value`s into the var slot, end re-uses the

cond-block index temp for the increment. Boxed loop vars (`:bfor`

locals entries) convert via `hey_llvm_value_to_i64` in integer contexts

and say through `says_value`. Frame tokens grew index/cond-temp fields.

Slice 19 (landed, boxed-arc step 4): boxed function signatures. Boxed

params render as `ptr %name.in` and bind to `.obj` slots through a

three-line sequence (alloca, null store, param store) — borrowed from

the caller and never released inside the fn; boxed returns evaluate,

CLONE (`hey_llvm_value_clone_ptr`), then `ret ptr` so the caller owns a

fresh value; call sites pass borrowed loads from `.obj` slots; and any

fn with boxed params or a boxed return keeps a bindings+ret-only body

(fn_restricted flag). Integer contexts reject boxed-returning calls.

Data plumbing of the boxed arc is now complete: literals, access,

stores, iteration, and signatures. Remaining: actors, workers, and the

long tail of builtins/expressions the 92 fail-closed examples need.

Slice 20 (landed, boxed-arc step 5 opening): echo-class actors. A

zero-param zero-state actor emits the paired functions —

`hey_actor_<name>` with the fixed receive loop (name-based `%<recv>` /

`%<recv>.obj` registers, `actor.recv/body/stop` labels, is_stop guard,

per-iteration release) and `hey_actor_step_<name>` with the same body

plus the `actor.step.store.0` epilogue — both bodies flowing the module

temp counter. Entry-side `:actor` locals join the predeclare/release

pre-scan, `spawn` interns the actor-name constant and passes the step

function to `hey_llvm_actor_spawn_pooled_values`, `actor.send` boxes

string literals via `hey_llvm_string_value`, `actor.stop` loads and

stops. examples/actor_echo.hey now compiles wholly through HeyLlvm

(9/100, zero divergent). Actor params/state/join and richer bodies

stay fail-closed with a proven stateful-actor negative — the next

actor slices need a dispatch refactor so bodies can reuse the general

statement machinery twice (actor fn + step fn) instead of the current

echo-class hand emission.

Slice 21 (landed, boxed-arc step 5b): stateful actors. The deep fact:

actor state is BOXED at runtime despite `:i64` MIR typing — the actor

fn initializes it through `hey_llvm_int_value` into a name-based `.obj`

slot before the receive loop and releases it on the stop path;

`store count:i64 = count + <lit>` lowers to boxed arithmetic

(`hey_llvm_add_value`) with the full clone/release store discipline —

NOT scalar add — which is why actor bodies keep their own per-shape

emission instead of reusing the general dispatch. The step fn loads

state from the arg array (`actor_arg_value(arg, 0)`) and writes it back

in the `actor.step.store.0` epilogue (arg_set + release); stateful

spawns build a `[1 x ptr]` initial-state array with the spawn-result

temp reserved before the array temps (the reserve-first quirk again).

Class: exactly one i64 state, statement tokens sayrecv/saystate/inc;

actor params and multi-state stay fail-closed. Survey 9/100, zero

divergent.

Slice 22 (landed, boxed-arc step 5c): actor params. A single boxed

param binds from arg index 0 into its BindingId `.obj` slot; the actor

fn releases the arg array right after binding (`actor_args_release`)

while the step fn does not — the args_release asymmetry. State follows

at index param_count, stop/epilogue paths release params before state,

the step epilogue's `arg_set` targets the state index, and spawns box

literal args (interned BEFORE the actor-name constant) into the

`[N x ptr]` array with the spawn-result temp reserved first. Class: one

boxed param + optional one i64 state; multi-param stays fail-closed

(negative updated). The remaining gap to real actor programs

(actor_counter et al) is message conditionals (`if msg == '...'`) and

`join`/`exit` — the next actor rungs.

Slice 23 (landed, boxed-arc step 5d): actor message conditionals.

`if msg == '<lit>'` inside actor bodies compares through

`value_to_raw_string_cstr` + `strcmp` (an i32 result) + `icmp eq 0`,

with standard if.then/else/end labels drawn from the shared module

counter in both the actor and step functions. The slice also fixed a

latent divergence: the `actor.step.store.N` label consumes a counter

slot allocated at step-emission start (before the step body's own

ifs) — the previous hardcoded `.store.0` would have diverged for any

actor program with later labels. Nesting rides a per-body frame stack;

else-branches in actor conditionals stay fail-closed. Remaining for

real actor corpus coverage: `exit`, `join`, else-dispatch chains.

Slice 24 (landed, boxed-arc step 5e): actor exit/join. `actor.exit`

terminates its branch differently per emitted function —

`hey_actor_exit_self` + release + `ret void` in the persistent actor

fn, but exit_self + release + a branch to the `actor.step.store.N`

label in the step fn so state still writes back — and the following

endif suppresses its leading br after a terminated branch (per-body

terminated flag). `actor.join` loads the slot and calls

`hey_llvm_actor_join_value_ptr`. The actor class now covers echo,
