Byte-keyed integer tables

`stdlib:tables` provides a low-level mutable table for parsers, counters,

indexes, and aggregation loops whose keys are strings or byte ranges and whose

rows contain a fixed number of integers.

import 'stdlib:tables'

let totals = tables.byte_i64(4, 1024)
let row = tables.insert(totals, 'Ada', [10, 10, 10, 1])
tables.set(totals, row, 0, 5)
tables.add(totals, row, 2, 7)
tables.add(totals, row, 3, 1)

For zero-copy parsing, use `find_range` and `insert_range` with immutable Bytes.

The table copies a new key only on insertion. It is mutable and intended to have

one actor/task owner; send or merge it only at deliberate ownership boundaries.

Fused delimited fixed-record aggregation

`tables.stats_scan_delimited_fixed(table, bytes, start, finish, field_byte, record_byte, scale)` scans a bounded byte range whose records have the shape `key<field>fixed-number<record>`. It updates four table columns as min, max, sum, and count and returns the number of complete records consumed.

let rows = tables.stats_scan_delimited_fixed(
  stats, mapped, range_start, range_finish, 59, 10, 1
)

The operation is general parser infrastructure, not a 1BRC-only shortcut. It is appropriate when per-record language/runtime ABI calls would dominate a large immutable byte scan. Each table remains actor-owned; merge worker tables with `tables.stats_merge`.
