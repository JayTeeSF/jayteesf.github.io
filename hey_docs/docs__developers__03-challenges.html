Hey challenge catalog

The challenge catalog follows the executable Tour. It is for practice, comparison,

and experimentation rather than another language tutorial.

./bin/hey challenge list
./bin/hey challenge show run-length
./bin/hey challenge verify run-length
./bin/hey challenge check run-length path/to/my_solution.hey
./bin/hey challenge verify

Each maintained challenge has:

  • an original Hey-owned statement;
  • a starter file;
  • a separate correctness fixture;
  • an enforced wall-clock verification deadline and an advisory memory budget for later sandbox enforcement;
  • an editorial solution that is not treated as the only valid style.

The first catalog contains four exercises:

1. `run-length` — string scanning and immutable output construction;

2. `balanced-delimiters` — stateful scanning and early rejection;

3. `word-frequency` — normalization and dynamic object keys;

4. `ordered-parallel-map` — bounded worker-backed streams with deterministic order.

Challenge source lives under `challenges/<slug>/`. `verify` executes the editorial solution, while `check` stages a learner candidate with the same fixture and compiler used by ordinary applications. The v0.99.276a static browser playground publishes challenge starters beside the Tour without an execution server. Browser catalog entries include starters but not editorial solutions; challenge verification remains grounded in the maintained local fixtures.

RubyQuiz provenance boundary

RubyQuiz inspired the social, project-driven shape of this catalog, but RubyQuiz

statements and solutions are not copied into maintained Hey documentation.

RubyQuiz currently serves its historical index and quiz pages over legacy HTTP. The archiver records that reachable origin without treating transport as a publication license.

The separate archival command is:

./bin/hey rubyquiz archive --out rubyquiz-archive

For a deterministic local or rights-review workflow:

./bin/hey rubyquiz archive \
  --index-file path/to/index.rss \
  --page-dir path/to/pages \
  --out rubyquiz-archive

The command writes raw pages and a canonical provenance manifest only to the chosen

archive directory. Each entry records quiz number, title, source URL, publication

date when present in the index, tags, SHA-256, raw path, and an explicit

`redistribution_status: unreviewed` / `publishable: false` boundary. A human rights

review is required before any copied statement or solution is published.
