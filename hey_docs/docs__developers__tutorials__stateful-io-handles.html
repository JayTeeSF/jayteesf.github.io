Stateful files, mappings, and byte search

Hey represents live operating-system resources with opaque `io_handle` values. Files, process streams, pipes, TCP connections, mappings, and cancellation handles are not JSON descriptors.

Bytes / byte views / byte buffers
              |
              v
       opaque io_handle
              |
   +----------+----------+----------+----------+
   |          |          |          |          |
 files      pipes     processes     TCP      mappings

Construct files with `stdlib:files`; import `stdlib:io` for the common receiver-oriented operation surface.

import 'stdlib:bytes'
import 'stdlib:files'
import 'stdlib:io'

Result-shaped reads

Fallible I/O returns a Result. Successful reads contain explicit byte data, count, and EOF metadata.

let opened = files.open('notes.txt', 'read')

if opened.ok
  let file = opened.value
  let read = file.read(4096)

  if read.ok
    says bytes.decode(read.value.data, 'utf-8')
    says read.value.count
    says read.value.eof
  else
    warn read.error
  end

  file.close()
else
  warn opened.error
end

A successful zero-byte read is distinct from an error.

Stateful sequential files

Supported public modes include `read`, `write`, `append`, `read_write`, and `create_read_write`.

let file = files.open('events.log', 'read').value
let first = file.read(1024)
let second = file.read(1024)
file.close()

Each sequential read advances the file offset.

Exact and bounded reads

let header = file.read_exact(16)
let line = file.read_line(1.mebibyte)
let framed = file.read_until(10, 1.mebibyte) # newline byte

Limits prevent malformed or hostile input from causing unbounded allocation.

Seek, tell, and positioned I/O

file.seek(0, 'start')
says file.tell().value

file.seek(100, 'start')
file.seek(-10, 'current')
file.seek(0, 'end')

let block = file.read_at(4096, 1024)
file.write_at(128, replacement_bytes)

Positioned operations do not change the shared sequential offset. They are appropriate when actors own independent file ranges.

Writing and durability

let file = files.open('output.txt', 'write').value
file.write_all("Hello from Hey\n")
file.flush()
file.sync()
file.truncate(4096)
file.close()

`close` is idempotent. Operations after close return a clean error.

says file.closed?
says file.eof?

Automatic-lifetime read-only mappings

For large immutable files, map bytes instead of decoding the entire source as text.

let mapped = files.map_readonly('../1brc/measurements.txt')

if mapped.ok
  let data = mapped.value
  let separator = bytes.find_byte(data, 59, 0) # ASCII semicolon
  let newline = bytes.find_byte(data, 10, separator + 1)
  let station = bytes.slice(data, 0, separator)
  says bytes.decode(station, 'utf-8')
end

`bytes.find_byte(value, byte, start)` searches from `start` through the end. Use `bytes.find_byte_between(value, byte, start, finish)` for an explicit bounded range.

For line-oriented or delimiter-oriented formats, find a field delimiter and the following record delimiter with one bounded scan:

let record = bytes.find_delimited_record(data, 59, 10, cursor, finish)

if record >= 0
  let field_length = bytes.delimited_field_length(record)
  let record_length = bytes.delimited_record_length(record)
  let separator = cursor + field_length
  let newline = cursor + record_length
end

The packed integer avoids allocating an object for every record. The helper functions lower to integer extraction in direct LLVM. `-1` means that the field delimiter was not found in the bounded range. If the field delimiter exists but the record delimiter does not, `record_length` reaches `finish`.

Mapped slices retain their backing storage and do not copy.

Explicit mapping lifecycle

Use `files.open_mapping` when the owner must close the mapping handle explicitly.

let mapping = files.open_mapping('measurements.txt').value
let shard = mapping.mapping_slice(0, 1.mebibyte).value

says mapping.capabilities().value.kind
mapping.close()

# Existing retained views remain valid.
consume(shard)

# Closed mappings cannot create new views.
says mapping.mapping_slice(0, 1).ok

Physical unmapping occurs after the mapping handle and every retained byte view release their references.

Streams versus mappings

Use stateful streams for bounded-memory sequential processing, interactive protocols, process pipes, sockets, and files that may change while being read.

Use read-only mappings for large immutable files, random access, independent actor-owned byte ranges, direct byte parsing, and workloads such as 1BRC.

A single actor sending one message per input line serializes the workload. Mapped or positioned ranges allow workers to scan independently and return one aggregate per range.

See `stdlib:files`, `stdlib:io`, and process/network handles.
