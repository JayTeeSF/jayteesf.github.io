v0.99.200a substantive handoff: HTTP foundation, value properties, and cross-vendor repair

Completed

  • **Attached verification reviewed:** v0.99.199a focused specs, package tamper protection, self-host LLVM emitter smoke, docs/script checks, LLVM build smoke, IR goldens, source archive, no-toolchain distribution, sanitizer, and the repaired Release-A path all produced positive evidence. The remaining concrete failure was Linux Docker vendoring, whose generated shell body was parsed as Hey source.
  • **Linux Docker vendoring fixed:** `vendor_container_command_text_value` now builds one escaped newline-delimited shell command. `set`, `apt-get`, environment assignment, and `make dist-host` are no longer interpreted as Hey statements. Container build/target verification and host atomic publication remain separate.
  • **Darwin boundaries made explicit:** `xattr` and `codesign` remain allowed platform operations only after target/mode checks. Non-Darwin targets return before either command is resolved or executed. Spec 890 pins the boundary and shell-payload contract.
  • **Generated package state ignored:** `.gitignore` now uses `.hey/` and `**/.hey/`. Manifests and `hey.lock.json` remain tracked; deliberate immutable vendoring belongs under a documented path such as `vendor/packages/`.
  • **Value property sugar:** `value.len`, `.length`, `.size`, and `.count` lower to the existing `len` primitive. Read-only `.keys`, `.values`, `.entries`/`.items`/`.pairs`, `.type`, `.to_json`, and `.json_canonical_text` are also available and may chain, such as `deck.keys.length`. Argument-taking/mutating operations remain calls. Interpreter, native-C, self-hosted-C, typed-IR-C string aliases, and LLVM collection chains are covered without backend-specific benchmark shortcuts.
  • **Core HTTP client foundation:** reusable URL parsing, request serialization, response parsing, bounded text reads, redirect limits, and plain `http://` requests now live in `stdlib:http`, not in the third-party `hey_web` package. `hey_web` remains the owner of higher-level application client/server conveniences.
  • **HTTPS truth preserved:** the client returns `http_tls_unavailable` for `https://`. Socket reads are string-oriented, so arbitrary binary downloads also remain blocked. The 1BRC helper retains one explicit `curl` invocation only for redirected HTTPS binary archives and now reads its capability contract from `stdlib:http`.
  • **Source-overlay contamination prevented:** `tools/hey_source_archive.hey` now packs a canonical allowlist of Hey repository roots. The attached run produced 3,518 files because the language source had been overlaid into `elders_prayer_app`; the same action now excludes foreign `src/`, `vendor/`, app specs, and unrelated root files. The clean current archive is near the expected ~1,170 files.
