Ergonomics to 100% checklist

This checklist turns the raised ergonomics priorities into concrete done gates. It is intentionally stricter than "helper exists": a priority reaches 100% only when it has stable user-facing porcelain, production-shaped Result/error boundaries, docs/tutorial examples, native-C and self-hosted-C coverage, and no known ad-hoc replacement still needed in examples/tools.

1. CLI scripting

Done enough for 100% when: runtime argv, explicit args, typed options, schema/help/validation, `--`, `--no-*`, negative numeric values, positionals, command selection, Result-form validation, and usage errors are all documented and used by at least one tool/example.

Open: env/file/path/glob/archive integration and cleanup of existing scripts/tools to prefer schema-backed `cli.*` over ad-hoc parsing. Exit/status shaping landed in v0.99.106a.

2. Error handling/recovery

Done enough for 100% when: fail-fast remains the bug path, Result values cover expected failures across files/process/text/http/net/package boundaries, validation aggregation is standard, and source-spanned errors can be attached without exceptions.

Complete for current gate: ordinary runtime failures now keep source-context stacks for interpreter/native-C paths, and native-C function calls preserve caller frames. A later Ruby-like `rescue` remains optional boundary syntax only if Result + supervisors are not enough.

3. Collections

Done enough for 100% when: common list operations, record helpers, grouping/counting, sorting/chunking/zipping, and nil handling are documented and used. Higher-order map/filter/reduce wait on function values and should not block the current ergonomic gate.

Open: higher-order helpers after function-values settle and example cleanup.

4. Text / regex / formatting

Done enough for 100% when: text casing/splitting/line/word/replace/padding/truncation, regex-lite match/find/span/scan/replace/split/grep/reject, classes/ranges/negation/alternation, and the explicit regex-lite compromise are documented.

Open: capture groups/capture-aware gsub. These are useful but not required unless JayTeeSF promotes them before LLVM resumes.

5. Package/library workflow

Done enough for 100% when: manifest/source/dependency/registry/lock/resolve/validate/import-path/checksum/trust semantics are pinned and a package layout is documented.

v0.99.107a landed real local cache/install execution with module copies and metadata/lock writes. v0.99.108a added local source digests, trust-enforced install, list/remove commands, file/remote source descriptors, and import aliases. v0.99.109a replaces the digest with full SHA-256, adds file and remote HTTP fetch with checksum enforcement, and links the tutorial to a full third-party package authoring guide. Complete for current gate: SHA-256 install/fetch, semver parse/range/caret matching, best-match resolve, generated package markdown, cache install/list/remove, and the third-party package authoring guide are specced. Registry signing/federation is a later distribution layer.

6. Debuggability

Done enough for 100% when: inspect/labels/snapshots/events/log lines/timers/result summaries/assertions are documented, and runtime/compiler failures can include source spans and eventual stack traces.

v0.99.107a landed source-spanned runtime crash plumbing through diagnostics. v0.99.110a adds automatic source-context breadcrumbs for ordinary runtime failures in the interpreter and native-C compiled output. Complete for current gate: source-spanned diagnostics plus ordinary multi-frame function/caller runtime errors are specced. Actor trace enrichment, richer pretty inspect, and debugger hooks are future polish.

7. Sockets/TLS/web runtime

Done enough for 100% when: descriptors exist for listeners/clients/TLS/routes/services, and real runtime plumbing exists for accept/connect/read/write, timeouts/cancellation, supervised listener workers, and clear TLS/reverse-proxy stance.

v0.99.107a landed real loopback TCP listen/connect/accept/read/write/close. v0.99.109a added host connect plus accept/read fd timeouts. v0.99.110a adds streaming body descriptors and supervised listener/cancellation/TLS-stance contracts. Complete for current gate: loopback/host TCP, accept/read timeouts, supervised listener plans, reverse-proxy TLS stance, and real chunked fd stream writes with poll-backed backpressure are specced. Native TLS, full production listener workers, and Windows socket parity are future web-stack layers.

8. Track B canonical symbols/imports

Done enough for 100% when: native-C, self-hosted-C, LLVM, and REPL registrations consume the canonical manifest/table instead of parallel ad-hoc name scans.

Progress v0.99.112a: compiler-owned manifest rows now cover imports/modules/functions/actors; `heyc symbols`, typed-IR symbol output, and LLVM known-function/facade classification consume the manifest. v0.99.113a first cleans bad backend-world rows from that slice. Open: move remaining native-C/self-hosted-C/REPL registration paths fully onto the manifest.

v0.99.107a current hard-gap status

The hard gaps are no longer purely plans: package install/cache/list/remove, SHA-256 digest/trust enforcement, file/remote fetch, source-spanned crash failures, ordinary runtime-error source breadcrumbs, OS socket round-trips, host connect, fd timeouts, streaming body descriptors, and supervised listener/cancellation/TLS-stance contracts have specs. The raised ergonomics hard-gap gate is complete for the current alpha scope as of v0.99.111a. v0.99.112a resumed Track B canonical-symbol/import consumers, and v0.99.113a cleans the resulting backend-world bad-row cluster before broad LLVM unsupported-row reduction continues.
