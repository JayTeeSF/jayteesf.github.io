Language influences

Ruby

Keep the joy: low ceremony, great strings, good names, and expressive code.

Native systems languages

Produce compact native executables through semantic HIR, ABI MIR, and LLVM. Keep manual memory, platform ABI, and toolchain details behind ordinary Hey values and runtime services.

JavaScript object practice

Hey object notation borrows the practical literal shape common in JavaScript style guides: use literal objects, prefer readable properties, and quote only keys that are not identifiers. Hey does **not** copy JavaScript's prototype model.

let user = {name:'Ada', visits:2, 'data-id':123}

Erlang/OTP

Isolated actors, mailboxes, fail-fast thinking, links, monitors, and supervisors are core influences. Actors remain the right abstraction for identity and persistent private state.

Pony

Data-race avoidance matters. Hey avoids shared mutable state across actors and workers by default.

Go

Hey borrows operational practicality: native binaries, useful tooling, runtime diagnostics, bounded OS-thread pools, worker-local scheduling, and managed parallel execution. Hey does not copy channels into the public language; stateless workers, Jobs, and streams provide the ordinary parallel-work surface.

Lisp

AST clarity and code transformation matter, but macros wait until the language core and compiler contracts are stable.

Concurrency direction

Hey presents one model:

immutable values
  + isolated stateful actors
  + replicated stateless workers
  + runtime-managed Jobs
  + worker-backed functional streams

Actors use `spawn`, messages, mailboxes, `receive`, `join`, `stop`, links, monitors, and supervisors when work requires identity or persistent state. Workers drain bounded shared work mailboxes when work is stateless. Jobs own operational policy. Streams provide familiar pipelines without exposing general-purpose channels.

Source `await` suspends an actor around native readiness; it does not create a separate task model. Internally, the runtime may use worker threads, queues, work stealing, readiness reactors, and continuations while preserving actor isolation, worker statelessness, mailbox order, and backpressure.
