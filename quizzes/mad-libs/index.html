Mad Libs

Build a tiny template engine for Mad Libs. Locate labeled placeholders, request one value per unique label, and render the completed story while reusing repeated answers.

templatesparsingI/O

Your task

  • Recognize placeholders without evaluating code.
  • Prompt once for each unique label.
  • Reuse the same answer when a label repeats.
  • Preserve all non-placeholder text exactly.

Example shape

template: The (adjective) actor greeted the (noun).
answers: adjective=curious, noun=compiler

Approach ideas

These are strategy summaries, not copied Ruby solutions.

  • Scan delimiters directly with string indices.
  • Tokenize the template into literal and placeholder records.
  • Separate parsing from answer collection so browser and CLI interfaces share the renderer.
