Processes, pipes, TCP, UDP, readiness, and cancellation

Files, child-process streams, in-process pipes, and TCP connections share the same opaque stream-handle operations from `stdlib:io`. UDP remains datagram-oriented.

                     common stream operations
                               |
          +--------------------+--------------------+
          |                    |                    |
       file handle         process pipe         TCP connection
          |
     positioned I/O

UDP sockets preserve datagram boundaries.

Live child processes

import 'stdlib:bytes'
import 'stdlib:io'
import 'stdlib:process'

let spawned = process.spawn(['/bin/cat'], {})

if spawned.ok
  let child = spawned.value.child
  let input = spawned.value.stdin
  let output = spawned.value.stdout
  let errors = spawned.value.stderr

  input.write_all("Hello from Hey\n")
  input.close()

  let response = output.read_exact(15)
  if response.ok
    says bytes.decode(response.value.data, 'utf-8')
  end

  let status = process.wait(child, 5000)
  says status.ok

  output.close()
  errors.close()
end

`process.spawn` does not invoke a shell. It returns immediately with a child handle and live stdin/stdout/stderr handles.

Process controls are:

process.poll(child)
process.wait(child, 5000)
process.terminate(child)
process.kill(child)
process.pid(child)

Bounded capture

Use `process.run_argv_timeout` when a convenience capture-and-wait call needs a hard deadline.

let result = process.run_argv_timeout([
  '/usr/bin/printf', 'hello'
], 5000)

Timed and untimed argv execution drain stdout and stderr concurrently. This avoids deadlock when either pipe fills. The timeout owns the child process group and escalates termination when required.

Independent bounded pipes

import 'stdlib:pipes'

let pipe = pipes.create(1.mebibyte).value
let reader = pipe.reader
let writer = pipe.writer

writer.write_all('payload')
writer.close()

let received = reader.read_exact(7)
if received.ok
  says bytes.decode(received.value.data, 'utf-8')
end

reader.close()

Closing every writer delivers EOF to readers. Bounded buffering provides backpressure.

TCP listener and client

import 'stdlib:net'

actor echo_server(listener)
  receive start
    let accepted = net.accept(listener, 5000)
    let connection = accepted.value.connection
    let request = connection.read_exact(4)
    connection.write_all(request.value.data)
    connection.close()
    exit
end

program
  let listening = net.listen('127.0.0.1', 0, {})
  let listener = listening.value.listener
  let port = listening.value.local_address.port

  let server = spawn echo_server(listener)
  send server, 'start'

  let connection = net.connect('127.0.0.1', port, {}).value
  net.set_nodelay(connection, true)
  connection.write_all('ping')

  let response = connection.read_exact(4)
  says bytes.decode(response.value.data, 'utf-8')

  says net.address(connection, 'local').value.port
  says net.address(connection, 'peer').value.port

  net.shutdown(connection, 'both')
  connection.close()
  join server
  listener.close()
end

A listen port of `0` asks the operating system to select an available port.

UDP datagrams

let bound = net.bind_udp('127.0.0.1', 0)
let socket = bound.value.socket
let port = bound.value.local_address.port

net.send_to(socket, 'hello', '127.0.0.1', port)
let packet = net.recv_from(socket, 65535, 5000)

if packet.ok
  says bytes.decode(packet.value.data, 'utf-8')
  says packet.value.address.host
  says packet.value.address.port
end

socket.close()

UDP preserves message boundaries and therefore does not pretend to be a sequential stream.

Blocking readiness selection

let selected = io.select([listener, connection], 5000)

if selected.ok
  says selected.value.count
  says selected.value.timed_out

  for ready in selected.value.ready
    says ready.index
    says ready.readable
    says ready.writable
    says ready.error
  end
end

Each ready record includes the input index, retained handle, and readable/writable/error flags.

Cancellation wake-up handles

let token = io.cancel_token().value
io.cancel(token)

let selected = io.select([token], 5000)
says selected.value.count
says selected.value.timed_out

token.close()

A token can wake a thread blocked in `io.select` without continuously polling application state.

Scheduler-safe asynchronous readiness

`io.select_async` registers one or more handles with a single multiplexing runtime reactor and returns immediately:

actor readiness_sink
  receive message
  if message.result.ok
    says message.tag
    says message.result.value['count']
  else
    says message.result.code
  end
  exit
end

program
  let sink = spawn readiness_sink
  let request = io.select_async([connection], 5000, sink, 'connection-ready')

  # Later, when readiness, timeout, or cancellation wins, sink receives:
  # {kind: 'io_completion', request_id: ..., tag: ..., result: Result}

  # Cancel an outstanding request explicitly:
  io.cancel(request.value.cancel_token)
end

The runtime duplicates registered descriptors so closing the original handle cannot cause descriptor-reuse corruption inside the wait. Deadlines and cancellation are resolved by the reactor, and completion is delivered as an actor message rather than blocking an actor scheduler worker.

The implemented architecture is:

Hey actor
    |
    | await handles, timeout_ms, tag
    v
single runtime reactor registration
    |
    +---- Darwin/BSD: kqueue
    +---- Linux: epoll
    +---- fallback: poll
    |
I/O becomes ready / times out / is cancelled
    |
    v
deliver tagged io_completion Result to the actor mailbox

Actor-native source `await`

Inside an actor, `await` registers a readiness operation and ends the current mailbox turn without blocking a scheduler worker:

actor reader(connection)
  receive message
    if message.kind == 'begin'
      await [connection], 5000, 'request-ready'
    else
      if message.kind == 'io_completion' and message.tag == 'request-ready'
        let request = connection.read(4096)
        # Process the ready stream, then await again or exit.
        exit
      end
    end
  end
end

The syntax is:

await <handles>, <timeout_ms>, <tag>

An actor containing `await` is automatically placed on the native lightweight worker pool. The runtime later sends an `io_completion` message containing `kind`, `request_id`, `tag`, and `result`. This is actor/mailbox re-entry, not a hidden lexical continuation: locals from the completed mailbox turn do not survive unless stored in actor state or included in the tag/message.

The product compiler lowers the same source and completion semantics to pooled direct-LLVM actor steps. Each completion re-enters through the actor mailbox.

`io.select` remains the simple blocking API. `io.select_async` remains the explicit registration API used when another actor should receive completion. TLS readiness also recognizes plaintext already buffered inside OpenSSL.

See `stdlib:io`, `stdlib:tls`, `stdlib:process`, and stateful files and mappings.
