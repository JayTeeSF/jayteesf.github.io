Hey reserved words and target-safe names

Hey source has its own syntax words. Names reserved by LLVM, a platform linker, C headers, or another implementation detail are **not** automatically reserved by Hey; the compiler gives generated symbols private safe names.

Hey structural words

These words introduce language structure in their corresponding positions:

program  module  import  private  fn  actor  worker  job  spec  end
let  set  state  if  else  while  until  for  in  forever  return
spawn  supervise  send  receive  join  stop  sleep  await  exit
says  print  warn  write  log  fail

Spec clauses also use:

expect stdout  expect stderr  expect error  expect exit

Do not use structural words as application bindings even where the alpha parser can infer an unambiguous position.

Expression-reserved words

true  false  nil  and  or

These have fixed expression meanings and cannot be ordinary variable, function, actor, or module names.

Target/toolchain words remain legal Hey names

A target implementation may reserve words such as `double`, `switch`, or `static`. They remain legal Hey names:

fn combine(double, switch)
  let static = double + switch
  return static
end

program
  says combine(20, 22)
end

The compiler preserves the Hey-level names and privately canonicalizes generated symbols. Application code never refers to the generated spelling.

Built-ins and standard-library names

Built-ins and imported modules are APIs, not another keyword list. Avoid shadowing familiar names such as `json`, `len`, `type`, `web`, or `jobs` because it harms readability, not because the compiler forbids it.

Object field names

Object keys are data and may use syntax words, built-ins, or target-toolchain words. Prefer dot access for ordinary identifier-like fields and brackets for dynamic or colliding keys:

let response = {'display-name':'Ada', length:42}
says response['display-name']
says response['length']

See Object field access.
