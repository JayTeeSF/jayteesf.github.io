Files

`stdlib:files` is the only public filesystem module. The earlier `stdlib:fs` split was removed during alpha development; use this module for contents, paths, traversal, and no-follow metadata.

`stdlib:files` is the single public filesystem module. It owns file contents, path helpers, directory traversal, and no-follow metadata; applications do not need to choose between separate `fs` and `files` namespaces.

import 'stdlib:files'

let path = files.join('build', 'message.txt')
files.write_text(path, "hello\n")
says files.read_text(path)
says files.basename(path)
says files.dirname(path)

Discovery and traversal

let markdown = files.files('docs', '.md')
let entries = files.entries('docs')
let sources = files.hey_files('.')

`files.files` uses the runtime recursive walker and returns a sorted array. `files.files_slow` is the readable Hey implementation used for parity and debugging.

No-follow metadata

Package installers and other security-sensitive tools must inspect a path without following a symbolic link:

let info = files.metadata('build/message.txt')

says info['exists']
says info['type']
says info['size']
says info['mode']
says info['executable']
says info['symlink']

The metadata record reports `file`, `directory`, `symlink`, `other`, or `missing`. The same API works through the interpreter, native-C, and LLVM.

Result-returning reads

Use `files.read_result(path)` or `files.lines_result(path)` at a boundary where a missing file should be represented as data instead of terminating the program.

Staging and durable publication primitives

`stdlib:files` now owns the reusable filesystem operations used by vendoring and package publication:

let stage = files.temp_dir('build', 'package-stage')
files.copy_tree('source', stage['value'])
files.make_executable(stage['value'] + '/bin/tool')
files.write_atomic('build/state.json', json.encode_canonical_line(state))
files.move(stage['value'], 'build/published')
files.remove_tree('build/published.old')

`files.write_atomic` writes and syncs a sibling temporary file, renames it into place, and syncs the parent directory. Tree copy rejects symlinks, staging stays on the destination filesystem, and executable changes inspect the file without following links. Policy such as backup naming and rollback order remains in the calling Hey module.
