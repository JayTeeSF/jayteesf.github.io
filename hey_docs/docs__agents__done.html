v0.99.249a

  • Added mutually exclusive fixed `count:` and adaptive `min_workers`/`max_workers`/`target_queue_latency_ms` Job pool configuration.
  • Added one shared adaptive-pool controller that scales lightweight pooled workers from oldest queue latency and backlog pressure, rate-limits lifecycle changes, and gradually returns idle pools to their minimum.
  • Added current, desired, minimum, maximum, and peak worker metrics plus oldest queue age, scale events, starts, retirements, policy intervals, and last decision reason.
  • Extended the deterministic Jobs clock to retry and scaling deadlines.
  • Preserved fixed-size Job behavior and rejected partial or mixed adaptive configuration through the REPL and both compiler lowering paths.
  • Added deterministic language and REPL coverage, a runnable adaptive Jobs example, repeated stress execution, and byte-equivalent two-host MIR/direct-LLVM product evidence.
  • Updated Jobs, concurrency, tutorial, generated API, roadmap, tracking, verification, and continuation-prompt documentation.
  • Verified 602 observable language/REPL/example files with zero failures in segmented execution, 9 HIR/MIR fixtures, 20/20 conformance rows, documentation smoke, and a 1,254-file source archive preserving 113 executable paths.
  • Recorded that Release-C remains blocked by the pre-existing `test/regression_mir_boxed_branch_dominance.sh` failure after its first two artifact-safety checks; the adaptive feature regressions themselves pass.
  • Recorded rather than concealed the still-failing real Linux Docker vendoring command and the independent v0.99.248a Release-A MIR dominance failure.

v0.99.248a

  • Repaired Linux vendoring against the full canonical `dist-host` graph: release tools now receive the exact release-local bootstrap compiler instead of falling back to source-checkout `build/heyc`.
  • Made LLVM stdlib delegation discover the installed SDK root/bootstrap, use temporary files outside the checkout, and pass `HEY_ROOT`, `HEY_STDLIB_PATH`, and `HEY_STDLIB_HEYC` to delegated compiler runs.
  • Made the compiled `hey` launcher propagate its resolved SDK root into cached applications, so relocated applications can use the copied stdlib from any working directory.
  • Lowered boxed `version.current`, tag, next-version, and C-define values through compiler-owned constants instead of runtime delegation.
  • Expanded Linux release regression coverage from merely creating `hey-selfc` to building the full SDK, relocating it, emitting HIR, and running target JSON/version verification with discovery overrides unset.
  • Restored the canonical roadmap ordering: 249 adaptive scaling, 250 parallel streams, 251 resequencing, 252 keyed partitioned Jobs, 253 thin Job syntax, 254-256 scheduler performance, 257 reactor closure, 258-259 Unix SDKs, and 260-265 Windows x64.
  • Removed the vendor-only `HEY_LLVM_POINTER_MODE=legacy` override that broke Linux-x64 self-hosted release compilation under current LLVM toolchains.
  • Added a focused contract plus a native Linux release-self-compiler regression that emits semantic HIR before distribution publication.
  • Closed the complete 598-file observable suite after adding the boxed-version regression and refreshing documentation-catalog expectations for the expanded Jobs API.
  • Added optional append-only canonical-JSON Job journals with synchronized enqueue and terminal records.
  • Added restart replay of accepted work without a terminal record, restored completed results/idempotency keys, and explicit at-least-once delivery semantics.
  • Added bounded atomic journal compaction and status metrics for durability, journal events/errors, and replayed work.
  • Added deterministic Jobs retry clocks so backoff tests advance time without wall-clock sleeps.
  • Added two-host durable Jobs MIR/product evidence plus language and runnable-example coverage.
  • Added a complete Workers and Jobs tutorial explaining every declaration, value, receipt, result, option, metric, and durability tradeoff.

v0.99.245a

  • Added full REPL parity for stateless workers and bounded worker pools, plus the optional `std:workers` helper module.

v0.99.244a

  • Established the public concurrency model: immutable values, stateful isolated actors, stateless workers, runtime-managed Jobs, and worker-backed streams; general-purpose channels remain runtime-internal only.
  • Added `worker Name(message)` declarations with exactly one immutable message parameter.
  • Added fixed-size `workers(...)` pools over the bounded managed scheduler, one bounded shared work mailbox, explicit enqueue backpressure, status metrics, and pool shutdown.
  • Preserved workers through semantic HIR and ABI MIR and lowered them through the direct LLVM consumer.
  • Rejected worker bodies that declare actor state, receive from their own mailbox, or use mutable `set` bindings.
  • Fixed an LLVM MIR-consumer recursion caused by canonical module calls such as `workers.enqueue(...)` being reinterpreted as receiver sugar.
  • Added queue-wait average/maximum metrics as the foundation for later latency-based Job scaling.
  • Rebased the roadmap so Jobs, reliability, adaptive scaling, parallel streams, resequencing, and partitioning precede remaining SDK/Windows work.
  • Preserved ordinary object fields named `status`, `stop`, and `enqueue`; only the parenthesized receiver forms are worker-pool methods. Added an outside-in compiled regression after the full suite exposed the collision in the Jobs example.
  • Updated current architecture, I/O, compiler, reserved-word, and source-prompt documentation contracts to the actor/worker/Job/stream model.

v0.99.243a

  • Fixed invalid LLVM dominance for actor and other pointer-backed MIR locals created inside loops by allocating their object slots in the function entry block.
  • Added an executable actor-fan-out dominance regression that compiles and runs through the default self-hosted HIR/MIR/LLVM pipeline.
  • Replaced persistent source-archive removal lists with per-download comparison of Git-tracked paths against actual archive entries.
  • Added opt-in `HEY_SCHEDULER_STATS` diagnostics for queue traffic, processed turns, steals, sleep events, and wake signals while keeping detailed counter maintenance off the ordinary path.
  • Profiled five repeated 20,000-actor/200-round fan-out runs: medians were 240 ms at one worker, 253 ms at two, 478 ms at four, 854 ms at eight, and 1,374 ms at sixteen. The roadmap now separates allocation/registry locality, mailbox/value fast paths, and scheduler-generation-2 experiments.
  • Removed an unsuccessful bounded-spin experiment after it increased steal storms and multi-worker elapsed time.

v0.99.240a

  • Made every generated source-zip handoff require explicit rm/unzip/status/add/commit/verification commands, and made `hey-zip-source` print install/apply commands with each archive.
  • Restored strict-C11 Darwin runtime compilation by exposing Darwin SDK extensions before POSIX feature selection and keeping current-RSS measurement on the Mach `task_info` API instead of compiling the incompatible generic `rusage` fallback. Added a focused portability contract.
  • Closed the shipped self-hosted frontend fixed point: one private native seed rebuilds final LLVM-hosted `hey-selfc`, which rebuilds `hey` and package tooling through semantic HIR and ABI MIR; the native seed still owns the final validated MIR-to-LLVM/object/link consumer.
  • Removed shell-wrapper recursion from LLVM stdlib delegation; a clean north-star bootstrap completed in 23.13 seconds and the release compiler/tool build in 27.35 seconds in the current Linux environment.
  • Fixed boxed `files.read_text` results passed to generic stdlib calls, restoring LLVM-hosted HIR/MIR sidecar publication and the maintained MIR-consumer matrix.
  • Moved ordinary statically selected direct-LLVM actors from one operating-system thread per actor onto the shared mailbox-step worker pool.
  • Fixed pooled state/join ownership so cleanup cannot race the final actor step.
  • Added adaptive ready-queue batching and bulk dormant-actor shutdown; one million resident compiled actors spawned in 672 ms at about 384 observed bytes each, reached about 458 MB RSS, and completed process shutdown within 0.89 seconds in the current Linux environment.
  • Added cgroup-aware total/available/RSS memory intrinsics and memory-capped actor-capacity plus multicore fan-out benchmarks written in Hey.
  • Fixed self-hosted boxed object stores inside loops, with a compiled product contract.
  • Added complete first-class function/module-callable/array-to-actor documentation and refreshed actor, I/O, and distribution guides.
  • Replaced Bootstrap-dependent documentation layout with semantic responsive markup, local reference/plain themes, optional Pico/Water.css CDN themes, and sidebar/topbar layouts.
  • Removed historical compiler-backend documents and the misleading Windows container placeholder.
  • Made distribution receipts state the truth: compatible precompiled binaries are run-ready; compiler SDK use still requires an external target toolchain; Windows is not implemented.
  • Rechecked deterministic 5-million-row 1BRC performance: equivalent C best 253 ms, Hey single best 405 ms (`1.601x`), and the actor sweep best 153 ms at six workers. No challenge-specific compiler/runtime shortcut was added.
