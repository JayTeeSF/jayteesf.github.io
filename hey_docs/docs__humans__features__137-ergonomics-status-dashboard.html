The remaining items are now future feature tracks rather than blockers to returning to backend parity: native TLS, full production listener workers, registry signing/federation, capture-aware regex, higher-order collection helpers, debugger hooks, and platform portability.

2. decide whether regex-lite is sufficient for now or whether capture groups must land before LLVM resumes.

The practical return-to-LLVM threshold is now met. The active top-priority section is Track B canonical-symbol/import consumption plus weighted LLVM unsupported-row reduction for the expanded ergonomic surface. v0.99.112a moved compiler symbol output, typed-IR symbol rows, and LLVM known-function/facade classification onto the canonical manifest; v0.99.113a cleans the v0.99.112a backend-world bad-row cluster before returning to unsupported-row reduction.

v0.99.112a Track B/backend parity restart

  • Compiler-owned canonical symbol manifest now records imports, modules, functions, and actors with normalized paths and source metadata.
  • `heyc symbols` prints the full manifest; typed IR keeps stable canonical fn/actor rows.
  • LLVM uses manifest-backed known-function/stdlib-facade classification and now lowers `crypto.sha256_text(...)` through a runtime SHA-256 facade.

v0.99.113a bad-row cleanup

  • Imported `debug.fail_at(...)` now routes through `hey_debug_fail_at_builtin(...)` in native-C and self-hosted-C builds.
  • Self-hosted-C now handles nested multiline module-call array arguments without confusing function-call array literals for top-level index access.
  • LLVM spec 775 no longer build-fails from leaked function-local object/string/bool locals.
  • Backend-world problem rows now suppress expected-error run failures that already match native-C, so real build/mismatch rows stay visible.

Plumbing vs porcelain boundary

  • Current hard-gap plumbing is in place for this alpha gate: runtime argv, safe argv process execution, SHA-256 package digest/fetch/install/list/remove, semver/docs generation, loopback socket round-trips, host connect, fd read/accept/write-stream timeouts, chunked fd streaming with poll-backed backpressure, ordinary source-context stacks, and service/cancellation/TLS-stance contracts. Future web/debug tracks include native TLS, full production listener workers, actor trace enrichment, rescue syntax if Result remains insufficient, and compiler diagnostics that consume richer import/module spans.
  • Porcelain already started: `cli`, `result`, `collections`, `text`, `re`, `packages`, and `debug`.

With the raised ergonomics hard-gap gate complete, resume backend parity work. Keep adding porcelain opportunistically, but do not let convenience helpers displace the Track B/LLVM priority unless a real blocker appears.

v0.99.107a hard-gap progress

  • Package workflow moved from descriptor-only to real local cache/install execution: `packages.install_local` creates the cache path, copies modules, and writes package metadata/lock metadata.
  • Debug/error workflow now has source-spanned crash plumbing: `errors.raise(errors.diagnostic(...))` fails through the runtime with source file/line/column text.
  • Net workflow now has real OS loopback TCP plumbing for `listen_loopback`, `connect_loopback`, `accept_one`, `write_all`, `read_some`, and `close_fd`. This is intentionally small but it proves accept/connect/read/write are not only descriptors.
  • Porcelain teaching started with `examples/ergonomics_porcelain_app.hey` and tutorial updates; keep cleaning older examples/tools as nearby code is touched.

v0.99.109a package and net workflow progress

  • Package workflow now uses real SHA-256: `crypto.sha256_text`, `crypto.sha256_file`, and `packages.source_digest` all produce 64-character SHA-256 values; `packages.trust_source` and `packages.install_verified_local` reject mismatches before copying modules.
  • Fetch is executable: `packages.fetch_file` copies a local artifact and verifies SHA-256; `packages.fetch_remote` performs an HTTP GET through runtime sockets and verifies the downloaded body; `packages.fetch` dispatches by source mode.
  • Cache management is executable: `packages.list_installed` reports installed package ids and `packages.remove` deletes a package cache entry.
  • Networking now exposes `net.connect_host`, `net.accept_one_timeout`, and `net.read_some_timeout`, pinning host TCP and timeout boundaries before supervised services land.

v0.99.111a hard-gap completion and next-priority start

  • Runtime errors now use a stack, not a single overwritten frame; native-C ordinary failures can show both function and program caller frames.
  • HTTP streaming now has deterministic chunked-transfer rendering and real fd execution through `net.write_stream(...)`, which writes chunked frames with poll-backed writable waits and backpressure metadata.
  • Package workflow now includes semver parsing, caret/range requirements, best matching resolution, and generated markdown docs.
  • LLVM/Track-B work has resumed with `http.*` facade classification, so imported HTTP internals are no longer the first false wall for LLVM diagnostics.

v0.99.110a debug and web runtime progress

  • Ordinary runtime errors now receive automatic source-context breadcrumbs in interpreter and native-C compiled output. This is a single-frame statement breadcrumb; multi-frame function/actor traces remain open.
  • HTTP now models streaming bodies through chunk and stream descriptors plus deterministic preview helpers. Real socket backpressure/chunked transfer is still a runtime worker task.
  • Net now models supervised listener workers, cancellation tokens, timeout policies, and the current TLS stance: terminate at a reverse proxy until native TLS lands.
