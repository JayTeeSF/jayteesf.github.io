Hey quick start

Hey is in alpha at **v0.99.74a**. The active compiler path is native `heyc` with the `native-c` backend. LLVM is selectable for a checked scalar/string/control-flow/stdout, string file-I/O, string slice/compare, literal for-loop, string-returning functions including helper calls and split-index reads, general boxed data-model subset, process/file/text facades, and the v0.99.48a+ jobs/actor retry-capable result-waiter subset. v0.99.55a started the real-app web/service safety ladder with `stdlib/static_files.hey` and documents the current promotion baseline: native-C and self-hosted match at good=354/362, while LLVM is clean but incomplete at good=196/362. Native-C remains the default/oracle while performance, LLVM coverage, no-toolchain distribution gates, canonical symbols, and typed-IR convergence continue. v0.99.74a fixes imported-module math parity for `*`, `/`, and `%` across native-C, self-hosted-C, LLVM, and typed-ir-c; adds `stdlib/env.hey` for environment defaults; and keeps release A/B/C scripts step-labeled with default `HEY_BACKEND_WORLD_JOBS=8` while allowing local overrides. Canonical module-qualified symbols are still consumed by scalar/string/imported typed-IR C user-call subsets, `heyc symbols` remains the canonical symbol-table surface, and no-toolchain smoke coverage now includes imported typed-IR C emission.

Build the source checkout

make clean
make
./bin/heyc --version
./bin/heyc --backends

Run a program

./bin/hey examples/hello.hey --run
./bin/heyc examples/json_arrays.hey --test

Prefer readable syntax in examples:

let xs = ['Ada']
says to_json([...xs, 'next'])

The lower-level `push(collection, value)` primitive still exists, but public examples should use collection spread syntax unless the primitive itself is being taught or tested.

Friendly Hey expressions

./bin/hey
./bin/hey hay
./bin/hey expressions
./bin/hey expressions --random

The catalogue lives in `stdlib/hey_expressions.hey`, so docs, commands, and future rotating taglines can draw from the same maintained list.

Use the REPL

./bin/hey-repl

Example transcript:

hey> let xs = ["Ada"]
hey> says to_json([...xs, "next"])
["Ada","next"]
hey> quit

Build a native app

./bin/heyc build --backend native-c examples/hello.hey -o build/hello
./bin/heyc build --backend self-hosted-c examples/hello.hey -o build/hello-selfhosted
./bin/heyc build --backend llvm examples/llvm_runtime_call_smoke.hey -o build/llvm-runtime-call
./build/hello
./build/hello-selfhosted

`native-c` is the active/default backend. `self-hosted-c` is promoted selectable with `--backend self-hosted-c` or shorthand `--selfhosted`; native-C remains default/oracle while performance and distribution gates continue. `llvm` is selectable with `--backend llvm` or shorthand `--llvm` for the checked scalar/string/control-flow/stdout, string file-I/O, string slice/compare, literal for-loop, string-returning functions including helper calls, data-model/file/text/process subset, and jobs descriptor subset. It still requires maintainer clang until the no-toolchain distribution slice is complete, and it intentionally does not lower 1BRC-specific helper calls as benchmark proof shortcuts.

Use `HEY_BACKEND` when you want one-binary side-by-side builds without repeating backend flags:

HEY_BACKEND=native-c ./bin/heyc build examples/hello.hey -o build/hello-cnative
HEY_BACKEND=selfhosted ./bin/heyc build examples/hello.hey -o build/hello-selfhosted
HEY_BACKEND=llvm ./bin/heyc build examples/llvm_runtime_call_smoke.hey -o build/llvm-runtime-call

Explicit `--backend`, `--cnative`, `--selfhosted`, and `--llvm` flags still win over `HEY_BACKEND`.

Run the full suite

Run the fast suite, dual timing, and the parallel backend-world gate as the final proof block:

./bin/test --full
./bin/test --dual-timing
HEY_BACKEND_WORLD_JOBS=8 ./bin/test --promotion-report

`./bin/test --full` is the native C oracle suite. `./bin/test --promotion-smoke` is the compact parser/build/spec/REPL promotion guard. `./bin/test --promotion-report` runs the full promotion evidence bundle, including backend-world, self-hosted build-time focus, and LLVM lowering-impact reporting.

Run challenge harnesses

./bin/test --perf-baseline
./bin/perf baseline --label local-smoke --no-compare --quiet
./bin/perf 1brc-scale --rows "1000 10000 100000 1000000" --workers "1 2 4 8" --implementations "native-c self-hosted llvm" --out build/1brc-scale
./bin/perf 1brc-full --data ../1brc/measurements.txt --rows 1000000000 --workers "8" --no-progress --out build/m3-1brc-proof-no-progress
ulimit -n 30000
./bin/perf c10k-proof --clients 10000 --out build/m3-c10k-proof
./bin/perf ring --nodes 1000 --hops 100000 --workers 8 --out build/m3-ring-proof

See performance challenge goals and 1BRC scale-confidence matrix.

Make a distribution

To copy the current platform distribution into another app repository, use `./bin/hey-vendor --repo PATH`. See Vendor stdlib.

make dist-host
./bin/test --dist-gates
cd dist/hey-0.99.70a-darwin-arm64
./bin/heyc --version
./bin/heyc --backends

Linux builders create `dist/hey-0.99.70a-linux-x64`.

Make the source handoff zip

./bin/hey-zip-source --downloads

This writes `~/Downloads/hey-lang-bootstrap-plan-v0.99.70a.zip` and excludes `.git`, `build`, `dist`, `.hey`, and generated dSYM artifacts.

No-toolchain bundle use

A release bundle can run Hey source and the REPL directly with its shipped binaries:

bin/heyc examples/hello.hey --run
printf 'says "hi"\nquit\n' | bin/hey-repl --quiet

Compilation is still split by backend: Native-C remains the default build path, while LLVM is selectable for a checked scalar/string/control-flow/stdout, string file-I/O, string slice/compare, and literal for-loop subset but still needs the next no-toolchain compiler bundle before end users can compile without a local compiler.

Timing/backends note

Use `./bin/test --backend-timing` or `./bin/test --triple-timing` when you need to see LLVM separately from native-C and self-hosted. LLVM is a third backend, not an accelerator for either of the other two. Sleep accepts decimal milliseconds such as `sleep 0.5`; prefer explicit synchronization over sleeps when correctness depends on ordering.

Promotion proof timing

Use the timed proof form when reporting full promotion evidence:

time HEY_BACKEND_WORLD_JOBS=8 ./bin/test --promotion-report

Repeatable release verification

Use the maintained release scripts instead of re-pasting the long smoke list:

./bin/test --release-c
./bin/test --release-a
./bin/test --release-b

`--release-c` is the per-slice latest-feature/all-three-backend proof and must be updated whenever a new `.hey`/`.hir` mini-spec lands. `--release-a` is the source-tree release gate bundle. `--release-b` verifies the packaged dist. Keep the final promotion report as a single command.

v0.99.73a typed-IR string trim/slice note

v0.99.73a continues the Track C proving-backend ladder: `typed-ir-c` now lowers `trim(...)` and `slice(...)` string helpers in mixed scalar/string programs. The mini-spec is `specs/language/692_typed_ir_c_string_slice_trim_backend.hey`, with a matching IR golden in `specs/ir/692_typed_ir_c_string_slice_trim_backend.hir`. It is expected to stay green across native-C, self-hosted-c, and LLVM.

v0.99.72a typed-IR helper note

v0.99.72a fixes the remaining typed-ir-c comparison warning from the v0.99.71a smoke run, tracks bool helper locals in typed IR, and widens `typed-ir-c` with `starts_with`, `ends_with`, and `find` for mixed scalar/string programs. The proving mini-spec is `specs/language/691_typed_ir_c_string_predicate_backend.hey`, with an IR golden in `specs/ir/691_typed_ir_c_string_predicate_backend.hir`.

v0.99.71a packaged promotion note

The packaged distribution now includes the REPL test harness needed by `./bin/test --promotion-report`. From inside `dist/hey-<version>-<platform>`, `./bin/test --repl-backend` and `./bin/test --repl` run without relying on source-tree Makefile targets.

v0.99.74a import/math parity note

v0.99.74a fixes boxed/native and self-hosted lowering for `*`, `/`, and `%` in imported module functions while keeping `typed-ir-c` and LLVM green. The mini-spec is `specs/language/693_imported_module_math_parity.hey`, supported by `specs/language/support/import_math_helpers.hey` and `specs/ir/693_imported_module_math_parity.hir`. It proves imported module math across native-C, self-hosted-c, LLVM, and typed-ir-c. The release A/B/C scripts now print step banners before each group so pasted failures show which gate was running.
