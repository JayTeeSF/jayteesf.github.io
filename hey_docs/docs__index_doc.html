v0.99.155a update: moves another remaining LLVM mass cluster through a cleaner compiler-services/jobs facade ABI instead of ad hoc shelling. LLVM now lowers Hey expressions catalog nullary/count/at calls, hey_c.emit_text/emit_file through public tiny-C compiler-services semantics, raw string extraction for boxed string fields in string contexts, and jobs start_default, string-actor start, cancel, and recover. New spec 805 pins the combined expressions/compiler-services/jobs path; existing rows 676, compiler emit pilots 440/450, and jobs rows 392/393/394 build/run byte-identically under LLVM. Latest pasted full-world evidence before this slice was LLVM good=421/521 with unsupported=86, build_failed=1, run_failed=13, output_mismatch=0, native-C good=505/521, and self-hosted good=504/521.

v0.99.153a update: fixes promotion-report expected-error AWK portability, single-quoted raw-string parsing in LLVM call/argument scanners, self-hosted docs/time runtime facades, and the LLVM imported-facade build-failure class. LLVM now lowers json.decode single-quoted raw strings, static_files.resolve/read, EHY src/result/new/encoding, docs metadata calls, and time.rows_per_second through runtime-ABI shims; specs 290/646/674/712/794/799 plus new spec 803 are the targeted proof cluster. Latest pasted promotion-report evidence before this slice showed LLVM good=400/519, unsupported=76, build_failed=31, run_failed=12, and output_mismatch=0, so the active goal is now closing those remaining full-world rows rather than the older 362/504 baseline.

v0.99.152a update: LLVM now lowers the public source-tooling facade cluster instead of isolated rows. Markdown link scanning/checking, hey_syntax classification/check_text, hey_parser/hey_parser_expanded parse_text/body shapes, hey_ast form/manifest/manifest_files/symbol_index_files, and hey_semantics analyze_text/analyze_file now route through checked runtime-ABI shims. Existing specs 397/420/430/470/480/568/595/596/656/679 build/run byte-identically on LLVM, and spec 802 plus its IR golden pin the cluster.

v0.99.150a update: LLVM now lowers the docs/introspection metadata facade cluster through general runtime-ABI shims for docs.lookup, docs.has, docs.symbols, and docs.format, and scalar functions with string parameters can call find/docs.has/fail paths without falling back to native. Specs 794 and 796 now build/run byte-identically on LLVM, and spec 797 pins the combined docs facade + string-param scalar-function path.

v0.99.148a update: the first alpha introspection CLI is live. ./bin/hey doc SYMBOL and ./bin/heyc --describe SYMBOL now print signature/argument/return-shape metadata from stdlib/hey_docs.hey via tools/hey_docs.hey and bin/subcmds/hey-doc, with spec 795 pinning both command surfaces. Next slices should route REPL help to the same catalog and then add shape-aware docs smoke.

v0.99.147a update: the tutorial is now contract-first instead of release-note-like, and docs/18-introspection-and-api-docs.md plus stdlib/hey_docs.hey establish the first testable doc/signature metadata foundation. The immediate introspection plan is docs-first metadata, ./bin/hey doc and heyc --describe, REPL help, shape-aware docs smoke, then source-near annotations, dot field access, better argument diagnostics, generated API reference, and explain/trace mode. LLVM priority work remains clean-but-incomplete from the latest uploaded evidence, with the v0.99.146a collections cluster preserved as priority proof good=42/42.

• `native-c` is the active/default backend and remains fully available.

• `llvm` is selectable for a checked scalar/string/control-flow/stdout subset. 1BRC/C10K program-shape shortcuts are intentionally unsupported; public runtime helpers may lower normally, and challenge rows can pass through ordinary language/runtime lowering.

• `self-hosted-c` has `supported=12 unsupported=0`, `direct_count=12`, and `bridged_count=0`; production jobs/imported-stdlib is now direct; parser/build/spec/REPL parity and backend-world are green; `self-hosted-c` is promoted selectable while native-C remains default/oracle.
