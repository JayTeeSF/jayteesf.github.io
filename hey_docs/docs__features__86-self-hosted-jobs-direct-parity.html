v0.98.42a: Self-hosted production jobs direct parity

This slice eliminates the final tracked dual-suite bridge row without changing the active/default backend. Native-C remains the oracle/default compiler for the full language, but every tracked self-hosted compatibility fixture now lowers through the Hey-written emitter instead of the native C-emission bridge.

What changed

• Moved `examples/jobs_production.hey` from bridged to direct self-hosted coverage.

• Added direct runtime-backed jobs lowering for the production jobs fixture:

• queue config objects

• `jobs.actor_spec(...)`

• `jobs.start(...)`

• `jobs.perform_with_id(...)`

• `jobs.result_to(...)`

• `jobs.stop(...)`

• worker `jobs.complete(...)` and `jobs.fail(...)`

• Added parser support for the multiline `let queue = jobs.queue({ ... })` call shape used by the production jobs example.

• Changed `hey_c.emit_file` to fail unsupported constructs explicitly instead of automatically using the native-oracle bridge.

• Kept the old native-oracle helper as an explicit compatibility function only, not the default emitter path.

• Added specs for implementation-independent multiline object behavior and direct jobs emission without the native-oracle marker.

Current coverage

`./bin/subcmds/hey-selfhost-coverage` reports:

hey_selfhost_coverage
true
12
0
12
0

The columns are `ok`, `supported_count`, `unsupported_count`, `direct_count`, and `bridged_count`.

What this does not mean

`bridged_count=0` means the tracked dual suite no longer depends on the native C-emission oracle bridge. It does **not** mean self-hosted is the primary compiler. Promotion still requires parser/build/spec/REPL parity gates, diagnostics parity, backend handoff parity, and performance confidence.

Spec debt follow-up

The next roadmap now includes a pass to review the language for missing implementation-independent specs discovered during self-hosting. The review should cover imports, modules, multiline literals, actor/job runtime calls, backend handoff forms, diagnostics, and REPL transcripts.
