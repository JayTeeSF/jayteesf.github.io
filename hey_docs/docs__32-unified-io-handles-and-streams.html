Unified I/O handles, streams, mappings, and actor suspension

Current truth

The first byte/mapping layer is implemented in v0.99.225a:

  • immutable refcounted `Bytes`;
  • retained zero-copy byte views through `bytes.slice`;
  • explicit text encode/decode;
  • byte indexing and delimiter search;
  • mutable `ByteBuffer` with append, clear, length, capacity, and freeze;
  • positioned byte reads and `files.map_readonly`;
  • general bounded argv execution with process-group termination;
  • mutable actor-owned byte-keyed integer tables used by parsers/aggregators.

The complete common live-handle model is **not implemented**. Current mappings

are returned as immutable `Bytes` whose backing unmaps automatically when its

last retained view is released; there is not yet an explicit mapping handle or

`close` method. Stateful files, child streams, pipes, normalized sockets,

reactor suspension, UDP, and TLS remain open work.

Do not represent live handles as JSON descriptor objects. Descriptors are useful

for plans and diagnostics, not operating-system resource ownership.

One runtime model

Use opaque typed handles with explicit capabilities:

  • `Readable`: `read`, `read_exact`, `read_until`, `read_line`;
  • `Writable`: `write`, `write_all`, `flush`;
  • `Seekable`: `seek`, `tell`, `read_at`, `write_at`;
  • `Closable`: idempotent `close`, `closed?`;
  • `DeadlineAware`: default and per-operation deadlines;
  • `Pollable`: readiness registration for actor suspension;
  • `HalfClosable`: socket/process-pipe shutdown where supported.

User-facing methods should lower to a small canonical MIR vocabulary:

io.read(handle, mutable_buffer, offset, length, deadline)
io.write(handle, byte_view, offset, length, deadline)
io.read_at(handle, file_offset, mutable_buffer, offset, length, deadline)
io.seek(handle, offset, origin)
io.flush(handle)
io.close(handle)
io.poll(operation_set, deadline)

LLVM cannot optimize a kernel syscall. It can optimize the surrounding Hey loop

when handles, offsets, lengths, byte views, buffer ownership, and Result tags are

typed rather than routed through strings and JSON objects.

Byte foundation now available

import 'stdlib:bytes'
import 'stdlib:files'

let mapped = files.map_readonly(path).value
let view = bytes.slice(mapped, start, count)       # retained, zero-copy
let separator = bytes.find_byte(view, 59, 0, view.length)
let text = bytes.decode(view, 'utf-8')

let output = bytes.buffer(64 * 1024)
bytes.append(output, view)
bytes.append(output, "\n")
says bytes.buffer_length(output)
let frozen = bytes.freeze(output)
bytes.clear(output)

`bytes.freeze` currently copies the buffer into immutable Bytes. Future ownership

transfer may eliminate that copy when the buffer is uniquely owned.

Files and mappings still to add

let file = files.open(path, 'read').value
let chunk = file.read(64.kibibytes)
let line = file.read_line(1.mebibyte)
let positioned = file.read_at(offset, 64.kibibytes)
file.seek(0, 'start')
says file.tell().value
file.close()

let mapping = files.map_readonly_handle(path).value
let shard = mapping.slice(start, finish)
mapping.close()

The existing `files.map_readonly(path)` remains useful porcelain returning

Result[Bytes]. The explicit handle form must make close/lifetime behavior clear:

views retain the underlying mapping, close prevents new views, and physical

unmapping occurs only after the last retained view is released.

Stateful sequential readers remain important for bounded-memory workloads:

for line in file.lines(limit: 1.mebibyte)
  consume(line)
end

Common stream surface

let chunk = stream.read(64.kibibytes)
let exact = stream.read_exact(16)
let frame = stream.read_until("\n", 1.mebibyte)
let wrote = stream.write(view)
let complete = stream.write_all(view)
stream.flush()
stream.close()

All operations return Results carrying partial counts, EOF, timeout, cancellation,

and operating-system errors. Empty bytes are valid data, not an error sentinel.

Processes and pipes

`process.run_argv_timeout` is convenience porcelain, not the final process API:

let child = process.spawn(args, {
  stdin: 'pipe',
  stdout: 'pipe',
  stderr: 'pipe'
}).value

child.stdin.write_all(request)
child.close_stdin()
let output = child.stdout.read(64.kibibytes)
let status = child.wait(timeout: 30.seconds)
child.terminate()
child.kill()

Child stdio and `pipes.create()` must implement the common stream capabilities,

with bounded buffering and backpressure.

TCP, UDP, and TLS

TCP listeners/connections should replace descriptor-shaped operations and

`close_fd` with opaque handles implementing the stream contract. UDP remains a

datagram handle rather than a stream. TLS wraps an existing byte stream and

validates certificates by default.

Actors, await, select, and cancellation

Actors should own handles when policy requires ownership; they should not proxy

every byte. Blocking I/O must suspend only the current lightweight process:

let read = await stream.read(64.kibibytes)
let accepted = await listener.accept()
let ready = await select([
  stream.readable(),
  timer.after(5.seconds),
  cancel_token.cancelled()
])

Use `kqueue` on Darwin/BSD and `epoll` on Linux, with a portable poll fallback.

Cancellation must unregister work and preserve handle ownership safely.

Implementation order

1. **Done:** Bytes/views, ByteBuffer, mapped bytes, positioned byte reads,

delimiter search, bounded argv timeout, byte-keyed integer table.

2. Opaque handle storage, capability checks, ownership, close, and Result errors.

3. Stateful files and explicit mapping handles.

4. Live child-process handles and bounded pipe streams.

5. Normalized TCP listener/connection streams.

6. Reactor-backed actor suspension, deadlines, cancellation, and `select`.

7. UDP, TLS, locking, and platform-specific advanced controls.
