Roadmap audit and open goals

This is the current aligned roadmap index. It reconciles the older feature notes, planned-version notes, open questions, backend plans, and performance docs so the next slices can be chosen from one ordered list.

Completed / aligned goals

1. **Core language surface** — Values, expressions, `let`/`set`, functions, modules, imports, JSON/hash/array literals, spread updates, strings/bytes/codecs, control flow, file/process/text facades, and readable public examples are implemented and covered by persistent specs.

2. **Actor and supervisor foundations** — Actors, mailboxes, actor refs in values/literals, registry helpers, supervisors, process-tree info, links, monitors, strategies, and negative named-actor/supervisor runtime errors are implemented and covered by specs.

3. **Jobs as concurrency porcelain** — Jobs queues, worker refs, owned pools, retries, dead letters, status/cancel snapshots, durable-backend descriptors, and await-style `jobs.result_to(...)` are covered. LLVM now lowers the descriptor layer (queue/duration/backend/job-id/envelope), while actor-backed queue execution remains an explicit ABI follow-up. Correctness no longer relies on human-scale sleeps.

4. **REPL parity surface** — REPL specs, backend matrix smoke, stdlib JSON imports, jobs/file/evented descriptors, and `stdlib/hey_c.hey` source imports are green.

5. **Runtime soundness gates** — Interpreter global-env access is guarded, actors are drained before teardown, and `sanitize-smoke` runs TSan/ASan coverage. Leak auditing remains open, but the known thread-safety crash class is closed.

6. **Native-C backend** — Native-C remains the active/default compiler and behavioral oracle. Full backend-world is green except intentional expected-error runtime-failure rows.

7. **Self-hosted-C promotion** — `self-hosted-c` is a promoted selectable full-parity alternative. JayTeeSF's v0.99.38a promotion report showed parser/build/spec/REPL/backend-world all green, with `native-c` and `self-hosted` both at `good=330 total=338`, `unsupported=0`, `output_mismatch=0`, and `build_failed=0`; parser/build/spec/REPL/backend-world gates all ended `status=ok`.

8. **Backend selection surface** — The one-binary path is `bin/heyc` / `bin/hey`; wrapper-script variants were removed. Use explicit `--backend native-c|self-hosted-c|llvm`, shorthand `--cnative|--selfhosted|--llvm`, or `HEY_BACKEND=native-c|selfhosted|llvm`.

9. **LLVM honest subset** — LLVM is selectable and should keep no output mismatches/build failures in backend-world, and unsupported constructs are explicit. It is not yet full-compliance or no-toolchain-ready.

10. **Backend evidence tooling** — `backend-world-full`, `backend-world-summary`, `backend-world-focus`, weighted LLVM lowering-impact rows, `promotion-smoke`, and `promotion-report` are the current evidence loop.

11. **Source packaging and handoff** — Source zips exclude generated/build artifacts, keep source handoff instructions, and use the short local loop: apply zip, run `promotion-smoke`, then `promotion-report`.

12. **Documentation cleanup** — Obsolete redirect/superseded docs were removed in v0.99.38a. The docs index now points at maintained canonical docs instead of redirect stubs.

13. **1BRC/C10K/ring challenge signals** — These are recurring performance/reliability signals. They are not beta blockers unless explicitly promoted to release gates.

Open goals / feature queue

Completed in v0.99.54a: decimal millisecond sleeps are supported, non-semantic fixture sleeps are trimmed, and LLVM remains timed as a separate backend rather than an accelerator for native-C/self-hosted.

2. **Planned: reduce self-hosted build time.** The slow rows are still self-hosted compiler/bootstrap/parser/tooling builds, often 14-32s per backend-world row. This is now performance work, not correctness work.

3. **Planned: Track B canonical symbols/imports.** Finish one canonical module/function/actor/import table consumed by native-C, self-hosted-C, LLVM, and REPL. Some practical routing is already aligned, but the structural table remains open.

4. **Planned: Track C typed IR as the real backend input.** Extend `hey-ir` contracts/goldens and make LLVM consume IR instead of rescanning source for the lowered subset. Later, make self-hosted emit the same IR.

5. **Planned: LLVM high-impact lowering.** Use the weighted lowering report. Current v0.99.49a data still says strings dominate (`strings` weighted_score=2964 of 4514), followed by other, actors/functions, and data-model. Prioritize string values in functions, predicates, remaining split, find/len/concat before more actor/job runtime lowering unless fresh data changes the ranking.

6. **Planned: LLVM full-compliance ladder.** Continue the general lowering rungs: data model, text/file/process, modules, actors, jobs, evented I/O. Avoid benchmark-specific shortcuts.
