v0.99.382a

  • Synchronized the native fast dispatcher with source-index v2 using an

unambiguous `request_mode` field, restoring true compiler-free warm dispatch.

  • Replaced `/usr/bin/time` in the maintained script-startup benchmark with Hey

`Time.now_ns` measurements and required `backend=llvm` on warm cache hits.

  • Moved source-archive HIR/MIR checker and script-audit builds onto the ordinary

LLVM product compiler; Stage-0 remains explicit comparison/recovery only.

  • Accepted the LLVM-first toolchain and REPL architecture: one parser, one

semantic analyzer, semantic HIR, ABI MIR, one LLVM product backend, and a

minimal native runtime. Stopped release products from forcing the

native-C-hosted Hey frontend.

  • Added a machine-readable inventory for every maintained native source file

and current-slice language/compiler contracts for the native boundary.

  • Follow-on slice: fixed a real `Process.run_env_argv`/`exec_env_argv` bug where

`hey_envp_from_object` let a stale inherited environment variable silently win

over an explicit override (duplicate `envp` keys, first-match `getenv()`),

which broke isolated-cache script-startup/source-runner benchmarks in any

shell with ambient `HEY_*` variables. Wired `.reflection.json` sidecar

generation into the native `stage0-build` path (`heyc_build_native_app`),

closing a reflection-metadata parity gap versus the LLVM lane. Fixed a stale

`--backend native-c` invocation in `tools/check_compiler_ir_contract.hey`.

Updated 11 compiler-contract specs whose literal-string assertions had gone

stale after the native-C-private/stage0 rename and unrelated cache-key version

bumps (`hey-run-v6`→`v7`, `hey-sidecars-v4`→`v5`, `hey-run-source-index-v1`→`v2`).

`test/run_compiler_contract_verify.sh` is green (`specs=189`).

v0.99.377a

  • Added `runtime/hey_semantic_builtins.h` as the shared source-level builtin

validation contract.

  • Enforced two-argument builtin `get` arity before HIR/MIR for `--check`,

`--run`, self-hosted `build`, and `build-dir`.

  • Kept module methods named `get` distinct from the unqualified builtin.
  • Covered valid and invalid multiline builtin calls.
  • Added interpreted/direct-LLVM parity for `HeySemantics.analyze_text`.
  • Added source/function/IR provenance to direct-LLVM unsupported diagnostics.
  • Proved valid boxed query-row, index, and property access still lowers through

direct LLVM.

  • Regenerated canonical HIR/MIR goldens for language version v0.99.377a.
  • Kept database adapters and product code outside Hey core.
  • Hardened source handoffs against case-insensitive-host false positives by

validating the complete canonical stdlib module ledger against exact ZIP

entry names and rejecting case-folded entry collisions before extraction or

tracked deletion.

  • Corrected process exit-status capture when Hey is launched with inherited

`SIGCHLD` auto-reaping, covering `Process.run_argv`, live `Process.spawn`/`wait`

handles, and compiler test subprocesses.

  • Restricted compact handoff verification commands to shell-fenced documentation,

preventing maintained fixture paths from appearing as commands.

  • Replaced operating-system command assumptions with one generated C child fixture

used by Unix native-C/direct-LLVM process tests and Windows native process receipts.

  • Added a current-slice verification tier that runs active release-candidate

regressions before the broad platform matrix.

  • Made malformed source-ZIP fixtures portable to case-insensitive filesystems by

mutating central-directory entries instead of creating case-colliding files on

disk. The behavioral shell regression now runs once with direct diagnostics,

followed by a fast structural language contract instead of a duplicate nested run.

  • Included the source installer fail-fast contract and shell regression in the

current-slice tier so changed handoff code is never omitted from the fast gate.

  • Removed duplicate nested execution and a redundant native checker build from the

source-handoff regression; behavior runs once with direct diagnostics, followed

by a fast structural contract.

  • Replaced stale prompt-compaction subtest assertions with stable current-slice

aggregate and fixed-string invariants that print exact missing/forbidden text, and made the

verifier report the failed phase label and status before exiting.

  • Hardened the adjacent source-handoff policy gate with named diagnostics and an

executable proof that source-ZIP checksum commands remain opt-in only.

  • Updated the legacy dynamic-removal compiler contract to prove the current exact-case

central-directory, canonical-ledger, staged-inventory, and deletion ordering instead

of requiring a retired error string.
