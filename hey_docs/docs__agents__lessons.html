Agent lessons

Measured scheduler changes

  • An atomic queue-depth hint can remove empty-victim mutex traffic without making the hint authoritative; the queue lock still decides ownership.
  • Bound ordinary steal probes, but retain a periodic full sweep so a local optimization cannot starve work on an unlucky peer.
  • Yielding after a failed probe round can suppress hot retry storms caused by short publication windows without adding a new scheduler thread.
  • Alternate old and new binaries on the same host. Separate one-worker preservation from multi-worker improvement and report medians, not one favorable run.
  • Do not promote registry partitioning when direct profiles show ready-queue and stealing costs dominate.

Preserve product truth

  • The public compiler is LLVM-hosted Hey -> semantic HIR -> ABI MIR -> direct LLVM. Do not reintroduce historical application backends into user docs or CLI choices.
  • Keep native Stage-0 and the C-hosted producer private for bootstrap, recovery, and byte-equivalent MIR evidence.
  • A run-ready native application is not the same as a self-contained compiler SDK. State toolchain, sysroot, runtime-library, and target limitations explicitly.
  • Do not advertise a platform until a native build/run/release gate exists there.

Keep actors honest

  • A lightweight actor is still a real mailbox, state, identity, and lifecycle—not a benchmark-only counter record.
  • Measure resident density separately from runnable throughput. A process ring often has only one runnable actor and is not a multicore scalability proof.
  • Warm fixed worker infrastructure before estimating per-actor memory.
  • Actor creation, scheduling, join, stop, and process shutdown must remain near O(1) or bulk-linear; small actor records alone do not prove scale.
  • A global ready queue can dominate fan-out even when actors are pooled. Profile lock contention before increasing worker counts.
  • Keep memory ceilings explicit and stop capacity tests before destabilizing the host.

Compiler/runtime ownership

  • Preserve mailbox ordering and wait for the final pooled actor step before releasing actor state.
  • Bulk shutdown may detach dormant actors, but must release messages, args, state, links, monitors, and registry ownership exactly once.
  • Do not release an object after a consuming/aliasing boxed store unless ownership is proven independent.
  • Live OS resources remain opaque handles; timeout and cancellation must be enforceable rather than post-hoc measurements.
  • Language semantics, HIR/MIR policy, scheduler policy, and workload algorithms belong in Hey where possible; the C runtime supplies general ABI services.

Performance and challenge integrity

  • Add general primitives only when they are independently useful and documented. Never branch on benchmark names, filenames, source shapes, or expected answers.
  • Compare byte-identical output and separate build/startup/map/scan/merge/format costs.
  • Alternate benchmark versions and report best plus distribution/mean; small changes can be noise.
  • Use target-machine profiles before adding more runtime surface.

Documentation and handoff

  • Remove obsolete implementation narratives rather than labeling them “historical” in the maintained docs; Git preserves history.
  • Keep callable, actor, I/O, compiler, distribution, and site-generation docs synchronized with executable contracts.
  • Documentation layout must be owned by semantic local markup/CSS; optional CDN themes may decorate it but must not control the structural grid.
  • Preserve executable modes in source and distribution archives.
  • Keep copy/paste command blocks free of pager-producing `git diff --stat` commands.

Scheduler and archive lessons

  • Every pointer-backed MIR local that can be loaded in cleanup or a later block must be allocated in the LLVM function entry block; loop-local `alloca` instructions do not dominate function-level cleanup.
  • Scheduler telemetry must be opt-in and should not add unconditional atomics or shared-lock traffic to the performance path being measured.
  • Per-worker queues remove one central lock but do not guarantee speedup when actor turns are smaller than registry, mailbox, allocation, reference-count, and wakeup costs.
  • Remove scheduler experiments that regress repeated multi-worker results. A lower park count is not success if steals and elapsed time rise.
  • Never persist a static list of paths to delete before applying a future source archive. Derive deletions from the current repository's Git-tracked paths and the exact downloaded archive.
  • A release graph is not closed merely because its self-compiler binary exists. Run the complete distribution graph and execute staged tools from an unrelated working directory with source-checkout environment variables unset.
  • Custom `BUILD_DIR` values expose hidden bootstrap coupling. Every release compiler invocation must receive the exact release-local bootstrap path rather than falling back to wrappers or global `build/` artifacts.
  • Cached applications cannot infer an SDK root from their own cache path. The launcher must propagate the resolved installed root and stdlib path into the application process.

Concurrency surface

  • Use actors only for identity or persistent private state.
  • Use stateless workers for replicated parallel handlers and keep their mailbox runtime-private.
  • Put retries, timeouts, scaling, backpressure, dead letters, metrics, and supervision in Jobs rather than duplicating policy in actors.
  • Keep streams familiar (`map`, `filter`, `each`) while implementing parallel stages with bounded internal queues.
  • Do not add general-purpose user-facing channels or expose shared mutable state.
  • Reject worker state/mutation in semantic analysis, not after LLVM lowering.
  • Receiver methods that require parentheses must not reserve the same bare object-field name; `pool.status()` and `message.status` are distinct syntax.

Keep one concurrency runtime

  • Do not preserve an obsolete actor-backed Jobs API beside worker-backed Jobs; Git is the compatibility archive.
  • Jobs bind real worker identity, not strings or hidden reply actors.
  • Result waiting timeouts are not execution timeouts; label them honestly until work can be interrupted.
  • Bound result and dead-letter storage, and reject work rather than silently violating configured capacity.
  • Delayed retries must wait outside worker threads; one bounded timer queue is preferable to sleeping worker instances.
  • Idempotency memory must be bounded, and duplicates must reuse the original receipt rather than enqueue another handler turn.
  • Imported workers must preserve declaration identity through HIR/MIR; never fall back to function-name strings.
  • REPL examples must be replayed exactly, including whitespace and multiline call formatting.

Adaptive Job scaling

  • Scale worker-backed Jobs from observable queue age and bounded backlog pressure, not raw CPU guesses.
  • One shared controller can manage every adaptive pool; do not create one operating-system thread per pool or worker.
  • Keep fixed and adaptive configuration mutually exclusive so operational intent is obvious.
  • Rate-limit both growth and retirement, expose desired/current/peak counts and decision reasons, and make policy clocks deterministic in specs.
  • Internal release contracts are not substitutes for a real cross-platform vendor invocation; record external failures explicitly.

Stream ordering

  • Completion order can be observed by polling only the already-bounded receipt window; a public channel is unnecessary.
  • Carry sequence tags privately and cap every reorder group at the originating queue/result window.
  • Retries resolve inside Jobs before stream ordering; failed, cancelled, timed-out, or missing terminal results must fail the stream instead of manufacturing a sequence hole.

Partitioned Jobs

  • Keep partition routing inside the existing Job pool; do not create one thread, actor, or public channel per key.
  • A fixed shard count gives bounded allocation and shutdown. Canonical key hashing makes routing stable within the runtime contract.
  • Hash collisions honestly serialize extra keys. Expose per-shard load and contention rather than claiming perfect key distribution.
  • Partitioning guarantees non-overlap; it is not an output-order mechanism. Use sequence tags and `.resequence` when callers require ordered results.
  • Preserve a receipt's shard across retries so one key cannot overlap itself during retry handling.

Thin concurrency syntax

  • Lower declarative concurrency sugar before HIR whenever it has no independent semantics. `job Name ... end` must remain equivalent to one ordinary `jobs.define(...)` binding.
  • Keep explicit construction available for conditional and function-local use; syntax convenience must not remove the underlying composable API.
  • Configuration blocks that contain `worker WorkerName` are not nested worker declarations; block scanners need a Job-specific terminator rule.

Measurement-first scheduler work

  • Separate registry, mailbox, message-pool, queue, handler-time, and value-ownership costs before changing scheduler policy.
  • Profile counters must be opt-in; even relaxed atomics distort tiny actor turns when maintained unconditionally.
  • Repeated `1..cores` matrices are more useful than one best run. Preserve raw runs beside medians.
  • A plausible bottleneck is not a measured bottleneck. Do not partition the registry when queue locks and steal misses dominate the observed workload.
  • Long verification blocks are unsafe to paste while a foreground build is still reading terminal input. Prefer one maintained synchronous verification command that stops at the first failing phase.

Post-Windows learning platform

  • Documentation needs layered learning paths: fast start, guided tour, task tutorials, explanations, and reference.
  • Preserve third-party challenge provenance separately from maintained Hey-native exercises, and review redistribution rights before publishing copied text or solutions.
  • A browser REPL must use a constrained capability profile and resource quotas; browser convenience cannot become a second language definition.

Generation-2 scheduler

  • Publish global ready-depth accounting before making a queued actor available to a concurrent popper; otherwise a pop can observe zero, skip the decrement, and leave stale positive ready depth.
  • Keep external injection separate from worker-local continuation queues. This preserves locality without forcing outside threads to select or lock one worker queue.
  • Drain and steal bounded batches so one queue lock can transfer useful work without monopolizing a victim or creating an unbounded local burst.
  • Running, spinning, and parked counts need explicit transitions and a spinner cap; an unbounded set of idle stealers simply recreates the contention the local queues were meant to remove.
  • After the largest measured bottleneck is fixed, structural scheduler improvements may produce modest timing gains. Retain them only when correctness, accounting, and one-worker behavior remain sound.

Persistent reactor ownership

  • Keep one native epoll/kqueue registration set alive for the reactor lifetime; rebuilding the complete set per wait adds churn and weakens ownership reasoning.
  • Duplicate live descriptors at admission so closing or reusing the caller's descriptor number cannot retarget an in-flight readiness request.
  • Unregister every descriptor before freeing its registration record or closing the duplicate. Native event userdata must never outlive its request storage.
  • Stop and join the reactor before draining actors and workers so cancellation completions are delivered while their managed recipients can still run.
  • Validate shutdown with pending requests in a compiled or process-lifetime test; an interpreter spec may evaluate expected output before global runtime shutdown.
