Glossary

Every term this book defines, alphabetized, with the chapter of first

use.

  • **ABI (application binary interface)** — a contract about how data

is represented at a boundary, which both sides agree never to break

casually. MIR is Hey's ABI boundary. (Ch. 2)

  • **Actor** — an isolated concurrency unit owning private state and a

mailbox; processes one message at a time. (Ch. 5)

  • **Array** — an ordered boxed collection: `[1, 2, 3]`. (Ch. 4)
  • **Asynchronous** — proceeding without waiting; an actor send does

not block the sender. (Ch. 5)

  • **Binding** — a name for a value, introduced with `let`, rebound

with `set`. (Ch. 3)

  • **Bootstrapping** — building a self-hosted compiler starting from a

seed written in another language. (Ch. 12)

  • **Boxed value** — a value allocated in memory with a header

describing its kind, handled by pointer; how Hey represents arrays,

objects, and other structured/dynamic data. (Ch. 4)

  • **Bracket continuation** — a statement continuing across physical

lines while a `(`, `[`, or `{` it opened is unclosed; the one

exception to one-statement-per-line. (Ch. 3)

  • **Byte-identical** — equal bit for bit; Hey's standard of proof for

compiler outputs. (Ch. 12)

  • **Compiler** — a program translating source code toward machine

code, in Hey via a fixed pipeline of stages. (Ch. 2)

  • **Concurrency** — structuring a program as independently progressing

activities. (Ch. 5)

  • **Conversion** — the Ruby-style builtins `value.to_i` (to integer)

and `value.to_s` (to string); how Hey code controls a value's type

instead of writing annotations. (Ch. 4)

  • **Data race** — two activities accessing the same memory at the same

time, at least one writing; inexpressible in Hey. (Ch. 5)

  • **Escape sequence** — notation like `\n` for hard-to-type characters

in double-quoted strings. (Ch. 3)

  • **Fixed point (of a compiler)** — the state where the compiler,

compiling its own source, reproduces itself byte-identically.

(Ch. 12)

  • **Fixed-point arithmetic** — representing fractional quantities as

scaled integers (1.5 → 15 tenths) so arithmetic stays exact; Hey's

idiomatic path for decimal data, via `parse_fixed`. Unrelated to

"fixed point of a compiler." (Ch. 4)

  • **Floating point** — hardware approximation of fractional numbers

(`Float`/`f64` in Hey); available but deliberately not the default

path for decimal data. (Ch. 4)

  • **Fragment** — the small complete program a REPL turn compiles.

(Ch. 11)

  • **Gate** — a test-suite wrapper printing a single green/red receipt

line; commits require green. (Ch. 11)

  • **HIR (high-level IR)** — Hey's first intermediate representation:

the resolved, type-annotated program. (Ch. 2)

  • **Immutable** — never modified in place; all Hey values. (Ch. 1)
  • **Import** — a statement pulling a module into a program;

`import 'stdlib:Text'`. (Ch. 3)

  • **Intermediate representation (IR)** — an internal data format a

compiler uses between source and machine code. (Ch. 2)

  • **Interpolation** — splicing an expression's text into a

double-quoted string with `#{...}`. (Ch. 3)

  • **JIT ("just-in-time")** — compiling/loading machine code in a

running process, used by Hey's REPL. (Ch. 11)

  • **Job** — a runnable unit the runtime schedules onto the worker

pool. (Ch. 5)

  • **Joining** — waiting for an actor (or job) to finish. (Ch. 5)
  • **Linking** — combining compiled code with the runtime and system

startup code into an executable. (Ch. 2)

  • **Lowering** — translating a representation into a less abstract

one. (Ch. 2)

  • **Mailbox** — an actor's queue of incoming messages. (Ch. 5)
  • **MIR (mid-level IR)** — Hey's typed, *textual* middle

representation; the ABI boundary between front and back ends.

(Ch. 2)

  • **Module** — a named source file whose functions are used

qualified: `Text.contains?`. (Ch. 3)

  • **Object (record)** — a named-field boxed collection:

`{name: 'Ada'}`. (Ch. 4)

  • **Parser** — the stage checking grammatical form and building the

structured program description. (Ch. 2)

  • **REPL** — read–evaluate–print loop; Hey's compiles every input

through the ordinary pipeline. (Ch. 11)

  • **Restricted capability** — a power (like `Eval`) available only by

explicit opt-in. (Ch. 11)

  • **Result** — a value that is either success-with-answer or

failure-with-explanation; Hey's error-handling shape. (Ch. 3)

  • **Runtime (library)** — the small C library providing scheduling,

boxed-value services, and OS access to compiled programs. (Ch. 1)

  • **Scalar** — a value held directly in a machine register: `Int`,

`Bool`, `Str`. (Ch. 4)

  • **Seed / Stage 0 / Stage 1 / Stage 2** — the bootstrap ladder: the

foreign-language first compiler; the self-hosted compiler built by

it; the self-hosted compiler built by itself. (Ch. 12)

  • **Self-hosting** — a compiler written in the language it compiles.

(Ch. 1)

  • **Short-circuit** — `and`/`or` evaluating the right side only when

needed. (Ch. 3)

  • **Signature** — a function's parameter and return types; builtin

signatures live in a compiler table. (Ch. 4)

  • **Spec** — an executable expectation: program + exact expected

output, byte-compared. (Ch. 11)

  • **Statically typed** — types determined and checked before the

program runs. (Ch. 4)

  • **Stream** — a worker-backed data pipeline with map/filter/collect

stages. (Ch. 5)

  • **Type** — a classification of values determining valid operations.

(Ch. 4)

  • **Type inference** — the compiler deducing types from evidence

rather than annotations. (Ch. 4)

  • **Value** — an immutable piece of data. (Ch. 1)
  • **Worker** — a stateless function the runtime may run in parallel.

(Ch. 5)

  • **Worker pool / work stealing** — the runtime's fixed set of OS

threads, with idle threads taking queued jobs from busy ones.

(Ch. 5)
