Hey standard library

The standard library provides readable application-level APIs over the language and runtime primitives. Import a module with the `stdlib:` scheme:

import 'stdlib:json'
import 'stdlib:jobs'
import 'stdlib:web'

Application modules

ModulePurposeGuide
`stdlib:web`routes, requests, responses, middleware, and live HTTP servingWeb
`stdlib:jobs`supervised worker pools, queued work, retries, and resultsJobs
`stdlib:json`decode, encode, and canonically encode JSON over Hey valuesJSON codecs
`stdlib:static_files`safe path resolution and file responsesStatic files
`stdlib:collections`higher-level array and object operationsLanguage guide
`stdlib:text`text splitting, replacement, formatting, and normalizationLanguage reference
`stdlib:files`filesystem paths, reads, writes, traversal, and publication primitivesFiles
`stdlib:result`one shared success/error value contract across stdlib boundariesResult values
`stdlib:process`direct process execution with argument and environment controlProcesses and CLI
`stdlib:cli` and `stdlib:env`command-line options and environment valuesProcesses and CLI

Tooling modules

ModulePurposeGuide
`stdlib:packages`package manifests, locks, checksums, verified installs, and migration statusPackages
`stdlib:archive`deterministic ZIP packing and safe bounded extractionArchive
`stdlib:crypto`SHA-256 text and file digestsCrypto
`stdlib:http`HTTP request/response and client-side boundariesHTTP client
`stdlib:vendor`copy and verify Hey distributions inside an applicationVendor
`stdlib:docs`generate the curated static HTML documentation siteDocs generator
`stdlib:ehy`render embedded-Hey templatesWeb

Design rule

Hey keeps syntax focused on capabilities that libraries cannot safely provide themselves. Stable runtime plumbing supports standard-library porcelain, and the same public library surface should behave consistently across native-C, self-hosted-C, and LLVM.
