v0.99.259a current handoff

Major north-star goals

Hey keeps one public compiler path: LLVM-hosted Hey -> semantic HIR -> ABI MIR -> direct LLVM -> native executable. Distribution closure must package that exact compiler plus the target link/runtime dependencies, while minimal run bundles contain only an application and the native libraries it actually needs.

Current milestone

v0.99.259a owns target-side Unix SDK closure. SDK staging now accepts explicit compiler-toolchain, target-sysroot, and native-library roots; the installed compiler discovers all three relative to its relocated SDK and embeds both SDK-relative and run-bundle-relative library search paths.

Version-slice progress

  • **v0.99.257a repair:** persistent-reactor stress is descriptor-budget-aware and passes under `ulimit -n 256`.
  • **v0.99.258a:** bundled Clang, LLD, and Clang resource files; exact Linux non-default release graph and PATH-scrubbed compilation passed.
  • **v0.99.259a:** added target sysroot/native staging, SDK-relative `--sysroot` and native-library linking, `$ORIGIN`/`@loader_path` run-bundle fallback, and native-library-aware run bundles.

Working and open

Working:

  • Linux builds a minimal explicit sysroot from compiler-discovered CRT, libc, GCC runtime, OpenSSL, zlib, and zstd files;
  • a relocated SDK reports compiler, sysroot, native libraries, and complete SDK status as ready;
  • the relocated SDK compiles and runs a crypto/TLS application with compiler and library discovery scrubbed;
  • `ldd` resolves OpenSSL from the relocated SDK;
  • the complete observable inventory passes in segmented execution: 502 language specs, 30 REPL transcripts, and 92 examples;
  • the complete compiler-contract inventory, HIR/MIR, deterministic compiler IR, 20/20 conformance, sanitizers, docs, the 162-path verification manifest, and source-archive gates pass;
  • after the compiler SDK is moved away, a separately relocated run bundle still resolves OpenSSL/zlib/zstd through `$ORIGIN/../lib/hey/native` and executes;
  • Darwin uses the same staged layout, `ld64.lld`, a copied macOS SDK, rewritten `@rpath` OpenSSL install names, and `@loader_path` run-bundle fallback.

Still open:

  • the Darwin arm64 execution gate must run on the target Mac; Linux cannot substitute for that receipt;
  • the real Linux Docker vendor command must be rerun on the target Mac before cross-platform vendoring is called green;
  • Release-C remains independently blocked by `test/regression_mir_boxed_branch_dominance.sh`;
  • Windows begins only after the Darwin and Docker receipts are reviewed.

Direction reflection

Explicit toolchain/sysroot/native roots are a stronger boundary than copying opaque host directories. They make dependencies inspectable, let SDK and run bundles share one linker policy, and keep clean-target proof separate from source inspection. Retain this approach and require target-host receipts before Windows promotion.
