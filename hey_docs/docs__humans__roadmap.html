Roadmap after v0.99.346a manifest-driven package imports

Product direction

Hey source -> LLVM-hosted Hey -> semantic HIR -> ABI MIR -> direct LLVM
one signed product archive per OS/architecture, with no recipient C/C++ toolchain
immutable values + actors + stateless workers + Jobs + bounded streams
small CLI porcelain + hidden implementation plumbing
independently versioned useful packages and maintained applications

General-purpose channels are not public language primitives. Native resources

remain opaque handles. Optional libraries, database engines/drivers, frameworks,

and ORM-style porcelain are packages, not language subsystems.

Immediate milestones

1. **Installed package commands:** expose locked package-owned executable discovery so generated wrappers need not rely on sibling checkouts.

2. **Android target receipt:** build the generated JNI shared library and APK with a real NDK/Gradle toolchain, install it on an emulator/device, and retain manifest/action execution evidence.

2. **Physical iOS receipt:** run `hey mobile ios device-app` on a paired iPhone/iPad and retain compilation, development-signing, installation, and launch evidence; keep IPA/TestFlight/App Store evidence application-owned.

2. **Native machine receipts:** retain Linux and Apple Silicon `.release.json` sidecars from the exact prepared SDK and archive digest; Windows remains a separate structural SDK/runtime-harness track.

3. **Windows release verification:** rerun Release-C/A on macOS to confirm the repaired ordering and relocated compiler closure.

4. **Windows policy receipt:** run the filename-independent `START-HERE.cmd` bundle without typed PowerShell; preserve endpoint policy and return all evidence.

5. **Darwin publication:** native archive closure passed in v0.99.292a; complete Developer ID signing, notarization, SBOM/licenses, and reproducible publication.

6. **Windows product archive:** build release `hey.exe`/`hey-selfc.exe`, contained linker/runtime/import-library closure, and an Authenticode-signed installer/archive that requires no Visual Studio on the recipient.

7. **Release publication:** complete the v3 inventory with exact component versions/licenses, a complete SBOM, bound native receipts/source provenance, platform signing/notarization, CI reproducibility, digest publication, and rollback-safe indexing.

8. **20x improvement objective:** measure compile/link/startup/runtime/throughput/latency/RSS/allocation across scalar, collection, JSON, I/O, TLS, FFI, actors, workers, Jobs, streams, parser, web, and 1BRC workloads. Make correctness and ratio budgets blocking. Pursue 20x only where measured headroom exists.

9. **Useful package/application vertical slices before beta:** HTTPS/web, configuration, persistence/migrations, pooling, calendar, and email packages; fifteen-minute EHY demo; maintained event-scheduling app; package-neutral language reductions from real usage.

10. **Mobile and television targets:** continue native iOS/iPadOS and Android distributions, then add tvOS and Google TV/Android TV. Keep browser-hosted applications a separate web target rather than a mobile substitute. See `docs/humans/mobile-tv-target-roadmap.md`. See `docs/humans/native-mobile-plumbing-porcelain.md` for core, stdlib, CLI, package, and application ownership; extract package candidates before stdlib/core promotion.

11. **Beta gate:** three signed promoted archives, stable package contracts, blocking performance budgets, migration policy, and at least two maintained applications.
