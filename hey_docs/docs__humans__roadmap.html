Roadmap after v0.99.382a LLVM-first consolidation

Priority order

1. Continue the LLVM-first migration from the preserved Git checkpoint: v0.99.379a retired the public native-C backend, v0.99.380a added the canonical compile request, and v0.99.381a adds complete LLVM REPL fragments.

2. Rerun RecallCoach's direct native build and capture source/function/canonical

IR provenance for every remaining valid unsupported expression.

3. Reduce each valid expression into a core-only fixture before changing LLVM

lowering; never add RecallCoach-specific compiler behavior.

4. Continue package-owned Apple TV/Android TV host receipts and a real

mixed-device `hey_party` match.

5. Keep language/compiler changes application-driven and package-neutral.

Language and compiler changes must remove a demonstrated application-neutral

blocker and preserve the frozen source -> semantic HIR -> ABI MIR -> direct LLVM

architecture. Product workflows must first use stdlib and lock-bound packages.

Completed source implementation through v0.99.382a

  • v0.99.379a makes release products use the LLVM-hosted Hey frontend,

inventories every native source, retires public native-C application builds,

and adds fail-safe WIP-to-Release finalization.

  • v0.99.380a introduces one compiler-owned scalar compilation request and binds

compiler/script cache identities to it.

  • v0.99.381a adds `hey-repl --llvm-fragment`, proving complete fragments use the

canonical request, semantic HIR, ABI MIR, direct LLVM, and a compiler-free warm

cache hit. Persistent interactive state remains the next REPL slice.

  • Native desktop/mobile structural targets, target archive provenance, Windows

runtime evidence, source-handoff safety, learning/docs surfaces, reflection,

restricted eval, actors/workers/Jobs/streams, and the fixed-point HIR/MIR/direct

LLVM compiler remain intact from earlier slices.

  • v0.99.360a compiled the package-index application through the ordinary

LLVM-only source cache and expanded strict closure to 66/66.

  • v0.99.361a added ordinary recursive scalar calls while preserving proven

`tail.self`/LLVM `musttail`; strict closure is 67/67.

  • v0.99.362a added complete package graph resolution, deterministic

highest-compatible backtracking, lock v2, exact offline restore, complete

installed-tree verification, online audit, command registration, reusable

schema validation, and observable isolated source handoffs.

  • v0.99.363a rewrote all maintained 1BRC applications with mapped bytes,

ordinary immutable objects, functions, loops, and actor-owned partials;

removed the specialized mutable byte-keyed table surface from every compiler

and runtime lane; and closed generalized division, parsed-JSON, mixed-value

stringification, and nested-receiver facts through HIR, MIR, and direct LLVM.

  • v0.99.364a migrates all 58 public stdlib modules and first-party references

to canonical CamelCase, closes the direct-LLVM expression failures exposed by

the prior release receipt, adds bounded/guarded 1BRC tuning commands, and makes

promotion mandatory and commit-blocking for full/report handoffs. Complete

Release C/A/B/promotion acceptance is performed by the returned 12-core script.

  • v0.99.365a assigns deterministic BindingIds at semantic HIR, preserves them

through ABI MIR, consumes them in direct LLVM local-slot allocation, exposes

replayable phase artifacts, and makes verification cost tiers explicit.

  • v0.99.366a through v0.99.375a proved script startup, Ruby-like

synchronous iteration, television adapters, room/lobby transport, and five

game reducers. Corrected v0.99.376a extracts all television, party, and game

porcelain from Hey core into `hey_tv`, `hey_ios_tv`, `hey_android_tv`, and

`hey_party`; core retains generic LLVM/native/package/stdlib plumbing only.

  • v0.99.377a enforces unqualified builtin `get` arity at the shared semantic

boundary, preserves valid boxed query-row lowering, and adds replayable

source/function/canonical-IR provenance. Its r3 source handoff also validates

the complete canonical module ledger with exact entry names, rejects

case-folded collisions, and derives deletions only from a matching staged

inventory.

The r4 correction also makes process status independent of an inherited

auto-reaping `SIGCHLD` disposition, so complete child output cannot be paired

with a false failed result. The r5 transport candidate corrects the macOS-incompatible

`/bin/true` regression fixture with one generated child executable shared by Unix

and Windows process receipts, and moves active-slice gates ahead of the broad matrix. The r6 handoff makes malformed ZIP fixtures independent of host filesystem case semantics, eliminates the duplicate nested archive regression, preserves direct shell diagnostics, and includes source-handoff tests in the fast tier. The r7 handoff also repairs

the stale candidate-specific prompt-compaction assertion and makes failed verification phases

print their exact label and status.

Immediate alpha slices

1. **Persistent LLVM REPL session ABI.** Add stable binding/function identities,

transactional generations, and shared-object fragment loading.

2. **Persistent language state.** Add bindings, imports, and specified function

redefinition through the LLVM session rather than the C evaluator.

3. **Restricted Eval migration.** Route Eval through the same fragment compiler,

then remove the C expression evaluator.

4. **Hey-owned MIR-to-LLVM progression.** Move textual LLVM emission policy out

of the native bootstrap while preserving semantic HIR and ABI MIR.

5. **Stage-0 isolation and fixed point.** Remove remaining ordinary tooling uses

of native-C, publish host bootstrap artifacts, and prove Stage 1/Stage 2.

Television application boundary

TV application
  |-- native TV question/display screen
  |-- QR code containing the session URL
  |-- package-owned HTTP/WebSocket or polling service
  |-- phone web application
  |-- session/question/answer persistence
  `-- scoring and lobby policy

The ownership split is deliberate:

  • **Hey core:** target triples, object emission, runtime ABI, lifecycle callbacks,

SDK inspection, capability discovery, and honest structural/native receipts.

  • **Packages:** native TV UI/lifecycle adapters, QR rendering, web-service

composition, configuration, persistence, and UI contracts.

  • **Application repository:** questions, lobby behavior, scoring, visual design,

deployment, credentials, and release cadence.

The seven named application packages are plausible owners, but implementation

must inspect their actual release manifests and documentation before binding the

demo to `hey_app_ecosystem`, `hey_app`, `hey_project`, `hey_config`, `hey_cli`,

`hey_data`, or `hey_ui_contract`.

Package ownership

Hey core owns project declarations, complete graph resolution, registry-neutral

indexes, release/archive/manifest/content verification, immutable locks, exact

restore, complete installed-tree verification, online audit, safe

filesystem/archive/process APIs, generic templates and schema validation,

collision-checked package command registration, and target/capability discovery.

Independent package tooling owns authoring workflow, documentation policy,

release generation, website/Git publication, credentials, and registry hosting

policy. Applications own dependency choice and product behavior.

Release verification

Fast iteration uses:

HEY_BUILD_JOBS=12 HEY_COMPILER_CONTRACT_JOBS=12 ./bin/hey verify --focused

The maintainer continue-all pass uses:

HEY_BUILD_JOBS=12 HEY_COMPILER_CONTRACT_JOBS=12 HEY_RELEASE_JOBS=12 ./VERIFY-HEY-RELEASE.sh

Documentation publication and application vendoring remain explicit. Promotion

is enabled by default and must be the final gate. Every failure stays visible

and logged; the verifier attempts all scheduled gates and exits nonzero after

the aggregate summary.

Beta gate

Beta still requires multiple signed promoted desktop archives, stable package

contracts and migration policy, blocking performance budgets over ordinary Hey

code, reproducible CI/SBOM/license evidence, truthful mobile/television receipts,

and at least two maintained applications.

Performance and beta acceptance

The compiler/runtime campaign retains a **20x improvement objective** for

ordinary Hey workloads while preserving the fixed-point HIR/MIR/LLVM path.

Useful package/application vertical slices before beta must remain independently

versioned and must not become benchmark-shaped core intrinsics.

The television application remains package-owned and should expose a separate web target

for the phone/browser participant surface rather than adding TV application policy to core.
