Archive tutorial (`stdlib:archive`)

`stdlib:archive` owns deterministic ZIP packing and safe extraction. Package and source-release tools use the same implementation.

import 'stdlib:archive'
import 'stdlib:result'

Validate untrusted entry paths

if archive.safe_entry_path('src/main.hey')
  says 'safe'
end

archive.require_safe_entry_path('docs/guide.md')

Unsafe entries include empty names, absolute paths, `.` or `..` segments, duplicate separators, backslashes, and Windows drive prefixes.

Create a deterministic ZIP

let packed = archive.pack(
  'package-root',
  ['README.md', 'src/main.hey'],
  'dist/package.zip',
)

if result.is_error(packed)
  fail result.message(packed)
end

Entries are sorted before writing. ZIP timestamps, modes, ownership-facing metadata, and compression choice are normalized so identical bytes produce identical archive SHA-256 values.

Extract safely

let extracted = archive.extract('dist/package.zip', 'build/unpacked')
if result.is_error(extracted)
  fail result.code(extracted) + ': ' + result.message(extracted)
end

Extraction validates the central directory before publication. It rejects traversal, duplicate names, symlink entries, encrypted entries, unsupported compression methods, malformed bounds, and oversized entries. Callers should still extract into a staging directory and atomically publish only after manifest/content verification; `hey package install` performs that complete sequence.

Executable modes

Deterministic ZIP entries preserve the one portable permission distinction needed by source and distribution handoffs: regular files are normalized to `0644`, while executable regular files are normalized to `0755`. Extraction restores that distinction. Symlinks and non-regular source entries are rejected.

This means a freshly extracted Hey source archive can run `bin/hey`, `bin/heyc`, subcommands, and executable regression scripts without a repair-time `chmod` sweep.
