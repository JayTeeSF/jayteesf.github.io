Actors and Mailboxes

Actors are runtime objects implemented in C with pthreads and FIFO mailboxes.

Long-lived actor

actor echo
  receive msg
  says msg
end

program
  let p = spawn echo
  send p, 'ping'
  stop p
end

One-shot actor

actor once
  receive msg
  says msg
  exit
end

program
  let p = spawn once
  send p, 'ping'
  join p
end

Actor state

actor counter(label)
  state count = 0
  receive msg

  if msg == 'inc'
    set count = count + 1
    says label + ': ' + count
  else
    if msg == 'stop'
      exit
    end
  end
end

Process ring

The process ring is now real Hey source:

./bin/hey examples/process_ring.hey --test
./bin/hey examples/process_ring.hey --run

Read examples/process_ring.hey for the full code.

Join vs stop

Use `join actor` when the actor is expected to exit by itself.

Use `stop actor` when the actor is blocked waiting for another message.

Current semantics

• `spawn` starts a pthread-backed actor.

• `send` clones a value into a FIFO mailbox.

• `receive` blocks inside an actor.

• `state` persists across messages.

• `set` updates an existing binding.

• `exit` leaves the current actor.

• `join` waits for an actor that exits by itself.

• `stop` sends an internal stop message and joins.

• mailboxes are FIFO and currently unbounded.
