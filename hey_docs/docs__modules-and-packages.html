Modules and packages

This is the canonical import path contract for Hey programs.

Application authors should not need to know where the bundled standard library lives on disk. Use an import scheme that describes the dependency kind, and let the compiler resolve the path.

Stdlib imports

Use `stdlib:` for bundled Hey libraries:

import 'stdlib:web'
import 'stdlib:json'
import 'stdlib:jobs'
import 'stdlib:collections'

`stdlib:web` resolves to `web.hey` under the configured stdlib root. The default root is `stdlib/` in a source checkout. Distribution and tests may override it with `HEY_STDLIB_PATH`.

`std:` is accepted as a short alias:

import 'std:web'

The older checkout-relative form still works for compatibility:

import 'stdlib/web.hey'

New documentation and examples should prefer `stdlib:name`.

Local modules

Use explicit relative paths for code that belongs to the current app or repo:

import './web/router.hey'
import '../shared/formatting.hey'

Relative imports are resolved from the file that contains the import. Imported modules keep their module namespace.

Third-party packages

Use `pkg:` for installed package modules:

import 'pkg:brainscape/cards@1.0.0/main'

That resolves to:

${HEY_PACKAGE_PATH:-.hey/packages}/brainscape/cards/1.0.0/main.hey

The package name and version are explicit so builds stay reproducible and compatible with lockfile-driven installs.

Safety rules

Stdlib and package import segments reject empty names, absolute paths, `..`, and backslashes. Local imports keep normal relative file-path behavior for app source trees.

Current non-goals

Imports do not yet provide aliases, `from ... import ...`, wildcard imports, or re-export syntax. Package alias helpers in `stdlib:packages` are manifest metadata, not language-level import aliases.

Private imports

Use `private import` for implementation dependencies that should not leak through a module to its importers:

private import './internal/slug_helpers.hey'

The current file can call the imported module, but files that import the current file cannot reach that private dependency as a transitive API. See `docs/visibility-and-private-imports.md` for the full visibility contract.
