JSON Codecs

JSON is standard-library porcelain over Hey-lang core values.

Hey-lang owns immutable core values:

map/record
array
string
bytes
number
bool
nil

Object and array literals are core map/array values:

let user = {name: 'Ada', visits: 1}
let tags = ['json', 'codec']

JSON encoding and decoding belongs to the standard library:

import '../stdlib/json.hey'

let user = json.decode("{\"name\":\"Ada\"}")
let text = json.encode(user)

Current compatibility bridge

The starter facade currently delegates to compatibility runtime helpers:

module json
  fn decode(text)
    return json(text)
  end

  fn encode(value)
    return to_json(value)
  end
end

Existing `json(...)`, `to_json(...)`, `json_text(...)`, and internal `hey_json_*` C names remain compatibility/runtime bridges while the standard-library boundary is established.

Immutability rule

JSON codecs must not introduce mutable shared objects. Decoding creates core immutable values. Transformations such as `put`, `delete`, `merge`, and `push` return new values.

Future work

  • Real package-style import: `import json`.
  • JSONL codecs: `jsonlines.encode`, `jsonlines.decode`, streaming line codecs.
  • Parser options as ordinary library values.
  • Alternative third-party JSON packages without new syntax.
