Current implementation queue after v0.99.245a

Completed foundation

  • [x] One public compiler path: LLVM-hosted Hey -> semantic HIR -> ABI MIR -> direct LLVM.
  • [x] Million-scale resident actors on a bounded managed scheduler.
  • [x] Explicit concurrency model: immutable values, actors, workers, Jobs, streams; no public channels.
  • [x] Stateless worker declarations through REPL, HIR, MIR, direct LLVM, and both Hey compiler hosts.
  • [x] Fixed-size worker pools with bounded queues, explicit backpressure, lifecycle, and queue-wait metrics.
  • [x] Named worker-backed Jobs with direct worker identity, receipts, results, metrics, and clean stop/shutdown.
  • [x] SDK/run-bundle separation and SDK-local LLVM discovery.
  • [x] Archive-derived deletion commands; no persistent stale rm list.

v0.99.246a — Jobs reliability

  • [ ] Add explicit success/failure acknowledgements.
  • [ ] Add retry limits and bounded backoff policies.
  • [ ] Add enforceable execution timeouts, not elapsed-time-after-completion checks.
  • [ ] Add cancellation and dead-letter storage.
  • [ ] Define and test at-least-once delivery and idempotency guidance.
  • [ ] Prune or persist completed result records instead of retaining every result until shutdown.
  • [ ] Add deterministic failure/retry/test helpers.

v0.99.247a — adaptive scaling

  • [ ] Minimum/maximum workers per Job.
  • [ ] Queue-depth and queue-latency scaling inputs.
  • [ ] Target-latency scale-up and bounded idle scale-down.
  • [ ] Worker creation/shutdown rate limits.
  • [ ] Replaceable scaling policies with safe defaults.

v0.99.248a-v0.99.250a — streams and ordering

  • [ ] Worker-backed `map`, `filter`, `each`, and `.through(Job)`.
  • [ ] Bounded stage queues and automatic backpressure.
  • [ ] Explicit `workers:` options with safe sequential defaults.
  • [ ] Internal sequence tags and `.resequence`.
  • [ ] Ordered/unordered modes and bounded reorder buffers.
  • [ ] Optional keyed `partition_by` with serial per-key execution.

v0.99.251a — syntax and documentation stabilization

  • [ ] Add `job Name ... end` as thin syntax over proven Jobs runtime semantics.
  • [ ] Finish actor-versus-worker-versus-Job guidance and migration cleanup.
  • [ ] Stress REPL, compiler hosts, distributions, and shutdown behavior.

v0.99.252a-v0.99.254a — scheduler performance

  • [ ] Profile registry, actor mailbox, worker mailbox, allocation, reference counting, message pools, and boxed values separately.
  • [ ] Add worker-local message caches and partitioned registries only when measured.
  • [ ] Add move-send and scalar fast paths using compiler-internal representation facts without changing source syntax.
  • [ ] Implement measured global-injection/local-queue/randomized-half-stealing/running-spinning-parked scheduler generation 2.
  • [ ] Prove million-actor residence and multi-core actor/worker/Job throughput across `1..cores`.

v0.99.255a — reactor and shutdown closure

  • [ ] Persistent kqueue/epoll registrations.
  • [ ] Thousands of waits, cancellations, closes, and reuse races.
  • [ ] Sanitizer-proven actor/worker/Job/supervisor shutdown ownership.

v0.99.256a-v0.99.257a — Unix SDK closure

  • [ ] Remove remaining native MIR/object/link ownership.
  • [ ] Package LLVM resources, linker metadata, sysroots, and startup objects.
  • [ ] Make native libraries relocatable.
  • [ ] Prove clean Linux-x64 and macOS-arm64 SDK/run bundles.

v0.99.258a-v0.99.263a — Windows x64

  • [ ] ABI/PE-COFF/bootstrap/console.
  • [ ] Threads/files/mappings.
  • [ ] Processes/pipes/dynamic libraries.
  • [ ] Winsock and scalable readiness.
  • [ ] TLS/HTTP/downloads.
  • [ ] SDK/run packaging and promotion.

Continuing performance

  • [ ] Keep 1BRC and concurrency benchmarks ordinary Hey with no challenge-specific C/LLVM shortcuts.
  • [ ] Run target-Mac one-billion-row C/Python/Hey evidence.
