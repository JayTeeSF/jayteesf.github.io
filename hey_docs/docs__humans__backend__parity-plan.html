Human maintainer documentationAudience-specific Hey documentation

One product compiler, two independent Hey hosts

Current policy (v0.99.239a)

The historical three-backend parity program succeeded in establishing the north-star product compiler. Public application compilation now has one route:

selfhost-llvm -> semantic HIR -> ABI MIR -> direct LLVM

`selfhost-c -> the same HIR/MIR -> direct LLVM` remains the independent bootstrap cross-check, and native-C remains the Stage-0 behavioral oracle/recovery seed. Old direct-source LLVM and typed-IR-C rows are private regression evidence, not user-selectable product alternatives.

The remaining parity obligation is stronger and simpler: both Hey hosts must preserve byte-equivalent semantic HIR/ABI MIR and the one direct LLVM consumer must pass outside-in application behavior. The historical execution plan below is retained as a record of how that convergence was achieved.

What "fully compliant" means

The normative language surface is:

1. `specs/language/*.hey`: implementation-independent language contracts. These are the language definition.

2. `specs/repl/*.in` and `*.out`: normative REPL session contracts.

3. `examples/*.hey`: conformance programs that also teach readable Hey syntax.

4. `tools/`, `stdlib/`, and `benchmarks/hey`: must build and run through the whole-repository backend gate; failures must be classified explicitly (`unsupported`, `build_failed`, `run_failed`, `output_mismatch`) and treated as named work items, never silently skipped.

Exit criteria per backend:

  • A backend is **fully compliant** when `./bin/test --backend-world-full` reports for it: `unsupported=0`, `build_failed=0`, `output_mismatch=0`, and `run_failed` only for rows in a named intentional-negative/expected-error ledger.
  • `self-hosted-c` additionally requires the parser/build/spec/REPL parity gates and the compiler-decision gate before promotion.
  • `llvm` additionally requires the no-toolchain distribution gates before it can become a user-facing default.
  • The REPL is a first-class conformance target: `make repl-specs` and `make repl-backend-matrix` remain release gates for every slice, and the REPL must migrate onto the same canonical symbol table as compiled builds.

Why the current path plateaus

Hey currently maintains four separate frontends: the `native-c` line-oriented emitter in `runtime/heyc.c`, the `llvm` line-oriented emitter in the same file, the typed-IR emitter in the same file, and the self-hosted parser/emitter in `stdlib/hey_parser.hey` + `stdlib/hey_c.hey`. Every language construct must be hand-taught to each of them, so parity cost grows as features × frontends. The `hey-ir` handoff exists (`heyc ir`, `--emit-ir`) but no backend consumes it as its real input yet. Growing `llvm` from 59/301 by per-line pattern matching is not maintainable for decades; convergence must be structural.

Tracks

Track R: runtime soundness (production blocker; leads)

Status: **core landed in v0.99.4a.**

1. Interpreter shared-state races: fixed. `global_env` access is mutex-guarded with clone-under-lock accessors (`env_lookup_clone`, `env_has`) so no `ReplVar` pointer into global storage escapes the lock; `repl_line_no` is atomic; `repl_return_value`/`repl_has_return` were already thread-local.

2. Teardown ordering: fixed. `hey_runtime_drain_actors()` stops and joins every actor thread before `heyc`/REPL paths free actor definitions and the global environment.

3. `make sanitize-smoke` gate: landed and green. TSan is strict (any warning fails); ASan runs with `detect_leaks=0` until the ledgered shutdown-leak audit slice lands.

4. Harness portability: `bin/subcmds/hey-backend-world` now falls back to running without `/usr/bin/time` on minimal containers.

5. Remaining ledgered items: shutdown actor-struct leak audit; interactive-REPL redefinition-while-running synchronization; making linux-x64 (docker) a recurring per-slice platform signal on the maintainer machine.

Exit evidence: jobs-pool specs pass 45/45 repeated runs on 1-CPU linux-x64; full suite 285 files failures=0 there; TSan zero warnings across the jobs/actor/supervisor/ring cluster.

Track A: contracts first (continuous)

Every construct that any backend lowers gets a tiny implementation-independent spec **before** the lowering lands, per the micro-spec direction. v0.99.3a adds `specs/language/623`–`628` for the data-model rung. The payoff is already demonstrated: the six specs isolated two previously unnamed self-hosted routing gaps (`at`, `json_text`) in one run while confirming the rest of the data-model surface routes correctly.

Track B: one canonical symbol/import model

Implement the function/module design contract: the parser manifest exposes canonical `Module__fn` symbols, and `native-c`, `self-hosted-c`, `llvm`, and the REPL all consume the same table. Exit: imported stdlib module calls lower identically across backends and the REPL, and the tooling-facade backend-world rows stop failing on symbol shape.

Track C: the typed IR becomes the real backend input

Grow `hey-ir` 0.1 beyond `say`/scalar into the shared contract the typed IR/ABI doc already promises:

1. Spec the IR ops as goldens first: data-model ops (array/map literals, `get`/`put`/`at`/`len`/`has`, spread, JSON codecs), control flow, functions/modules with canonical symbols, then actors/supervision/jobs/evented-I/O as runtime ABI calls.

2. Make the `llvm` backend consume IR instead of re-scanning source lines. `native-c` may migrate later; do not destabilize the active/default backend to do it.

3. Make the self-hosted compiler emit the same IR.

Consequence: one LLVM lowering serves both compilers. The C-implemented `heyc` and the Hey-implemented self-hosted compiler produce the same `.hir`, so "the LLVM backend for the C and self-hosted versions of the language" is one artifact, not two parallel efforts. `native-c` remains the behavioral oracle throughout.

Track D: LLVM general-lowering ladder (spec-clustered)

  • Rung 1, data model: **started in v0.99.6a** with boxed `hey_llvm_*` shims (constant-literal interpreter folding, get/at/index, len, has, put, spread updates, to_json/json_text, boxed let/set/says). Remaining rung work: variable-carrying literals, boxed locals in functions, arrays of mixed construction, keys/values/entries facades, and Track C lifetime handling for boxed temporaries.
  • Rung 2, text/file facades: builds on `620` (string file I/O) and `622` (slice/compare); add `text.*` module facades.
  • Rung 3, modules/imports on canonical symbols: specs `612`, `613`, `618`.
  • Rung 4, actors minimal: spawn/send/receive/join echo contracts as tiny specs before any larger actor fixture.
  • Rung 5, jobs porcelain shapes.
  • Rung 6, evented-I/O descriptors.

Rules: no benchmark-specific shortcut lowering, ever. `one_brc_*` helpers and `c10k_ping` run under `llvm` only when general lowering can express them. Every rung lands with parity fixtures green under `native-c` + `llvm` and honest `unsupported` elsewhere.

Track E: promotion and no-toolchain distribution

Backend-default decisions use whole-repository evidence: the compiler-decision gate, `backend-world-full`, and the recurring 1BRC/C10K/ring challenge signals. The open no-toolchain packaging questions stay in the LLVM plan and open questions; decide shell-out vs linked LLVM libraries before Rung 4 lands so actor-heavy lowering targets the packaging reality.

Milestones

  • **M1**: Track R exit, plus self-hosted routing fixes for `at` and `json_text`. Measurable: jobs-pool specs green on linux-x64; self-hosted gains the two new-spec rows.
  • **M2**: canonical symbol table consumed by all backends and the REPL. Measurable: module/tooling backend-world clusters move; no symbol-shape build failures remain.
  • **M3**: IR ops spec'd with goldens; `llvm` consumes IR for Rungs 1–2. Measurable: `llvm` good ≥ 150 in backend-world-full.
  • **M4**: self-hosted emits the same IR; parser/build/spec parity gates pass. Measurable: `llvm` good equals `native-c` good minus ledgered rows; self-hosted `build_failed=0`.
  • **M5**: REPL on the canonical table with `repl-backend-matrix` across backends; no-toolchain dist gates pass. Beta review begins.

Evidence and honesty rules

  • Every backend-parity slice reports the parallel `backend-world-full` command, with summary lines, problem_row lines, and slow-row output pasted back so good/total, mismatch, and source-path deltas are comparable.
  • New specs change denominators; say so explicitly when comparing runs.
  • Unsupported constructs must fail loudly. No hidden native-oracle fallback. No output shortcuts.
  • Keep an intentional-negative/expected-error ledger, starting with the M3 `native-c` `run_failed=8` class.

Alternatives considered (re-evaluable)

  • REPL-parity-first for self-hosted: deferred; the REPL rides the Track B symbol table instead of leading.
  • Emitter-only parity without a shared IR: rejected as features × backends whack-a-mole.
  • Adding sanitizer gates before fixing the races: rejected; gates must start green.
