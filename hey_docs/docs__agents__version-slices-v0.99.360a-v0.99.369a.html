Agent handoff documentationAudience-specific Hey documentation

Alpha version slices from v0.99.360a through v0.99.369a

This ledger is the durable implementation order for the compiler, package,

standard-library, and television-target work. Every slice must be independently

buildable, verified, documented, archived, and applicable by the source-handoff

processor. Experimental work from a later slice must not be presented as part of

an earlier completed archive.

v0.99.360a — package-index strict-LLVM closure

**Status: complete.** The namespaced package-index fixture publishes five exact

links atomically through an LLVM cache miss and hit. Semantic HIR and ABI MIR

retain string versus boxed facts, program exit reaches the `i32` cleanup

epilogue, and the bounded strict inventory passes 66/66.

v0.99.361a — ordinary recursive expression calls

**Status: complete.** Recursive return inference is seeded from non-recursive

base returns. Top-level arithmetic owns its expression shape, and MIR marks a

self call `tail.self` only when the matching call parenthesis completes the

entire return expression. Naive Fibonacci returns `9227465` for input 35 through

two ordinary LLVM calls while the million-step accumulator retains `musttail`.

v0.99.362a — complete Hey-owned package consumption

**Status: complete.** A clean maintained focused verification passed, including

the deterministic 1,938-file/278-executable source archive, safe application,

and isolated apply/verify/commit/fast-forward handoff regression. Release C/A/B

remain separate maintainer-host evidence and are not claimed by this slice.

**Milestone.** Replace application-local dependency wrappers with one generic,

registry-neutral project transaction:

hey package ensure --project DIR --update-lock
hey package ensure --project DIR --locked
hey package verify --project DIR
hey package audit --project DIR

**Implemented contract.**

  • resolve the complete transitive graph from a machine-readable registry index;
  • use deterministic highest-compatible selection with backtracking rather than

a greedy first answer;

  • reject cycles, impossible intersections, command collisions, unsafe paths,

and partial publication;

  • verify release JSON, archive, manifest, canonical content, and every installed

file hash;

  • reject same-name/same-version content replacement against the previous lock;
  • publish the complete package tree and lock atomically;
  • restore `--locked` directly from exact lock-bound archive sources and hashes

without consulting mutable registry metadata;

  • keep `verify` entirely offline and make `audit` recheck registry, release,

archive, graph, and same-version immutability evidence;

  • bind collision-checked package CLI extensions into the immutable lock and

reverify the installed tree before dispatch;

  • expose reusable immutable `stdlib:Schema` descriptors and structured error

paths rather than embedding package-specific JSON checks;

  • retain safe filesystem, structured argv process execution, generic EHY/text

templates, and target/capability discovery as general core/stdlib facilities.

Package authoring policy, release creation, documentation requirements, website

publication, and Git publication remain owned by independent tooling such as

`hey_packager`.

**Blocking evidence.** Local and loopback-HTTP graph transactions, parent-version

backtracking, conflict/cycle/no-partial-publish failures, immutable locked

restore, online audit, same-version replacement rejection, command registration,

schema native-C/direct-LLVM parity, complete-tree tamper rejection, source

handoff isolation, and source-archive application must all pass.

v0.99.363a — ordinary-Hey 1BRC and generalized LLVM closure

**Status: complete.** All four maintained applications compile and run through

the LLVM-hosted frontend, semantic HIR, ABI MIR, and direct LLVM with exact

default output. The specialized mutable byte-keyed table module and its public

builtins, interpreter/REPL dispatch, native-C lowering, runtime kind, LLVM ABI,

documentation, fixtures, and product tests are removed from maintained source.

**Implemented contract.** The applications now use mapped immutable bytes,

bounded delimiter scans, zero-copy slices, ordinary immutable object records,

reusable Hey merge functions, actor-owned partial values, and general byte-buffer

rendering. Semantic HIR classifies integer division as `Int`; parsed JSON is a

boxed `Value`; mixed integer/boolean/boxed concatenation uses the general boxed

LLVM emitter; and nested chains such as `entry[1].type` bind the property to the

actual receiver. No filename, benchmark, or program-shape shortcut was added.

**Blocking evidence.** Rewritten language contracts, two independent compiler

hosts, identical HIR/MIR, native-C parity, direct-MIR LLVM execution, ordinary

statistics product lanes, a repository-wide retired-surface audit, and exact

single/case/scale/actor application output are maintained gates. Release C/A/B

and promotion are separate host-intensive evidence and must be run before a full handoff commits.

v0.99.364a — canonical CamelCase module surface

**Status: implementation complete; 12-core release/promotion receipt pending.**

All 58 public standard-library files and declarations use canonical CamelCase.

First-party `.hey` source, fenced Hey documentation/templates, package fixtures,

generated source, compiler dispatch/inference strings, examples, specs, and

source inventories have been migrated without lowercase aliases.

**Implemented contract.** `docs/migrations/v0.99.364a-module-renames.tsv` is the

58-row filename/module/prefix ledger and

`v0.99.364a-qualified-renames.tsv` records 809 known calls. The executable audit

uses lexical bindings to avoid misclassifying fields such as `result.ok` and

rejects retired imports, stdlib paths, qualifiers, declarations, and compiler

symbol strings. Direct LLVM now also handles the boxed string parameter,

implicit return, bare scalar final expression, and parenthesized arithmetic

cases exposed by the failed v0.99.363a host receipt.

**Performance and handoff contract.** `1brc sweep` tests C-reference,

`hey-single`, and `hey-actors` worker/chunk variants on bounded row ladders,

verifies hashes, and writes conservative projections. `1brc full` validates the

winner, machine, input sample/size, binary hash, five-minute bound, and optional

five-second requirement before launching. Full/report handoffs require final

promotion and aggregate failures; no failed verification, promotion, or source

integrity check may commit or fast-forward.

**Blocking evidence.** The CamelCase audit, LLVM expression regression, guarded

1BRC CLI regression, clean fixed-point build, four-application LLVM product lane,

current-version verifier, Release C/A/B, final promotion report, and source

archive/apply regression are maintained gates. The returned 12-core processor is

the authoritative final acceptance receipt.

v0.99.365a — nanosecond-coherent Time and Duration

**Milestone.** Give `Time` and `Duration` one explicit nanosecond-based model.

**Required implementation.** Store durations canonically in nanoseconds and

provide explicit constructors/converters (`nanoseconds`, `milliseconds`,

`seconds`, `to_nanoseconds`, `to_milliseconds`, and so on). Align elapsed time,

timeouts, scheduler delays, process waits, streams, timers, native-C, HIR/MIR,

and direct LLVM. Remove helpers whose names hide a unit.

v0.99.366a — independent `hey_*` package migration receipts

**Milestone.** Apply package graph, CamelCase, and time contracts to every

maintained independent package actually supplied to the working environment.

**Required implementation.** Resolve/package/verify each repository with its own

checks, commits, tags, and release cadence. Record exact repository and version

receipts. Repositories or release archives not present are explicit external

blockers and must never be claimed complete. The first integration set includes

`hey_app_ecosystem`, `hey_app`, `hey_project`, `hey_config`, `hey_cli`,

`hey_data`, and `hey_ui_contract` when their source releases are available.

v0.99.367a — native tvOS target and party-quiz vertical slice

**Milestone.** Add a truthful native tvOS structural/application path and prove

it with a small package-owned living-room quiz rather than target-specific

language syntax.

**Required implementation.** Reuse the iOS HIR/MIR/direct-LLVM object path with

tvOS SDK, target triples, linking, entitlements, signing, simulator/device

receipts, focus/input lifecycle, and package-owned UI adapters. Build a small

Jackbox-like sample: the TV presents questions and a QR code; phones open a

web application; a simple Hey web server owns room/session state and answer

submission. Core owns only target ABI/callback plumbing, QR/image primitives if

genuinely general, HTTP/network capabilities, and small target porcelain. The

seven app packages are adapters/dependencies, not language subsystems.

**Blocking evidence.** Structural tvOS object/application generation, simulator

launch where available, explicit physical-device/signing blockers, QR payload

roundtrip, multi-phone loopback browser participation, deterministic scoring,

and package graph/lock verification all have machine-readable receipts.

v0.99.368a — Google TV / Android TV application profile

**Milestone.** Add the television application profile over the existing Android

NDK/JNI path, targeting Google TV/Android TV rather than inventing a

Chromecast-only language target.

**Required implementation.** Add Android TV manifest/launcher intent, lean-back

or Compose/package-owned UI host, remote/focus behavior, emulator/device

receipts, and reuse the same quiz server/protocol. A Cast sender/receiver may be

an independent package/application later; it is not a substitute for Android TV

native evidence.

v0.99.369a — target receipts and safe verification parallelism

**Milestone.** Make slow release verification observable, resumable where safe,

and parallel only where output/cache isolation is proven.

**Required implementation.** Retain loud continue-all verification receipts,

per-gate logs, timing, and final failure aggregation. Use bounded jobs for

compiler contracts and Release C/A/B; add isolation before parallelizing specs

that share build/cache paths. Record tvOS and Android TV structural/native/signing

truth without upgrading structural compilation into a native receipt. Keep

Linux vendoring timed and bounded by container-visible CPU capacity.
