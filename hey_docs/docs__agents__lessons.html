Agent handoff documentationAudience-specific Hey documentation

Television and party package boundaries

  • A successful product prototype is not evidence that its generator belongs in

the language repository. Core may own target-neutral IR, ABI, callback, Web,

Jobs, and package-command plumbing; UIKit/LEANBACK adapters, QR rendering,

rooms, lobbies, cards, dictionaries, and scoring remain packages/applications.

  • Built-in top-level commands are scarce porcelain. Reusable product commands

should be declared by immutable packages and dispatched only after complete

lock-bound tree verification.

  • Generated-project tests must assert semantic receipts, not brittle adjacency

in a long summary line. Version assertions must track the generator's actual

immutable package constraints.

  • A forwarding command must preserve the platform selector it consumes. Better,

avoid a core forwarding layer when package commands already provide the

correct ownership and update cadence.

Canonical module migrations and fail-closed handoffs

  • A generated-prompt regression must validate stable aggregate commands, not a

replacement-candidate-specific subtest that changes with the next repair. Use

named fixed-string assertions so `set -e` cannot turn a stale expectation into an unexplained status 1.

  • A phase runner should print the failed phase and exit status before returning;

otherwise a silent assertion makes a broad receipt look like an infrastructure

failure instead of a stale contract. A policy regression must also prove its

negative condition; a grep wrapped in a no-op `if` documents nothing and cannot

prevent a forbidden command from escaping its guard.

  • A case-collision regression must not create both spellings in a normal directory:

the default macOS filesystem cannot represent them. Build the valid ZIP first,

then append the alias as a separate central-directory entry from an isolated

staging tree. Always validate the malformed fixture inventory before asserting

installer behavior. Run the behavioral shell regression once with direct

diagnostics, then use a separate fast language contract for source-shape and

gate-registration assertions; do not nest and duplicate the expensive regression.

  • A fast current-slice tier must include every gate for files changed by the slice.

Moving a test earlier is not enough if the fast tier exits before reaching it.

  • A malformed-archive regression should not compile the entire runtime merely to

invoke an existing Hey-owned checker. Use the maintained interpreter/tooling

path for negative fixtures, and reserve native checker builds for contracts that

specifically require native emission evidence.

  • Child-process status is part of the runtime contract, not merely captured

output. A launcher that inherits `SIGCHLD=SIG_IGN` or `SA_NOCLDWAIT` can let a

command print its complete success receipt while the parent receives `ECHILD`

and reports `ok=false`. Normalize inherited auto-reap state before spawning,

preserve explicit handlers, and never translate an unknown live-child status

into exit zero.

  • Wall-clock acquisition and UTC calendar formatting are runtime services, not shell scripting. Keep monotonic duration measurement separate, and expose deterministic epoch-based formatting so every compiler lane can prove parity without depending on host `date` or Ruby.
  • A compiled-script cache key must cover the absolute entry path and transitive dependency hashes. Entry-file bytes alone can alias separate scripts and cannot detect a changed imported module. Parser caches may overwrite validated records; content-addressed executable caches retain older generations until explicit cleanup.
  • Correct content identity does not require reparsing and rehashing an unchanged graph on every launch. Keep a path-keyed metadata index in the OS user-cache directory, validate exact size/mode/second/subsecond metadata, and fall back to authoritative parser hashes whenever any dependency or compiler input changes. Never make project-local `.hey/` directories the default for disposable language caches.
  • Public launchers installed through symlinks must resolve the link target before deriving the SDK root. Using `dirname "$0"` directly makes `~/bin/hey -> SDK/bin/hey` search for `~/bin/heyc` and breaks the standard shebang path.
  • A source installer must run before the language exists, so POSIX shell is legitimate bootstrap plumbing. It should detect the host package manager, require explicit authorization before changing OS packages, build the fixed-point compiler, publish stable commands, and generate one reviewable environment file rather than asking users to hand-edit several variables.
  • A filename-only module rename is incomplete. Clean bootstrap exercises source

imports, generated source, compiler inference/dispatch strings, package

fixtures, and HIR/MIR consumers that an incremental build can hide.

  • Qualified-name migration must distinguish module receivers from lexical values;

blind replacement of `result.ok`-shaped tokens corrupts ordinary record code.

Keep machine-readable module and known-symbol ledgers and audit lexical scopes.

  • A verification cache must not live beneath a directory that the same gate

intentionally deletes. Put disposable handoff caches in the surrounding

workspace, and make compiler cache-directory creation recursive so a fresh

nested override is self-healing rather than order-dependent. A compiled

runtime cache key must hash every directly included implementation file, not

only the top-level C source and public headers.

  • Atomic rename is not enough for a shared compiled-object cache: concurrent

misses still duplicate expensive compilation. Serialize publication with a

process-owner lock, recover locks whose owner no longer exists, and prove one

compile plus residue-free publication under concurrent callers.

  • A relocated compiler helper cannot infer its SDK from its cached executable

path. Prefer an executable-adjacent distribution, then accept an explicit

`HEY_ROOT` only when it has the complete packaged layout; this preserves

checkout isolation while keeping contained SDK toolchains relocatable.

  • Backend matrices must query the public selectable-backend surface rather than

preserving retired aliases in test code. Transcript specs must also exercise

the lexical value actually returned (`result.value`), not a similarly named

module constant (`Result.value`).

  • Sanitizer instrumentation and optimization policy are independent. Make TSAN

and ASAN binaries incremental targets, permit separate compiler selection,

and avoid spending release time optimizing a binary whose purpose is dynamic

correctness instrumentation.

  • A continue-all shell function must aggregate every phase status explicitly.

Returning the last command can convert failed Release C/A/B gates into a

successful handoff. Promotion must run last, contribute to the aggregate, and

remain commit-blocking.

  • Performance projection and performance qualification are different facts. A

five-minute safety bound may authorize a full experiment, while a five-second

product target must be a separate receipt gate that can refuse the run.

  • Once lexical names are rewritten to internal identities, declaration-allocation

checks must use that exact resolved identity. Falling back to the source spelling

can make an outer local suppress the inner alloca while later instructions still

reference the scoped slot, yielding undefined LLVM values far from the feature

that exposed the defect.

Complete package graphs and trustworthy source handoffs

  • Dependency resolution is a graph search, not a sorted loop. Highest-compatible selection must snapshot state and backtrack when a later root or transitive requirement invalidates an earlier choice; abandoned staged packages must disappear before publication.
  • `ensure --locked` and `verify` should be registry-independent. Reproducible restoration uses exact archive sources and hashes from the lock, while `audit` is the explicitly online/local operation that rechecks registry entries, release metadata, archive bytes, and same-version immutability.
  • Same-name/same-version replacement is a supply-chain violation even when the new archive is internally self-consistent. Compare lock-bound release/archive/manifest/content identity before allowing an update.
  • Publish a complete package tree and lock as one transaction. Conflict, cycle, checksum, command-collision, or extraction failure must leave neither a partial package tree nor a changed lock.
  • A package CLI extension is executable supply-chain state. Bind its package, version, declared source path, and collision checks into the lock, then verify the entire installed tree before each dispatch.
  • Generic validation belongs in immutable data descriptors with structured paths. Package manifests and registry indexes can consume `stdlib:Schema`; they should not cause a second package-only validation language.
  • Included C implementation files must be explicit build dependencies. Otherwise an incremental `make` can silently link stale package code and produce misleading green regressions.
  • HIR/MIR type facts and a private LLVM consumer classifier can disagree. The immediate fix must remain generalized, but the long-term direction is one authoritative ABI fact flow rather than parallel return-shape inference tables.
  • Source handoffs should verify in a disposable worktree, stream output and heartbeat receipts, commit the verified candidate there, then fast-forward only an unchanged clean maintainer checkout. A release verification command should continue after failures and finish with a loud aggregate nonzero result so one slow gate does not hide later defects.
  • Package manifest module keys are public API; archive paths are private implementation details and must be resolved before checksum verification.
  • A package registry path, documentation policy, and Git commit workflow are author/hosting policy, not standard-library package plumbing; keep them in an independently versioned packager package.
  • Generated package projects should contain complete docs and self-documenting controls from the first commit, rather than relying on copied examples or chat instructions.

Ordinary values after benchmark-surface retirement

  • A benchmark-only mutable collection can hide several unrelated compiler defects. Rewrite the workload with ordinary values first, then classify each exposed gap independently instead of preserving the special API.
  • A function returning `String` does not imply that every parameter has the raw-string ABI. Infer each parameter from its uses and route mixed integer, boolean, and boxed values through the general stringification emitter.
  • Receiver syntax owns the callable operation. Once `entry[1].type` is parsed as a receiver chain, later native-C or LLVM emission must not let an unrelated lexical callable or local named `type` capture it.
  • Keep performance ideas, not benchmark vocabulary: mapped immutable bytes, bounded scans, zero-copy slices, compact buffers, partitioning, and actor ownership are reusable; a public shared-mutable-looking byte table is not.
  • A cache hit is not fast merely because it avoids compilation. If the generic launcher boots a full language runtime before reading an already-published key, the launcher can cost as much as the application. Keep miss policy in Hey, but let a narrow native dispatcher validate precomputed stamps and `exec` the cached artifact directly.
  • A native dispatcher must classify an existing source path as a program whether or not it has a shebang. Before any fallback `exec`, export the SDK root and stdlib derived beside the dispatcher; otherwise a distribution launched from a temporary directory can mistake that directory for the SDK.
  • Environment inspection is bootstrap infrastructure. `Env.cwd`, `Env.executable_path`, `Env.os`, `Env.arch`, `Env.get`, and `Env.default` must remain in-process primitive facades; recursively compiling a delegation runner to discover the compiler root is circular.
  • Hidden HIR/MIR identity records are metadata, not source statements. Consume actor parameter BindingIds before source-level `state`/`receive` validation or a correct MIR artifact will be rejected as an invalid actor body.
  • A private callable is a capability once its owning scope has resolved it. Higher-order stdlib code must not re-run private-name visibility checks from the library frame; doing so rejects legitimate callbacks during source execution. At the HIR/MIR boundary, callable values must store the resolved canonical symbol identity used by the closed-world LLVM dispatcher, not a pre-canonical dotted spelling.
  • A boxed local slot that will release its previous value on `set` must own its current pointer. Initializing it directly from another boxed local is a borrow, not a move; clone the value before storing it or reassignment will consume the original owner and later calls can double-free the same box.
  • Interpolation is an expression property, not an output-only feature. Any LLVM shortcut that converts a quoted token directly into a global string constant must first reject or dynamically lower `#{...}` markers, including boxed returns and array/object values.
  • A runtime benchmark is invalid when a supposedly compiled primitive facade launches the compiler for every call. Primitive-backed stdlib facades used in hot loops or worker threads need an in-process ABI path; retain JSON/compiler delegation only as a compatibility boundary for higher-level facades.
  • Receiver sugar must preserve prefix-unary precedence. Rewrite `!value.empty?` as `!Collections.empty?(value)`, never `Collections.empty?(!value)`; the latter changes both type and meaning and can fail only when the branch is exercised.
  • Cross-lane receiver sugar must be proven on dynamic values, not inferred from literal examples. When comparator-free dynamic sorting and module-qualified callable values disagree across hosts, an ordinary immutable loop is an honest application-level fallback while the compiler gap remains documented.
  • A 12-core maintainer workflow should propagate one explicit worker budget to build and isolated release/compiler lanes, but shared trees, caches, and output paths must remain ordered until isolation is proven.

Native Android view-stack extraction

  • A JSON screen manifest is sufficient to prove native navigation, lists, forms, secure fields, and accessibility without making the manifest vocabulary language syntax.
  • Keep the platform renderer in a generated host template and the application policy in ordinary Hey; this makes package extraction possible without changing compiler semantics.
  • JNI registration should carry generated callable and class identity, while the Java host should return actions immediately to Hey and avoid owning product policy.
  • Secure input widgets prove UI behavior, not secure persistence. Keystore and biometric capabilities require separate opaque platform contracts.
  • Do not call a generated project a device receipt until the NDK library, APK, install, Activity launch, and callback have executed on Android.

Ordinary-Hey Android application closure

  • A literal manifest and integer action ABI can prove the complete source-to-JNI path without pulling Android classes or annotations into Hey source.
  • Cross-target ELF objects that contain only exported application functions can be generated portably and linked by the NDK-selected CMake build for each ABI.
  • Register JNI methods in `JNI_OnLoad` instead of encoding application package names into exported C symbols; this keeps application identifiers porcelain-owned.
  • A generated Android Studio project is stronger than structural ELF evidence but remains weaker than a built APK and native emulator/device receipt.
  • Keep Activity/view rendering package-shaped. Target triples, ABI MIR, object emission, and safe callback entry are the irreducible Hey-owned plumbing.

Release-gate regression closure

  • A public query-suffixed builtin must be recognized by every name-dispatch and type/lowering table; retaining only an internal compatibility spelling can leave the REPL and promotion suite inconsistent.
  • Product archive staging cannot use one fixed checkout-relative directory when release contracts run concurrently. Allocate an identity-independent unique stage per invocation and remove only that stage.
  • A relocatable compiler closure should preserve internal aliases but prune symlinks that intentionally point back into the host operating system. Copying those aliases into a second root makes a later fail-closed SDK copier reject an otherwise valid toolchain.
  • Exact source-shape contracts age quickly after ergonomic migrations. Assert the maintained semantic boundary and machine receipt, not obsolete bracket syntax or retired library lists.

Native mobile ownership

  • Direct target object emission, ABI/callback safety, opaque handles, scheduler attachment, and target metadata are compiler/runtime plumbing.
  • SDK discovery, sysroot selection, build/link/sign/install/launch orchestration, and evidence receipts are small Hey CLI porcelain.
  • UIKit, Android Views/Compose, lifecycle facades, Keychain/Keystore, biometrics, notifications, and store helpers begin as independently versioned packages, not syntax or stdlib.
  • Do not promote a mobile abstraction inward from one application. Prove it on iOS and Android, extract a package, then consider stdlib only after multiple consumers stabilize the contract.
  • Keep Android evidence truthful: a JNI probe shell is not ordinary-Hey native application parity.

Interactive evaluation

  • Reflection and evaluation should form one tooling loop, but evaluation must reuse the ordinary frontend rather than become a second parser or language definition.
  • A callable returned from REPL evaluation must retain `<eval>` source metadata and the restriction policy for every nested stdlib call.
  • Denying only direct builtin calls is insufficient; capability restrictions must propagate through helper functions invoked by evaluated code.
  • Precompiled native applications should report eval availability explicitly and remain compiler-free.

Reflection metadata ownership

  • JSON convenience accessors may consume their receiver; defensive lookup must check each level without double-releasing values.
  • REPL source metadata belongs to the callable definition, not to special cases in the stdlib facade.
  • Missing metadata is ordinary absence and must return `nil`, never expose internal JSON-shape errors.
  • Physical REPL input lines and imported file declaration lines are distinct from executed-statement counters.

Application-driven extraction and physical devices

  • A generated mobile output directory is a disposable build receipt, not a third source repository.
  • Simulator, development-signed physical-device, IPA/TestFlight, and App Store approval are distinct evidence levels and need distinct commands and receipts.
  • On macOS, target compilation must select its SDK explicitly; inherited `SDKROOT` can silently combine a MacOSX sysroot with an iPhone target.
  • Extract application findings package-first. Promote to stdlib only after multiple independent consumers prove a stable application-neutral contract; change language core only when package/stdlib expression is insufficient.
  • Persistent private coordinators fit actors, keyed mutations fit partitioned Jobs, replicated validation fits workers, and large revision flows fit bounded streams.
  • Native target success should be emitted as an archive-bound machine receipt, not left only in terminal output. Bind it to complete source identity and archive SHA-256, and keep publication readiness false until signing, licenses, SBOM, and reproducibility are proven.
  • A fixture can prove archive mechanics but not release closure. Export the exact SDK only after its relocated compiler/run-bundle gate passes, then install that archive and compile from an unrelated directory with host discovery scrubbed.
  • Source-archive pruning must be impossible until the archive has passed integrity checks, full staging, sentinel/root validation, executable-mode checks, and a minimum-size sanity gate; extract the installer from the candidate archive itself so older checkouts can apply the safer process.
  • A browser learning surface should call the ordinary product compiler and enforce denied capabilities again at runtime; client-side token filtering is usability guidance, not a security boundary.
  • A native value's Hey representation is not sufficient to select its C ABI: opaque handles and signed integers may both appear as Hey integers. Generated descriptors must carry machine-readable argument and return tags, and nested module/predicate calls must canonicalize symbol identity before selecting LLVM call signatures.

Agent lessons

Semantic inference must fail conservatively

  • Unknown parameters and calls must remain boxed `Value` until source or HIR facts prove a narrower ABI; globally defaulting unknowns to integers hides representation bugs.
  • Compatibility spellings such as bare `now_ns()` need explicit semantic entries. Do not reintroduce an unsafe default merely because one maintained workload depended on it.
  • Qualified-call detection must respect identifier boundaries and zero-argument method shape so a name like `path` is not inferred callable from `relative_path(` and a module `collect(a, b)` is not rewritten as stream `.collect()`.

Application-shaped language ergonomics

  • Parsing JSON Lines and sorting records are independent general standard-library concerns. Keep application code declarative: decode/read records, provide a comparator, and encode/write results.
  • Comparator sorting must be stable and must call program-owned functions through every compiler/runtime lane; an interpreter-only higher-order helper is not a language feature.
  • Use dot access for statically known object fields and bracket access for dynamic keys, reserved receiver-property names, or bare local names that still collide with module-qualified callables. Enforce the distinction lexically so a migration remains clean.
  • Method-sugar parsers must reason about expression boundaries, not reject a receiver simply because its source contains spaces; array and object literals are valid receivers.
  • Heredocs improve generated-source construction only when emitted bytes and compiler phase behavior remain equivalent. Revert phase-sensitive conversions that leak interpolation markers into HIR/MIR.
  • Existing generic JSON float formatting can expose binary expansion; shortest-roundtrip serialization deserves its own compatibility-tested slice rather than an incidental formatter change.

SDK copying, cross compilation, and native extensions

  • A copied compiler executable is not a contained compiler. On Darwin, merge every non-system dylib root adjacent to separately installed LLVM/LLD tools, ad-hoc sign the relocated executables, and execute them before publishing a cross-SDK.
  • Do not mix a symlink-preserving first-stage SDK copy with a symlink-rejecting publish copy. Use one root-contained, non-traversing SDK-copy contract at every SDK boundary.
  • Cross-target truth has three independent levels: target object emitted, target executable linked, and target executable run natively. Never promote the third from evidence for the first two.
  • Long manually pasted verification sequences are part of the failure surface. Maintain one guarded script per complex matrix and stop at the first real failure.
  • A stdlib native facade must be recognized as a module call before receiver-method sugar and must carry explicit boxed HIR types; otherwise product MIR can silently diverge from the interpreter lane.
  • Native package handles are capabilities, not exposed pointers. Remove a handle from the registry before waiting for active calls so new work cannot race close.
  • Blocking metadata is an admission-control boundary. Reject blocking foreign calls on the ordinary path and require the application to choose a bounded worker/Job policy explicitly.

Measured scheduler changes

  • An atomic queue-depth hint can remove empty-victim mutex traffic without making the hint authoritative; the queue lock still decides ownership.
  • Bound ordinary steal probes, but retain a periodic full sweep so a local optimization cannot starve work on an unlucky peer.
  • Yielding after a failed probe round can suppress hot retry storms caused by short publication windows without adding a new scheduler thread.
  • Alternate old and new binaries on the same host. Separate one-worker preservation from multi-worker improvement and report medians, not one favorable run.
  • Do not promote registry partitioning when direct profiles show ready-queue and stealing costs dominate.

Preserve product truth

  • The public compiler is LLVM-hosted Hey -> semantic HIR -> ABI MIR -> direct LLVM. Do not reintroduce historical application backends into user docs or CLI choices.
  • Keep native Stage-0 and the C-hosted producer private for bootstrap, recovery, and byte-equivalent MIR evidence.
  • A run-ready native application is not the same as a self-contained compiler SDK. State toolchain, sysroot, runtime-library, and target limitations explicitly.
  • Do not advertise a platform until a native build/run/release gate exists there.

Keep actors honest

  • A lightweight actor is still a real mailbox, state, identity, and lifecycle—not a benchmark-only counter record.
  • Measure resident density separately from runnable throughput. A process ring often has only one runnable actor and is not a multicore scalability proof.
  • Warm fixed worker infrastructure before estimating per-actor memory.
  • Actor creation, scheduling, join, stop, and process shutdown must remain near O(1) or bulk-linear; small actor records alone do not prove scale.
  • A global ready queue can dominate fan-out even when actors are pooled. Profile lock contention before increasing worker counts.
  • Keep memory ceilings explicit and stop capacity tests before destabilizing the host.

Compiler/runtime ownership

  • Preserve mailbox ordering and wait for the final pooled actor step before releasing actor state.
  • Bulk shutdown may detach dormant actors, but must release messages, args, state, links, monitors, and registry ownership exactly once.
  • Do not release an object after a consuming/aliasing boxed store unless ownership is proven independent.
  • Live OS resources remain opaque handles; timeout and cancellation must be enforceable rather than post-hoc measurements.
  • Language semantics, HIR/MIR policy, scheduler policy, and workload algorithms belong in Hey where possible; the C runtime supplies general ABI services.

Performance and challenge integrity

  • Add general primitives only when they are independently useful and documented. Never branch on benchmark names, filenames, source shapes, or expected answers.
  • Compare byte-identical output and separate build/startup/map/scan/merge/format costs.
  • Alternate benchmark versions and report best plus distribution/mean; small changes can be noise.
  • Use target-machine profiles before adding more runtime surface.

Documentation and handoff

  • A source-development ZIP is a transport container, not a release identity. Accept source changes through tests and a reviewable diff; require only structural staging safeguards for destructive apply operations. Keep cryptographic checksums for immutable packages and published target artifacts, where byte identity is the contract.
  • Remove obsolete implementation narratives rather than labeling them “historical” in the maintained docs; Git preserves history.
  • Keep callable, actor, I/O, compiler, distribution, and site-generation docs synchronized with executable contracts.
  • Documentation layout must be owned by semantic local markup/CSS; optional CDN themes may decorate it but must not control the structural grid.
  • Preserve executable modes in source and distribution archives.
  • Keep copy/paste command blocks free of pager-producing `git diff --stat` commands.

Scheduler and archive lessons

  • Every pointer-backed MIR local that can be loaded in cleanup or a later block must be allocated in the LLVM function entry block; loop-local `alloca` instructions do not dominate function-level cleanup.
  • Scheduler telemetry must be opt-in and should not add unconditional atomics or shared-lock traffic to the performance path being measured.
  • Per-worker queues remove one central lock but do not guarantee speedup when actor turns are smaller than registry, mailbox, allocation, reference-count, and wakeup costs.
  • Remove scheduler experiments that regress repeated multi-worker results. A lower park count is not success if steals and elapsed time rise.
  • Never persist a static list of paths to delete before applying a future source archive. Derive deletions from the current repository's Git-tracked paths and the exact downloaded archive.
  • A release graph is not closed merely because its self-compiler binary exists. Run the complete distribution graph and execute staged tools from an unrelated working directory with source-checkout environment variables unset.
  • Custom `BUILD_DIR` values expose hidden bootstrap coupling. Every release compiler invocation must receive the exact release-local bootstrap path rather than falling back to wrappers or global `build/` artifacts.
  • Cached applications cannot infer an SDK root from their own cache path. The launcher must propagate the resolved installed root and stdlib path into the application process.

Concurrency surface

  • Use actors only for identity or persistent private state.
  • Use stateless workers for replicated parallel handlers and keep their mailbox runtime-private.
  • Put retries, timeouts, scaling, backpressure, dead letters, metrics, and supervision in Jobs rather than duplicating policy in actors.
  • Keep streams familiar (`map`, `filter`, `each`) while implementing parallel stages with bounded internal queues.
  • Do not add general-purpose user-facing channels or expose shared mutable state.
  • Reject worker state/mutation in semantic analysis, not after LLVM lowering.
  • Receiver methods that require parentheses must not reserve the same bare object-field name; `pool.status()` and `message.status` are distinct syntax.

Keep one concurrency runtime

  • Do not preserve an obsolete actor-backed Jobs API beside worker-backed Jobs; Git is the compatibility archive.
  • Jobs bind real worker identity, not strings or hidden reply actors.
  • Result waiting timeouts are not execution timeouts; label them honestly until work can be interrupted.
  • Bound result and dead-letter storage, and reject work rather than silently violating configured capacity.
  • Delayed retries must wait outside worker threads; one bounded timer queue is preferable to sleeping worker instances.
  • Idempotency memory must be bounded, and duplicates must reuse the original receipt rather than enqueue another handler turn.
  • Imported workers must preserve declaration identity through HIR/MIR; never fall back to function-name strings.
  • REPL examples must be replayed exactly, including whitespace and multiline call formatting.

Adaptive Job scaling

  • Scale worker-backed Jobs from observable queue age and bounded backlog pressure, not raw CPU guesses.
  • One shared controller can manage every adaptive pool; do not create one operating-system thread per pool or worker.
  • Keep fixed and adaptive configuration mutually exclusive so operational intent is obvious.
  • Rate-limit both growth and retirement, expose desired/current/peak counts and decision reasons, and make policy clocks deterministic in specs.
  • Internal release contracts are not substitutes for a real cross-platform vendor invocation; record external failures explicitly.

Stream ordering

  • Completion order can be observed by polling only the already-bounded receipt window; a public channel is unnecessary.
  • Carry sequence tags privately and cap every reorder group at the originating queue/result window.
  • Retries resolve inside Jobs before stream ordering; failed, cancelled, timed-out, or missing terminal results must fail the stream instead of manufacturing a sequence hole.

Partitioned Jobs

  • Keep partition routing inside the existing Job pool; do not create one thread, actor, or public channel per key.
  • A fixed shard count gives bounded allocation and shutdown. Canonical key hashing makes routing stable within the runtime contract.
  • Hash collisions honestly serialize extra keys. Expose per-shard load and contention rather than claiming perfect key distribution.
  • Partitioning guarantees non-overlap; it is not an output-order mechanism. Use sequence tags and `.resequence` when callers require ordered results.
  • Preserve a receipt's shard across retries so one key cannot overlap itself during retry handling.

Thin concurrency syntax

  • Lower declarative concurrency sugar before HIR whenever it has no independent semantics. `job Name ... end` must remain equivalent to one ordinary `Jobs.define(...)` binding.
  • Keep explicit construction available for conditional and function-local use; syntax convenience must not remove the underlying composable API.
  • Configuration blocks that contain `worker WorkerName` are not nested worker declarations; block scanners need a Job-specific terminator rule.

Measurement-first scheduler work

  • Separate registry, mailbox, message-pool, queue, handler-time, and value-ownership costs before changing scheduler policy.
  • Profile counters must be opt-in; even relaxed atomics distort tiny actor turns when maintained unconditionally.
  • Repeated `1..cores` matrices are more useful than one best run. Preserve raw runs beside medians.
  • A plausible bottleneck is not a measured bottleneck. Do not partition the registry when queue locks and steal misses dominate the observed workload.
  • Long verification blocks are unsafe to paste while a foreground build is still reading terminal input. Prefer one maintained synchronous verification command that stops at the first failing phase.

Post-Windows learning platform

  • Documentation needs layered learning paths: fast start, guided tour, task tutorials, explanations, and reference.
  • Preserve third-party challenge provenance separately from maintained Hey-native exercises, and review redistribution rights before publishing copied text or solutions.
  • A browser REPL must use a constrained capability profile and resource quotas; browser convenience cannot become a second language definition.

Generation-2 scheduler

  • Publish global ready-depth accounting before making a queued actor available to a concurrent popper; otherwise a pop can observe zero, skip the decrement, and leave stale positive ready depth.
  • Keep external injection separate from worker-local continuation queues. This preserves locality without forcing outside threads to select or lock one worker queue.
  • Drain and steal bounded batches so one queue lock can transfer useful work without monopolizing a victim or creating an unbounded local burst.
  • Running, spinning, and parked counts need explicit transitions and a spinner cap; an unbounded set of idle stealers simply recreates the contention the local queues were meant to remove.
  • After the largest measured bottleneck is fixed, structural scheduler improvements may produce modest timing gains. Retain them only when correctness, accounting, and one-worker behavior remain sound.

Persistent reactor ownership

  • Keep one native epoll/kqueue registration set alive for the reactor lifetime; rebuilding the complete set per wait adds churn and weakens ownership reasoning.
  • Duplicate live descriptors at admission so closing or reusing the caller's descriptor number cannot retarget an in-flight readiness request.
  • Unregister every descriptor before freeing its registration record or closing the duplicate. Native event userdata must never outlive its request storage.
  • Stop and join the reactor before draining actors and workers so cancellation completions are delivered while their managed recipients can still run.
  • Validate shutdown with pending requests in a compiled or process-lifetime test; an interpreter spec may evaluate expected output before global runtime shutdown.

Native database and cross-target compiler-cache lessons

  • A compiler-emitter cache keyed only by source and language version can collide across operating systems and architectures. Include the concrete Stage-0 executable identity and place release caches below the target-specific release directory.
  • A status-127 child failure during cross-target bootstrap can mean an incompatible cached executable, not a missing SDK or failed semantic lowering. Inspect cache provenance and binary format before changing the compiler pipeline.
  • Database compile/load proof, expected connection-refusal proof, and a configured live-server transaction receipt are three different evidence levels. Never promote the third from the first two.
  • TLS intent and negotiated TLS are distinct. Keep disabled/preferred/required/certificate-verifying modes explicit and expose a negotiated-session check.
  • Connector results own borrowed row/field views. Keep each connection and all derived result handles in one bounded or partitioned Job lane, and free results before closing the connection.

Long compiler-contract matrices

  • A serial wrapper that takes hours is itself a release-risk surface. Run independent compiler contracts through a bounded parallel worker count, retain one log per spec, print results in deterministic path order, and fail if any spec produces no success marker. Parallelism must not remove, reorder semantically, or weaken a contract.

Interactive shell and target-receipt lessons

  • Never paste bare `set -e` or `set -eu` into a user's interactive shell. The option persists after the block and can make a later expected test failure terminate the whole terminal session. Run strict mode inside a child `sh -eu` process and keep commit steps inside its success branch.
  • Structural compiler contracts and native target receipts are different evidence. A structural matrix may explicitly skip unavailable target-host prerequisites, while the direct promotion command remains strict and must execute on the target before promotion.
  • Cleanup traps should preserve the original status without calling `exit` recursively from an EXIT/signal handler. Trap EXIT for normal diagnostic cleanup and keep signal policy separate.

Target archive and launcher lessons

  • A one-click archive must derive its root from its own launcher, not from a prescribed ZIP/folder name or a command copied from documentation. Keep PowerShell internal, expose one obvious CMD entrypoint, and distinguish test execution from prerequisite installation/elevation.
  • A deterministic archive and valid checksum are not publication evidence. Keep file inventory, third-party/license inventory, native receipt attachment, SBOM completeness, and signing status as separate machine-readable fields, defaulting publication readiness to false.
  • A target archive should have two checksum layers: per-payload checksums verified before installation and an archive-level digest for transport/publication. Installers must remove only command links they own, never arbitrary files at the same PATH location.
  • MSVC-compatible compiler drivers need an explicit `/link` boundary before `.lib` inputs. Without it, `cl.exe` and `clang-cl.exe` can hand system libraries to the C frontend and report misleading source-file C1083 failures even when the Windows SDK is installed correctly.
  • A launcher located at the archive root must derive its root from its own directory, not its parent. Test the exact extracted layout; checking only that files exist inside the ZIP does not prove the launcher resolves them.
  • Promotion harness prerequisites are not product prerequisites. Installing Visual Studio to compile a receipt may be acceptable temporary evidence, but it must not become the Windows language installation story.
  • A relocated compiler cannot rely on driver-default linker discovery after `PATH` is scrubbed. Stage the linker, require it in the distribution contract, and pass its exact path to the compiler driver.
  • Go-style distribution means one target archive and no external compiler; it does not mean all compiler, standard-library, runtime, linker, sysroot, and metadata assets must be fused into one opaque executable.
  • A large performance goal must be attached to baseline workloads and correctness gates. Report 20x only where the same workload actually improves 20x; do not reinterpret it as “20x faster than optimized C.”

Endpoint security, signing, and target publication

  • An identical endpoint-protection popup across several executables does not identify a runtime API. Run a shared control image, preserve distinct hashes/signature/zone state, continue the complete matrix, and correlate native policy event logs before changing code.
  • A successful compile followed by PowerShell `Access is denied` before the child emits any output is not evidence of a compiler failure. Record the executable hash, signature status, execution root, and exact exception, and keep native promotion false.
  • Never disable CrowdStrike, Defender, Gatekeeper, application control, or similar policy to obtain a release receipt. Use an approved clean host, approved policy exception, and production code signing.
  • Run a minimal launch probe before a broad runtime receipt. If a kernel32-only executable cannot start, the host cannot distinguish runtime defects from launch policy.
  • Do not execute promotion artifacts from Downloads when the intended product installs elsewhere. Stage them under the product's versioned install root, while preserving policy and recording the location.
  • Authenticode signing must use SHA-256 for the file digest, an explicit RFC3161 timestamp URL, and post-signature verification. A self-signed or merely present signature is not equivalent to a trusted publisher.
  • Cross-host archive inspection may verify layout, checksums, modes, and symlinks, but only a matching native host may claim compiler execution or promotion.
  • Portable Unix installers should accept both GNU `sha256sum` and macOS `shasum -a 256`; requiring GNU coreutils on the recipient defeats a self-contained Darwin distribution.
  • One target release command should consume the exact SDK exported by the strict closure gate. Never rebuild a second, similar SDK after the receipt and assume they are equivalent.

Command porcelain and target evidence

  • A source checkout should not expose one top-level executable per implementation helper. Keep a small stable porcelain (`hey`, `heyc`, `test`, `perf`) and dispatch to private `bin/subcmds` implementations.
  • Product target archives, structural cross-SDKs, vendored application copies, and native runtime-test harnesses are different artifacts. Do not hide those evidence levels behind one ambiguous `release` spelling.
  • A native-host mismatch regression must deliberately select a non-native target. Accidentally choosing the current host can turn a structural guard test into a slow or prerequisite-sensitive release run.
  • When endpoint policy allows a tiny executable but blocks a rich aggregate, run independently compiled components least-to-most before changing runtime code. The first blocked component is stronger evidence than another undifferentiated aggregate denial.
  • Dynamically loaded Windows APIs should not also be linked through their import library unless the binary actually needs static imports. Smaller import surfaces improve portability and make endpoint-policy evidence easier to interpret.
  • A separately installed Darwin `ld64.lld` carries its own `@loader_path/../lib` runtime closure. Stage the library trees adjacent to both Clang and LLD; a copied linker binary plus LLVM-only libraries is not relocatable.

Alpha ergonomics migrations

  • A repository-wide syntax rename is not complete when the source merely parses. Exercise every renamed public predicate through interpreter, native-C comparison where supported, and direct LLVM; specialization tables can otherwise retain an old spelling and silently return the wrong value.
  • Query functions should end in `?`; commands that validate, assert, print, mutate, or return rich Result values should not receive `?` merely because one path returns a boolean.
  • Reject `== true` and `== false` in maintained Hey source. Use the value directly or prefix `!`, and enforce the rule with a compiled Hey-owned audit rather than a one-time search.

Resumable native release closure

  • A native target closure may legitimately outlive one shell, CI, or tool-execution window. Preserve the exact exported SDK and closure log instead of rebuilding a merely similar artifact.
  • Bind resumable work to a deterministic complete source-archive digest plus version, target, and host. Refuse stale work rather than guessing from timestamps.
  • Print stage boundaries before long captured commands. `prepare` and `finish` are product-release phases, not weaker evidence levels; both consume one identical SDK.

Release-integrity throughput

  • Correctness checks on product-sized archives are part of the release critical path. A byte-correct algorithm compiled without optimization can still make the release operationally unusable.
  • Keep file digests streaming and bounded, and use the platform crypto provider already required by the runtime. Retain portable internal hashing where bootstrap independence is the actual constraint.

Archive-stage checkpoints

  • Native closure and archive staging are separate expensive boundaries. Preserve a checksum-complete stage receipt before tar creation so command/session interruption cannot force another full SDK copy.
  • A reused stage must match version, target, and complete source identity. Never infer reuse from directory presence alone.

Semantic inference and entry-exit lessons

  • A parameter name appearing inside another identifier is not a callable use. Resolve call boundaries or canonical symbols; substring matching can turn `path` into a callable because `relative_path(` contains the same characters.
  • Unknown semantic values should remain boxed until an operation proves a scalar or string contract. Defaulting every unknown occurrence to an integer produces invalid ABI MIR for dynamic collection loops.
  • Stream receiver sugar must validate the method's arity. A namespaced call such as `PackageIndex.collect(root, names)` is not zero-argument stream `.collect()`.
  • Typed string calls and boxed object properties can occur in one expression: `Text.trim(result.stdout)` returns a raw string while `Result.ok` remains boxed property access. Preserve the HIR/MIR fact instead of applying raw-string-first source-shape matching broadly.
  • Program termination and actor termination share source spelling but not ABI. Route program `exit` through the `i32` main cleanup epilogue; keep actor exit on the actor lifecycle path.

Recursive expression and tail-position lessons

  • Recursive functions cannot infer their return ABI from a self call while that same ABI is still unknown. Seed the fixed point from non-recursive base returns, then infer the complete body.
  • An expression beginning with a call is not necessarily one call. Resolve depth-zero operators before treating the identifier head as the expression's complete callable shape.
  • Tail-call eligibility is structural: the matching close parenthesis of the self call must end the complete return expression. A self call used by addition, comparison, indexing, or another call is an ordinary call.
  • Keep ordinary-recursion and `musttail` regressions paired. Fixing one call path can silently box or over-classify the other even when small semantic specs still pass.
  • Current focused verification must execute the newest product regression directly; relying only on historical broad suites allowed the v0.99.360a package-index receipt to miss a self-tail typing regression.

Race-safe release verification

  • Case-insensitive filesystems cannot prove a case-only rename with `exists?`;

enumerate the parent directory and compare exact entry names.

  • Process regressions should not invoke `true`, `env`, a shell, or another

operating-system command. Compile one tiny repository-owned child fixture and

pass its exact path to every host and target lane, including Windows.

  • When a raw `posix_spawn` probe returns `ENOENT`, validate the executable path

before changing runtime signal or file-descriptor code.

  • Parallel verification requires isolated writable parser, compiler, runtime,

sidecar, and output roots. Prewarm an immutable compiler template once and

clone it per worker rather than sharing a publication directory or rebuilding

the same compiler in every chunk.

  • A failed prerequisite invalidates promotion evidence. Stop before promotion,

retain the durable failure receipt, and never spend minutes running a gate

whose result cannot authorize installation.

  • ThreadSanitizer counters and diagnostics are shared state too. Use atomics for

concurrent error accounting even when the counter is only test-visible.

  • Module migrations must distinguish a qualified module root from nested object

fields. A token such as `message.result.ok` is not a use of module `Result`.

Bounded nested handoff verification

  • A regression that creates temporary Git repositories should disable only

external commit side effects such as signing and hooks inside those repos;

preserve ordinary system/global Git policy unless a specific setting is

proven to make the test nondeterministic.

  • Test-only processor execution must not inherit production release policy.

Ignore outer `HEY_RUN_PROMOTION` values in test mode and require a distinct,

explicit test-only opt-in when promotion behavior itself is under test.

  • Redirected child output without a named phase turns a deterministic failure

into a silent outer `set -e` exit. Preserve each child transcript and print it

automatically when the enclosing regression fails.

  • Run source prompt, archive, installer, and nested-handoff checks immediately

after the clean build. A packaging defect should fail before the compiler and

platform matrix, not after it.

  • Repeated compiler regressions are not evidence for a wholesale rewrite by

themselves. First replace string-derived LLVM symbol identity with structured

semantic/HIR/MIR IDs and add phase-local deterministic fixtures; rewrite only

a component whose contract remains duplicated or untestable after that split.

  • A disposable compiler checkout must resolve its own adjacent source tree before consulting installed `HEY_ROOT`, and must pass that resolved root/stdlib explicitly to child HIR/MIR producers; otherwise a newer consumer can reject stale metadata emitted from another checkout.
  • A required-path checker is not protection unless the ordinary archive-producing command invokes it on the final bytes. Root-level files need an explicit source-archive allowlist entry, and publication must follow extraction/inventory/mode validation.
  • A nested broad regression invoked from parallel focused contracts must never

use a fixed writable output directory. Give each invocation an isolated run

root, preserve failed diagnostics, and allow an explicit stable output only

for callers that own serialization.

  • Do not hide a complete matrix inside multiple feature contracts. Focused

contracts should prove the feature and its inventory membership; run the

expensive authoritative closure once at the broad release tier.

  • An executable in a disposable checkout must prefer its adjacent source tree over inherited installer variables; otherwise a local binary can silently route commands into an older live checkout.
  • A native fast-path split changes contract ownership: `bin/hey` should prove only that it delegates non-script commands, while porcelain assertions such as locked package-command dispatch belong against `bin/hey-command`. A runtime regression can pass even when a static contract still inspects the retired owner.

v0.99.375a: simultaneous games need a reveal barrier

Do not stream private answers to the television as they arrive. Accept one

v0.99.376a: dictionaries are versioned application inputs

Word validation must not be hidden in a television client or an unversioned

v0.99.377a: validate once before every backend and preserve failing IR provenance

A public compiler entry point that delegates directly to another frontend can

skip semantic validation even when `--check` and `--run` are correct. Source

validation therefore needs a shared pre-HIR contract that is explicitly called

by delegated build paths. Backend subset errors must include source, function,

and canonical IR identity; otherwise unrelated application expressions collapse

into generic names such as `at` and encourage unsafe product-specific fixes.

v0.99.377a r3: validate archive casing before extraction

Post-extraction sentinel checks are insufficient on a case-insensitive host: a

lowercase ZIP entry can satisfy a CamelCase path and still fail on Linux. Audit

the ZIP central directory before extraction, validate the complete canonical

module ledger with exact names, reject entries that collide after case folding,

and compare the extracted file inventory before deriving tracked deletions.

v0.99.380a: inventory native debt before deleting it

A migration from duplicated native semantics to one LLVM product pipeline needs

a mechanical baseline before broad deletion. Inventory every native source,

classify the few files that still own language meaning, fail on new unclassified

files or semantic markers, and first remove native-C compiler-host selection from

normal release products. Keep Stage-0 recovery until the LLVM bootstrap fixed

point is proven; calling it private is not enough unless normal recipes stop

selecting it.

v0.99.380a: exact-case checks must inspect directory entries

On a case-insensitive macOS filesystem, both `test -e stdlib/Collections.hey`

and `test -e stdlib/collections.hey` succeed for the same file. Never use path

existence to validate a case-only migration. Validate the ZIP central-directory

entry, delete the old tracked alias before extraction, and inspect the actual

post-apply directory entry spelling with `find | sed` or an equivalent exact

listing.

v0.99.380a: package expansion must preserve builtin result facts

Imported forms must retain the same builtin semantic facts as inline source.

`find(String, String)` is an integer operation before and after module prefixing,

package provenance attachment, HIR emission, and MIR lowering. Do not repair the

loss in LLVM with a variable-name guess or an application conversion. Also

classify overloaded builtins such as `slice` from their operand facts rather

than assigning one representation globally.

v0.99.380a: filesystem casing and Git index casing are separate contracts

An exact directory entry can be correct while the Git index still records the

retired lowercase path. On `core.ignorecase=true`, plain `git add -A` may remove

the old entry without adding the case-only replacement. After archive apply,

force-remove both old/new spellings from the index and re-add only the canonical

path with `git -c core.ignorecase=false add`. Test the resulting `git ls-files`

paths and a clean commit, not only `find` output.

v0.99.382a: environment overrides must deduplicate against the inherited environment

`Process.run_env_argv`/`exec_env_argv` built a child's `envp` by copying the

parent's full `environ` first and appending the caller's overrides after.

`getenv()` returns the first match, so any override of a key already set in the

parent's environment was silently ignored — the override was present in `envp`

but unreachable. This stayed invisible in CI-style shells with no ambient

`HEY_*` variables, but broke every isolated-cache benchmark and test in a normal

developer shell that exports `HEY_CACHE_HOME`/`HEY_ROOT`/etc., producing

intermittent "could not find cached script executable" / "did not publish LLVM

output" failures that looked like compiler-pipeline bugs. Always drop base

`environ` entries whose key is also supplied as an override before appending the

overrides, so each key appears exactly once. When a subprocess-isolation test

passes standalone but fails only inside a wrapping harness (or vice versa), check

whether the two invocations differ in ambient environment before assuming the

regression is in compiler/runtime logic.

v0.99.382a: a retired-flag migration leaves stale literal-string spec assertions behind

Contract specs that grep source/regression-script text for an exact marker

(`--backend native-c`, a cache-key version literal like `hey-run-v6`, a phrase

like `stage0-comparison`) go stale the moment the checked-for code is

intentionally renamed, version-bumped, or split into two commands — the spec

then fails even though the behavior it was protecting is still correct or has

moved to an equivalently-scoped replacement. Before treating such a failure as a

regression, diff what the spec expects against what the referenced file actually

contains now; if the current text still satisfies the same intent (e.g. native-C

moved from `build --backend native-c` to explicit `stage0-build`, or a MIR cache

fingerprint bumped `v4`→`v5`), update the literal to match rather than loosening

or deleting the check. A batch of specs failing this same way right after a

"(KNOWN BUGS)" migration checkpoint is a strong signal the whole batch is this

class of staleness, not independent new bugs.

v0.99.382a: cross-unit calls resolve by parse-time closure inlining, never MIR linking

The self-hosted pipeline has exactly one MIR system and no linker: the parser

splices every transitively-imported file's forms into one merged tree, HIR/MIR

carry that single closed unit, and `build-mir` accepts exactly one MIR file.

`unit library` only relaxes the entry-point requirement; nothing consumes a

library MIR into another build. Making a stdlib module callable from compiled

code therefore means marking it compiled-in, not inventing a linker. A

compiled-in module needs BOTH halves of the Streams pattern: a parser-gate

exception in `hey_parser_external_runtime_import?` placed BEFORE the

`HEY_HIR_APPLICATION_MODE` check (product builds set that mode and it

force-delegates everything after it), and a source-path exemption in

`hey_hir_runtime_facade_form?`. One half alone silently does nothing.

v0.99.382a: the lowercase stdlib import allowlist is dead code

`hey_parser_stdlib_import_supported?` matches lowercase suffixes

(`stdlib/hey_semantics.hey`) with a case-sensitive compare, but since the

v0.99.364a CamelCase migration the resolver emits CamelCase paths

(`stdlib/HeySemantics.hey`) — verified empirically: no HIR golden contains any

inlined stdlib function, so every entry (including the `streams` special-case's

parser half) stopped matching at that migration. All stdlib calls in compiled

code flow through the runtime-facade op tables instead. Adding a name to the

dead list does nothing; new compiled-in modules must match the CamelCase

spelling the resolver actually produces. Reviving the whole list is a separate,

wide-blast-radius decision — do not do it as a side effect.

v0.99.382a: the parser cache does not observe the parser's own implementation

`hey_parser_cache_path_value` keys cached parse trees by schema string + cwd +

source path, and validity re-hashes only the parsed source files. Changing

parser behavior within a language version therefore leaves every previously

cached tree "valid" while stale — imports merged under the old rules kept being

served after the import-gating logic changed, visible only as compiled-vs-fresh

divergence (`forms=1` cached vs `forms=9` uncached). Bump the schema token in

`hey_parser_cache_schema_value` (`hey-parser-graph-vN`) whenever parser

behavior changes without a version bump, and suspect this cache first whenever

an interpreted parse disagrees with the pipeline's compiled parse of identical

source.

v0.99.382a: MIR guaranteed tail-call elimination is scalar-only

`__mir_tail_self` lowering in the MIR consumer supports scalar (i64/bool)

arguments; a self tail call carrying string parameters fails with "cannot

lower" on the parameter expression. Compiler stdlib modules that become

compiled-in must therefore write their scanners as ordinary while-loops rather

than the tail-recursive style used elsewhere — the loop form is also cheaper.

If a compiled-in module's build fails inside its own function bodies with

`__mir_tail_self` in the diagnostic, convert the recursion to iteration before

suspecting the emitter.

v0.99.382a: compiled-in module helpers need explicit type annotations

Untyped helper functions in a compiled-in stdlib module infer boxed locals

(`let line = trim(lines[index])` becomes `__hir_local boxed`), which the MIR

consumer then cannot lower for string operations. Annotate every helper's

params and return (`str`/`i64`/`bool`/`boxed`) — and when a string local must

come from a boxed array element, extract the access into a tiny helper with

an explicit `-> str` so the classifier gives callers a string local.

v0.99.382a: same-name locals with different types across sibling branches corrupt the compiled lane

Declaring `let label_a` as an integer in one dispatch branch and as a

string (a frame-token slice) in a sibling branch of the same huge

function made the integer reads resolve to nil in the selfc-compiled

lane while the interpreted lane stayed correct — emitting `if.thennil`

labels that only surfaced when a label-bearing program was built through

the product pipeline. Rename branch-local variables uniquely when their

types differ, and give every compiled-in-module emission lane a

byte-compare gate through the COMPILED pipeline (interpreted-only

verification cannot see host-compiler inference divergences). A

performance A/B harness, not a correctness gate, found this: measurement

runs exercise lanes the fixtures never did.

v0.99.382a: inline boxed-field integer coercion is not lane-stable

`'%t' + (record.field + 0)` inside a longer concat chain evaluated as

integer addition in the interpreted lane but as string concatenation in

the selfc-compiled lane, emitting `%t60` where `%t6` was meant — caught

only when the REPL-session specs exercised the compiled pipeline.

Always extract boxed record fields into typed locals

(`let n = record.field + 0`) before using them in concatenations; the

inline coercion idiom is safe only for strings (`+ ''`). Relatedly:

never commit a slice on receipt-tail evidence while its gate is still

running — wait for the aggregate `gate_exit`/`compiler_contracts_ok`

line; a slice was committed with a red gate this way and needed a

follow-up fix commit.

v0.99.382a: survey with a divergence check, not just a pass/fail tally

When an emitter's supported class widens, run the whole corpus through

both emitters and byte-compare every in-class result as part of the

survey — a program that newly enters the class can emit successfully

with subtly wrong output (here: program-fn calls wrongly flipping

runtime-required), and only the corpus-wide cmp catches it before a

fixture exists. Also: a direct call to a program function is not a

runtime-facade call; only hey_llvm_*/hey_runtime_* emissions flip

runtime-required.

v0.99.382a: byte-identity emitter migration starts from captured output

When replacing a native emitter with a Hey-owned one behind byte-identity

goldens, capture the native output first and mechanically generate the fixed

prelude into Hey source rather than transcribing it by hand (single-quoted

Hey strings are raw: backslashes and double quotes embed verbatim). Restrict

each slice's supported class so every transformation is identity (e.g.

literals whose native escaping is a no-op) and fail closed with no partial

output otherwise; parity then rests on a `cmp` gate instead of on

reimplementing escape semantics before any covered input needs them.

A new stdlib module imported by tools/hey_selfc_pipeline.hey must join

`heyc_llvm_direct_emit_compiler_library_record` (the HeyHir/HeyMir-style

compiler-library list — distinct from the parser-gate/HIR-facade halves)

or the selfc-compiled pipeline leaves `@hey_fn_<Module>__*` calls undefined

and the whole contract suite cascades. The LLVM subset also cannot lower a

string return that chains several call results with `+`; accumulate into a

local with one `set out = out + part()` per statement instead.

v0.99.382a: shallow native mirrors of stdlib functions are silently wrong

The native `HeyParser.parse_text` mirror understood only program-only

sources and returned wrong trees for anything with functions — compiled

callers got silently different results than interpreted ones. When a stdlib

function's compiled lane needs correctness beyond a mirror's subset, delete

the mirror and route the call through the stdlib JSON delegation path (which

runs the real Hey source); parity between interpreted and compiled behavior

is worth the subprocess cost for tooling calls.

v0.99.382a: compiled-lane parity must be gated per newly-classed example

The `!pool.enqueue(...)` negation had a compiled-lane temp-numbering

divergence (`%t141` where native says `%t15`) that predated the slice that

found it: the fixture had been verified interpreted-lane-only, and the selfc

smoke's compiled-lane parity loop covered just two programs. Two rules:

(1) when a survey moves an example into the supported class, add it to the

compiled-lane parity loop in `test/regression_selfc_llvm_emitter.sh` in the

same slice; (2) any `record.next_t + 0` read feeding further arithmetic that

must render numerically goes through the typed passthrough

(`hey_llvm_label_base_value(x + 0)`) — the bare `+ 0` idiom is not

lane-stable for record-field reads.

v0.99.382a: compiled-host arrays built with push() are not lane-safe

A `[]`-seeded boxed array push()ed with string variables and later

element-read (`arr[i] + ''`) works interpreted but renders RAW POINTER

ADDRESSES in the selfc-compiled host — silently, until arithmetic on the

garbage crashes (`slice expects non-negative`). split()-produced arrays

read correctly. For dynamic string collections inside compiled-in

modules use newline-joined string ACCUMULATORS plus one split() at the

read site, with an explicit count variable (split('') yields [''], not

[]). Corollary: a compiled-lane parity check that compares two outputs

is VACUOUS when both sides silently took the same fallback path —

assert the ROUTE (receipts) as well as the bytes.

v0.99.382a: launder record-field reads BEFORE arithmetic, not around it

The typed passthrough only stabilizes what reaches its parameter: write

`hey_llvm_label_base_value(x.field + 0) - 1`, never

`hey_llvm_label_base_value((x.field + 0) - 1)` — in the second form the

subtraction still runs on the unlaundered field read inside the argument

expression and raises `` `-` expects two integers `` in the compiled

host (this was the sole cause of the last receipts fallback,

markdown_tooling). Grep target for audits: `((` around a `+ 0` read

feeding arithmetic inside a passthrough call.

Version-bump corollary (v0.99.383a): anything EMBEDDED IN OUTPUTS is

part of cache identity. The language version appears in every emitted

HIR/MIR/.ll header, but the selfc pipeline fingerprint did not hash

stdlib/Version.hey — so after a version bump, cached sidecars kept

emitting the old version string and the selfc parity cmp failed at

line 1 while every fresh-emission gate stayed green. The fingerprint

now hashes Version.hey; when adding any new output-embedded datum,

add its source to the fingerprint in the same slice.

Sibling (probe-proven on one_brc_single): even after `let x = field + 0`,

a parenthesized addition on that record-field-derived local INSIDE a

string-concat chain compiles to string concat in the selfc host —

`'%t' + (x + 1)` rendered `%t41` where `%t5` was correct, while

`next_t: x + 2` in the same return record stayed numeric, and `(lc + 1)`

on a clean i64 param in the SAME emitted line was also fine. The trigger

is specifically field-derived operand + concat-chain context. Hoist the

arithmetic into its own local first (`let x1 = x + 1`) and concat the

local. Found by the first genuinely-live compiled-lane parity run over

actors; the interpreted lane never sees it.
