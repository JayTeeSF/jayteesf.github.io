Historical backend parity execution ledger

> This document records completed migration slices. Current product policy is one public LLVM-hosted Hey -> HIR -> MIR -> LLVM compiler, with native-C/C-hosted paths retained privately for Stage-0 recovery and MIR equivalence.

Backend parity execution plan (v0.99.36a → M3)

This is the working execution plan for closing the remaining `self-hosted` and `llvm` gaps against `native-c`, sequenced as one-alpha-version slices. It refines parity-plan.md (the standard) into concrete slices with fallbacks, and the remaining self-hosted compiler tier is detailed in self-hosted-emitter-tier.md. Keep this file updated as slices land; move completed slices to a short "Done" list with their measured gate deltas.

Context

JayTeeSF's latest v0.99.31a parallel backend-world gate finished with discovered=336 and reported native-c good=328/336 correctness=336/336, self-hosted good=314/336 correctness=320/336 with 14 build_failed and no output mismatches, and LLVM good=142/336 unsupported=194 build_failed=0 output_mismatch=0. v0.99.32a targets that evidence directly: it fixes the two remaining self-hosted expected-error subtraction rows, constant-folds compile-time `hey_c.emit_text`/`emit_file` rows, embeds current coverage/strategy report data, and fixes a bridge-marker false positive in the full-world harness. The remaining non-intentional blockers are the dynamic self-hosted dual-run/timing tier and the LLVM unsupported rows. The LLVM Rung 1/Rung 2 lowering plan continues with additive micro-specs and preserved existing specs.

The original 14 self-hosted build_failed rows were the **emitter-self-hosting tier**: rows importing `stdlib/hey_c.hey`, `hey_dual_suite.hey`, `hey_compiler_strategy.hey`. Import-following works (dedup + whitelist landed v0.99.9a); these three stay un-whitelisted as whole modules because (a) emitter defect classes remain when the emitter compiles itself (defect ledger item 14), and (b) emitting `hey_c.hey` can cost minutes per row. v0.99.32a closes only the safe constant-data subset and leaves `hey_dual_suite.run()` as dynamic harness work unless M4 IR emission or a direct cheap lowering closes it. LLVM's unsupported rows still cluster around broader language constructs, and parity-plan Track C mandates the typed IR become LLVM's real input (M3 target: llvm good ≥ 150).

Resolved design questions

  • **(A) Self-hosted completion:** hybrid. Land general perf primitives (in-place `push`, new `join` builtin) and fix the two defect classes as general contracts, then decide at a measured checkpoint: whitelist the trio only if per-row emission ≤ ~15s AND both defect classes are fixed; otherwise ledger the 14 rows as a named intentional exclusion closed by M4 (self-hosted emits IR). Skipping straight to IR emission does not dodge the cost: an IR emitter walks the same 2,400 lines with the same accumulator/clone costs.
  • **(B) LLVM growth:** IR-contracts-first, consumption-second, cheap wins not blocked. Amend parity-plan Track C with: *golden `.hir` fixtures for an op must land before or with any backend lowering of that op; a lowering may land in the line emitter first only if it calls exactly one runtime shim whose signature matches the spec'd IR op 1:1, so the later port to IR consumption is mechanical substitution.*
  • **(C) Expected-error parity:** mirror native's `heyc_emit_actor_registry_helpers_c` (`runtime/heyc.c:~4023`) in `stdlib/hey_c.hey` so `spawn_named` of an unknown actor compiles and fails at **runtime** (`hey_runtime_error`, exit 70) exactly like native (spec 361 contract).
  • **(D) Gate-time guardrail:** `--jobs N` parallelism in `bin/subcmds/hey-backend-world` (per-file work units, deterministic merge; timing noise permitted per ledger item 8, classification loss never), a per-row build timeout classified loudly as `build_timeout`, a top-10-slowest-rows report, and item-9 oracle hardening (cross-check native's captured output against `expect stdout` where a spec exists → `native_reference_suspect`, loud failure).

v0.99.36a impact-analysis update

JayTeeSF's v0.99.32a full gate shows LLVM still at good=142/336 with unsupported=194, while self-hosted is now good=325/336 and correctness=333/336. The next LLVM decision should be based on weighted payoff, not raw unsupported-row count. `tools/hey_backend_world_focus.hey` now emits `llvm_lowering_impact_*` rows after a full backend-world run by combining the TSV with per-row LLVM build logs. The first probes of the slow parser/bootstrap rows all stop on string helper/function-param blockers (`starts_with`, `ends_with`, `trim`, string values inside scalar function bodies), so the likely 80/20 slice is string-param/string-return lowering for parser/syntax/bootstrap code before actor or jobs lowering.

Workstream 1 — self-hosted

S1 (v0.99.14a): string-builder primitives — in-place `push` + `join` builtin + emission measurement

  • Micro-specs first: `634_join_builtin.hey` (join array with separator; empty; non-strings stringified like `hey_add`), `635_push_value_semantics.hey` (pins immutable-value contract: `let a=[1] let b=a set a=push(a,2)` leaves `b==[1]`).
  • `runtime/hey_runtime.c`: the runtime ABI is callee-consumes-arguments, so `hey_json_push` (~:1580) cloning the whole array per push is redundant — append in place on the owned argument. Audit the call sites first (`hey_repl.c:~532`, `:~1033`, `heyc.c:~1052`, `:~1256`, plus `stdlib/hey_c.hey` routing). **Fallback:** if any site retains a reference (aliasing), add internal `hey_json_push_take` used only on proven-owned paths; public `push` keeps cloning.
  • Add `hey_join_builtin(array, sep)` + declare; wire interpreter dispatch (`hey_repl.c` builtin cluster), native emitter map (`heyc_emit_call_c`, `heyc.c:~1045`), self-hosted routing (`stdlib/hey_c.hey` expr router), REPL. LLVM: honest-unsupported until S5 specs the op.
  • Measure `hey_c.hey` self-emission cost before/after (probe, not a committed gate).
  • Verify: full suite, `sanitize-smoke` (in-place mutation must not create cross-thread sharing), repl-specs, full gate. Expected delta: +2 specs, no coverage change, measurement artifact.

S2 (v0.99.15a): self-emission defect classes + accumulator refactor

  • Class 1 ("function definition is not allowed here", 13×, cascade suspected): bisect `hey_c.emit_text` over `stdlib/hey_c.hey` fragments to a minimal reproduction (suspect body termination in `stdlib/hey_parser.hey` walkers); micro-spec `636_*` before the fix.
  • Class 2 (vanishing `let` with surviving uses, e.g. `direct_fn_call_before_access`): likely parser body-slicing drop, not the emitter (the emitter's unsupported-let path fails the build honestly); reduce, micro-spec `637_*`, fix at the true layer.
  • Perf refactor: convert `hey_c_runtime_emit_body_value` (:~2024), `hey_c_runtime_function_defs_value` (:~2095), `hey_c_runtime_actor_defs_value`, `_decls_` helpers, and `emit_tree`'s final assembly from recursion-with-string-accumulator to `while` loops pushing parts + one `join(parts, "")` (item-13 loop precedent). Statement-local `+` stays.
  • Also attach (same scanner class): native-c boxed emitter's escaped-quote `+`-chain gap (open ledger item) — apply the backslash-skip fix to the native chain splitter in `runtime/heyc.c`.
  • **Fallback:** if either class roots in deep parser architecture, stop at slice budget, ledger the analysis, and flip S3 to the ledger path. If the loop refactor breaks byte-parity on green rows, land defect fixes only.
  • Verify: full suite; byte-identical spot re-verification of green self-hosted rows; re-run measurement probe; full gate.

S3 (v0.99.16a): expected-error parity + run_failed cleanup + whitelist decision

  • Completed first part: `spawn_named`/`supervise_named` registry helpers in `stdlib/hey_c.hey` now mirror native-c's helper shape and route from the expression router. Unknown actor names build under self-hosted and fail at runtime with exit 70. Spec 361 is the existing `spawn_named` contract; spec 637 adds the `supervise_named` negative. Targeted backend-world for specs 361/637: native-c expected_error_ok=2/2; self-hosted expected_error_ok=2/2; LLVM unsupported=2.
  • Remaining: from the fresh TSV, identify the other self-hosted run_failed rows and wrong-mode expected-error rows; fix or ledger each by name.
  • Whitelist decision gate: if S2 classes fixed AND per-row emission ≤ ~15s → whitelist `hey_c.hey`, `hey_dual_suite.hey`, `hey_compiler_strategy.hey` one at a time with green importers re-verified per entry (item-12 rule), gate wall time ≤ ~16 min. **Fallback:** ledger the 14 rows ("emitter-self-hosting tier, closed by M4"), keep whitelist closed, plan proceeds.
  • Expected delta (primary): self-hosted build_failed 14→0, expected_error_ok 8/8, run_failed 6→0-2. (fallback): good 293→~297-300 with the tier named in the ledger.

Workstream 2 — LLVM

S5 (v0.99.19a started; extended v0.99.21a): hey-ir 0.2 — real ops + golden `.hir` fixtures + plan amendment

  • Completed seed: header `hey-ir 0.2`; explicit op-shape contract lines for data-model ops mirroring the existing `hey_llvm_*` shims 1:1 (const_json, get/at, put, len, has, spread, to_json, parse) and the text facade shims already used by S6 (`text.starts_with`, `text.ends_with`, `text.find`, `text.slice`).
  • Completed seed: golden fixtures for specs 620-628 and 638-642; new `make ir-golden-smoke` byte-diffs `./bin/heyc ir` output against those fixtures.
  • Remaining: structured if/while/for blocks (replace `control boxed` markers), `boxed` param/return kinds on fn defs, and the explicit LLVM IR consumer. v0.99.21a extended the op list with `keys`, `values`, and `entries`; v0.99.22a extended it with file/process/text parse facade signatures (`file`, `read_lines`, `command`, `write`, `text.split`, `text.parse_int`, `text.parse_fixed`); v0.99.24a extended it with allocation/type/shape/json codec signatures (`allocations`, `reset_allocations`, `type`, `shape`, `shape_id`, `json.encode`, `json.decode`).
  • **Fallback:** cut 0.2 scope to exactly what S6 needs; structured control flow deferred to S7 where the consumer forces the design.

S6 (v0.99.17a started; v0.99.18a continuation): LLVM Rung 2 text/file facades (op-shaped) — biggest single win

  • First blockers landed in v0.99.17a: LLVM now honors the `\#{` literal marker before interpolation and lowers `starts_with`, `ends_with`, and `find` through op-shaped runtime shims with micro-specs 638-641. v0.99.18a continued with module-qualified `text.starts_with`, `text.ends_with`, `text.find`, `text.index_of`, and `text.slice`. Completed next data-model method-facade step in v0.99.21a: object/array `has?`, `keys`, and `to_json` method sugar now lower through boxed shims. Completed next file/process/text-parse step in v0.99.22a: command, read_lines/lines, write target/mode, boxed file reads, text.split, text.parse_int, and text.parse_fixed now lower through general runtime shims. Completed next allocation/type/shape/json-codec step in v0.99.24a: allocations/reset_allocations, type, shape/shape_id, json.encode, and json.decode now lower through general runtime shims with exact-call classifier guardrails. v0.99.29a completed variable-carrying boxed literals, stdlib process/files/fs facades, and text trim. Continue with string `has?`, for-in, actor forms, and the remaining unsupported LLVM buckets.
  • Shims with S5-spec'd signatures: ends_with/starts_with/has?-over-strings-arrays, `command`, `read_lines`, write target/mode forms; route from `heyc_llvm_emit_expr`/`_emit_call`/boxed path. Declares land in the same edit (item-6 rule; `llvm-declare-smoke` guards). Micro-spec per facade.
  • **Fallback:** if rows have second blockers, produce a first-blocker histogram from the TSV note column and re-rank; never partial-lower with silent skips (item-5 rule).
  • Expected delta: llvm good 85 → ≥115 (stretch 135), unsupported ≤190.

S7 (queued): LLVM consumes IR for Rung 1

  • IR parser + lowering section in `runtime/heyc.c` (or new file): lower Rung 1 ops from `.hir` to the same shim calls; per-construct explicit switch — a construct is either ported (IR path) or line-emitted, never a hidden mixed mode; unported constructs stay honest-unsupported under the IR path.
  • New `make llvm-ir-consume-smoke`: byte-identical program output pre/post port for specs 623-628 + `examples/json_values.hey`.
  • **Fallback:** incremental port across S7/S8 is allowed; divergent constructs stay on the line emitter behind the explicit switch, recorded.
  • Expected delta: flat (refactor slice); M3 architectural precondition met.

S8 (started in v0.99.21a): Rung 2 method facades, then variable-carrying literals + for-in → M3

  • Port S6 facades to the IR consumer (mechanical by the op-shaped rule); then: variable-carrying literals (~25 rows), general for-in over boxed collections (IR `for_in` op), boxed locals in functions, keys/values/entries. Micro-spec + golden per op.
  • **Fallback:** if good ≥ 150 (M3) is out of reach on these buckets, pull Rung 4 minimal actors forward (spawn/send/receive/join echo as runtime-ABI calls — the IR models them as calls) instead of grinding long-tail text rows.
  • Expected delta: llvm good ≥ 150 (M3), unsupported ≤ ~140.

Cross-cutting — S4 (any time after S1; recommended before S6): world-gate guardrails

1. `--jobs N` in `bin/subcmds/hey-backend-world`: per-file work unit writing `$OUT/rows/$safe.tsv`, driven by `xargs -P`, deterministic sorted merge before the awk summary. Default `--jobs 1` (bit-identical); `bin/test` alias opts into parallelism. Expected 13 min → ~3-5 min.

2. Per-row build timeout via existing `time_cmd` plumbing → loud `build_timeout` classification (never a skip).

3. Top-10 slowest rows appended to the summary (cost trend per slice).

4. Oracle hardening (items 9/16): where a spec has `expect stdout`, cross-check native's captured output → `native_reference_suspect`, loud gate failure.

  • **Fallback:** if POSIX-sh parallelism is flaky on minimal containers, land 2-4 only; `--jobs` stays opt-in.
  • Verify: `--jobs 1` vs `--jobs N` TSVs identical after sort; wall times recorded.

Sequencing

S1 ──> S2 ──> S3 (whitelist decision; S4 buys its gate-time headroom)
 └────────> S4 ────────┘
S5 ──> S6 ──> S8 → M3
  └──> S7 ──┘
Independent starts: S1 and S5. One commit per slice.

Self-hosted M4 (self-hosted emits hey-ir 0.2) deliberately follows this window: it reuses S5's op set and S2's iterative-emitter architecture, and closes any rows S3 ledgers.

Verification protocol (every slice)

1. Micro-spec(s) before the lowering/fix (`specs/language/6xx`), green under native-c and each claimed backend.

2. `./bin/test --full` green; denominator changes noted. In final responses, run it in the last proof block before `./bin/test --dual-timing` and `HEY_BACKEND_WORLD_JOBS=8 ./bin/test --backend-world-full`.

3. Targeted gates for touched surfaces: `sanitize-smoke` (runtime/interpreter), llvm smokes + `ir-golden-smoke`/`llvm-ir-consume-smoke` (LLVM slices), repl gates (REPL surface), dual-timing/selfhost-coverage (self-hosted slices).

4. Run the parallel `backend-world-full` block last; paste summary lines plus problem_row source rows into slice notes; compare against this plan's predicted delta.

5. Green rows spot re-verified byte-identical after emitter changes; whitelist entries one at a time.

6. One commit per slice; version bumps only in carriers (item-7 rule); new ledger entries for new defect classes; parity-plan version map updated.

Done

  • **Self-hosted emit constants and expected-error parity (v0.99.32a)** — self-hosted C now constant-folds compile-time `hey_c.emit_text(...)` and `hey_c.emit_file(...)` calls when arguments are known strings, carrying simple string constants through `let` and invalidating them on `set`. Current `hey_dual_suite.coverage_summary()` and `hey_compiler_strategy.report()` lower to static data, while the dynamic `hey_dual_suite.run()` tier remains explicit. Subtraction expressions route through the boxed runtime path so intentional type-error fixtures fail at runtime like native-c. Backend-world bridge detection now matches only the exact generated marker comment, avoiding false positives for marker text used as data. No existing specs were removed.
  • **REPL real source imports and LLVM string-local arrays (v0.99.31a)** — REPL `import` now reads source files recursively, de-duplicates nested imports, defines imported functions/modules/actors, skips comments in executed bodies, and normalizes multiline object/array literals in imported function bodies. Existing REPL jobs import coverage was strengthened, and new REPL mini-spec `180_stdlib_hey_c_import` pins `hey_c.emit_text('inline', source)` interactively. LLVM now treats string locals inside array literals as string pointers, fixing the v0.99.29a mismatch rows `570_data_model_tiny_unicode_escape` and `619_self_hosted_unicode_string_escape_lowering`; added mini-spec `652_llvm_string_local_array_value.hey`. No existing specs were removed.
  • **S3 evidence/tooling guardrail (v0.99.29a)** — `backend-world-full` now emits `problem_row=...` lines directly after `winner_by_coverage` for every `output_mismatch`, `build_failed`, `build_timeout`, or `run_failed` TSV row. This preserves JayTeeSF's short final proof command (`HEY_BACKEND_WORLD_JOBS=8 ./bin/test --backend-world-full`) while still surfacing exact source paths for mismatch/build-failure triage. Added self-hosted-emitter-tier.md to document why `hey_c.emit_text(...)` is valid Hey syntax, why the current self-hosted failure is a backend/compiler-tier gap rather than a bad spec, and why specs must not be removed or bridged through native-c to hide it.
