Roadmap after the v0.99.269a package candidate

Product direction

Hey source -> LLVM-hosted Hey -> semantic HIR -> ABI MIR -> direct LLVM
immutable values + actors + stateless workers + Jobs + bounded streams

General-purpose channels are not public language primitives. Native database pointers remain opaque handles, and blocking connector work is admitted through bounded Jobs.

Completed implementation through v0.99.269a

  • **v0.99.249a-v0.99.257a:** adaptive Jobs, streams/resequencing, partitioning, thin Job syntax, measured generation-2 scheduling, and persistent reactor ownership.
  • **v0.99.258a-v0.99.259a:** compiler-contained Unix SDKs, explicit target sysroot/native-library closure, SDK-relative linking, and relocatable run bundles.
  • **v0.99.260a-v0.99.265a:** Windows x64 target metadata, Win32 platform/process/IOCP/WinHTTP boundaries, and a contained cross-SDK candidate; native Windows execution remains required.
  • **v0.99.266a-v0.99.267a:** generated native ABI and versioned Unix loader with typed invocation, blocking/thread-safety enforcement, and lifecycle closure.
  • **v0.99.268a:** SQLite package with prepared statements, typed bindings, transactions, explicit ownership, and ABI-v2 typed handle marshalling.
  • **v0.99.269a candidate:** MariaDB/MySQL Connector/C package with bounded timeout/TLS policy, deterministic diagnostics, query/result/transaction APIs, product-lane compile/load/refusal proof, nested boxed-boolean ABI proof, target-specific release compiler caches, a passing 641-file observable suite, HIR/MIR, 20/20 conformance, sanitizer/docs/archive gates, a passing 76-contract compiler wrapper, and a passing complete Release-C matrix through two compiler hosts and product distribution layout. Configured live-server and fresh Docker receipts remain required.

Receipt closure before the next slice

1. Run `test/regression_mysql_live_server.sh` with explicit `HEY_MYSQL_TEST_*` configuration and retain `configured=true`, native-C, LLVM, transaction, row, and TLS fields.

2. Rerun `./bin/hey-vendor --target linux-x64 --container docker` from a checkout that has already built/vendored Darwin, proving the foreign-emitter cache collision is gone.

3. Keep native Windows execution and contained SQLite source packaging explicit rather than blocking ordinary alpha development.

Planned slices

1. **v0.99.270a:** documentation rewrite and executable Tour of Hey.

2. **v0.99.271a:** RubyQuiz provenance preservation and Hey-native challenge catalog.

3. **v0.99.272a:** browser-safe embedded REPL/playground and integrated learning portal.

4. **v0.99.273a:** calendar, email, HTTPS/web, migration, configuration, and test-transport packages.

5. **v0.99.274a:** fifteen-minute EHY/SQLite/Jobs blog demo.

6. **v0.99.275a:** maintained event-scheduling application and reproducible Rails-class video with honest measurements.

See Package-level C FFI roadmap and post-Windows learning platform roadmap.

Constraints

  • Never weaken valid specs to promote a release.
  • Keep target dependencies explicit and SDK-relative.
  • Treat clean target-host execution receipts as stronger than structural contracts.
  • Keep public examples readable and ordinary Hey.
  • Do not begin the Tour slice while required v0.99.269a receipts are still being represented as complete.
