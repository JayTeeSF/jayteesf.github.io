5. Pre-retirement income support
- Added `incomePlans` to model earnings before retirement, with directed savings to accounts.
- A bug / incomplete implementation caused absurd growth and duplicated income treatment.
- The intended fix was:
  - add explicit `sourceId`
  - group repeated rows by `sourceId`
  - count gross income once per `sourceId` per year
  - allow multiple rows with same `sourceId` to split that one gross income source across several destinations
  - allow contributions by percent or fixed amount
  - treat `taxTreatment: "ordinary"` as ordinary income flowing into household taxes automatically
  - `endYear: 0` should mean continue until owner retires
  - percent values should accept either `80` or `0.8` for 80%

Important semantics for `incomePlans`:
- `name` is only a label
- `sourceId` is the grouping key
- repeated rows with the same `sourceId` represent one paycheck / income source being split
- repeated rows with different `sourceId`s are separate gross income streams
- `targetAccount` currently matches account name strings (fragile)
- a future improvement that was suggested but not yet confirmed is switching to stable `accountId` instead of matching by name
